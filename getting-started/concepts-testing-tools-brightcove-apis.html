概念：Brightcove API 的測試工具

在本主題中，您將了解Brightcove Learning Services提供的可幫助您測試和學習Brightcove RESTful API的工具。

工具清單

本指南中討論了以下工具：

shell 腳本 (cURL)

命令行工具是一個名為api-tester.sh獲取訪問令牌並進行 API 調用。api-tester 腳本使用 cURL 進行 http 調用。 cURL 內置於 Mac MacOS 和所有 Unix/Linux 系統。要在 Windows 上獲取 cURL，請參閱設置捲曲 .然後下載api-tester.sh 腳本

您可以在以下位置查看 shell 腳本的完整代碼獲取訪問令牌 .

運行 shell 腳本

下載Shell腳本後，將其解壓縮至所需位置，然後轉到同一文件夾中的命令行。要運行腳本，請鍵入：

	bash api-tester.sh

每次運行腳本時，都會提示您輸入5個變量：

  • 客戶編號（請參閱下面的部分獲取客戶憑證）
  • 客戶機密（請參閱下面的部分獲取客戶憑證）
  • 完整的 API 調用（例如：https://cms.api.brightcove.com/v1/accounts/12345678/videos
  • 請求動詞 - 以下之一：GET |發布|補丁|放置|刪除（默認為“獲取"）
  • 請求數據-用於POST，PATCH和PUT；對於GET和DELETE請求，您將其保留為空白，只需按返回

命令行工具將返回與在線工具 - 但是，它也會回顯完整的響應，如果請求由於某種原因失敗，它允許您查看標頭和更多信息。它還會回顯響應的原始JSON和打印精美的版本。

在線工具

可以找到在線測試工具這裡 .像命令行工具一樣，它會要求您：

  • 客戶編號
  • 客戶機密
  • 完整的 API 請求 URL
  • 請求類型
  • 要提交的 JSON 數據（如果有）

獲取憑證

要使用 Brightcove API，您需要正確的客戶端憑據，包括client_id和client_secret .見OAuth 概述以更好地了解客戶憑證流。

得到一個client_id和client_secret，您需要轉到 OAuth UI 並註冊一個應用程序：

您還可以通過 CURL 或 Postman 獲取您的憑據 - 請參閱：
