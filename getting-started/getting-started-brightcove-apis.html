介紹

Brightcove 平台 API 是 RESTful API，允許您獲取數據並將數據發送到 Brightcove 平台的各個部分。一些使用示例包括：

  • 將視頻攝取到Video Cloud中
  • 在Video Cloud上管理視頻
  • 為視頻雲帳戶創建自定義分析報告
  • 創建和管理Brightcove播放器
  • 創建Brightcove Live作業以進行實時流式傳輸
  • 在Zencoder中為視頻創建轉碼作業

RESTful API

如果您不熟悉RESTful API，則它們具有一些共同的特徵：

  • 通過使用不同的方法發送HTTP請求來發出請求，例如GET，POST，PATCH， 和DELETE，具體取決於執行的操作類型。
  • 響應通常採用以下形式JSON字符串，任何編程語言都可以將其解析為本地數據類型。

REST 的美妙之處在於 RESTful API 不依賴於任何特定的編程語言。只要您可以發出HTTP請求和解析JSON響應，該API便不會在乎您使用的是哪種語言。

CORS

您需要了解的一件事是，大多數Brightcove API並非CORS已啟用。您可以閱讀鏈接文檔以獲取完整說明，但結果是幾乎所有情況下的 API 請求都無法從 Web 客戶端（例如 Web 應用程序）發出。

仍然可以通過服務器端代理路由API請求來將Web界面與API結合使用，就像我們在此站點上的許多示例應用程序中所做的那樣，並在使用REST API。

您可以使用以下工具從自己的聯網計算機發出請求捲曲 , 郵差， 或者失眠

驗證

大多數平台API的身份驗證都是通過臨時OAuth2訪問令牌進行的。某些API改用永久api密鑰或策略密鑰。有關詳細信息，請參閱有關各個 API 的文檔。
