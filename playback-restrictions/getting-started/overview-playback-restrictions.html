介紹

無論您的內容是面向內部觀眾還是外部觀眾，免費還是貨幣化，Brightcove Playback Restrictions 都能提供合適的內容安全級別以適應您的用例，同時仍允許觀眾從任何設備、任何地方訪問內容。

此功能允許您定義用於流式傳輸內容的不同場景，包括：

  • 您的內容在哪裡可用
  • 內容如何流式傳輸
  • 何時可以查看內容
  • 誰可以觀看您的內容

查看博文：這不僅僅是安全——它關乎品牌誠信和您對客戶的承諾

哪些產品支持播放限制？

以下 Brightcove 產品支持播放限制：

  • 視頻雲：點播和直播
  • 虛擬活動
  • 活動
  • 從事
  • 畫廊
播放限制目前不適用於 Brightcove Beacon。

如何應用播放限制？

播放限制可以如下應用：

  • 資產級限制
  • 運行時限制
  • 許可證密鑰保護

資產級限制

資產級別限制在“播放權限"中定義。

當您想限制以下內容的流式傳輸時，您可以在資產級別使用播放限制：

  • 地區（地理限制）
  • 時間（調度）
  • 領域
  • 代理類型
默認情況下，所有新的、攝取的資產都將擁有播放權，允許隨時隨地向所有人流式傳輸。

地區

地理限制限制了您可以在世界各地查看流內容的位置。選項包括：

  • IP地址
  • CIDR
  • 郵政編碼（僅適用於美國郵政編碼）
  • DMA（僅適用於美國DMA）
  • 國別

TBD - 地理限制有一個全局規則，它定義瞭如下整體行為：

  • 當全局規則設置為允許所有，它將允許所有播放，您可以將區域列入黑名單。
  • 當全局規則設置為全部屏蔽，它將阻止所有播放，您可以將區域列入白名單。

時間

調度允許您設置可用性窗口以通過兩個選項流式傳輸您的內容：

  • 具體日期
  • 定期重複特定的日期和時間

領域

域限制允許您在要允許或阻止流的位置設置域。

代理人

代理限制可防止在有代理的情況下進行內容流式傳輸。有五種可用的代理類型：

  • 匿名-客戶端的IP地址不可用
  • 公用-可以從某個位置代理的多個用戶允許公用互聯網訪問
  • 企業-識別是否有多個用戶通過一個或多個中央位置代理，並且可以共享一個網絡上的IP地址
  • 透明 - 客戶端的 IP 地址可通過 HTTP 標頭獲得，但該值不一定可靠
  • 代管
隨著 VPN 提供商不斷改進他們的服務，解析 IP 地址可能會與您為阻止的 IP 地址定義的不同。正因為如此，代理阻塞並不是 100％ 的保證。

資產級用例

用例

以下是一些可能的用例：

用郵政編碼限制內容

  • 目標

    Shari（Sports League 的網絡製作人）希望通過郵政編碼限制內容訪問，以便體育賽事郵政編碼內的觀眾無法流式傳輸賽事

  • 解決方案

    Shari 可以定義播放權限以將美國每個地區的郵政編碼列入黑名單。由於內容是公開的（沒有中間件應用程序處理權限），因此需要將播放權限設置為不安全。定義播放權限後，Shari 會將這些權限分配給每個媒體資產。

按國家/ IP地址限制內容

  • 目標

    Dylan（CMO）希望按國家/地區限制內容訪問，以便觀看者可以從烏克蘭和韓國以外的任何地方進行直播。由於開發團隊位於韓國，Dylan 需要允許從韓國的特定 IP 地址進行流式傳輸。

  • 解決方案

    迪倫可以定義“播放權"來阻止烏克蘭和韓國，並添加允許來自韓國的IP地址。由於 Dylan 的內容是公開的，因此每個播放權限都需要允許不安全的訪問。定義播放權限後，Dylan 會將這些權限分配給每個媒體資產。

設置資產級別限制

播放權限允許您指定視頻播放的限制。這些權利將存儲在播放權限 API並且每個集合都被分配了一個唯一的 id ( playback_right_id）。一組權利可以與零個或多個視頻相關聯。

當一個視頻有playback_right_id，播放權限對像中指定的限制適用於對該視頻的所有請求。

此圖顯示了從用戶的視頻和許可請求到播放的流程。

資產級別的限制可以使用以下任一方法進行定義：

  • 使用 Brightcove API
  • 使用影音工作室

使用 Brightcove API

您可以使用 Brightcove 的 API 來定義資產級別的限制。這些步驟可用於 Brightcove Player、本機 SDK 或您自己的播放器。

  1. 首先，使用播放權限API創造一個限制。

  2. 然後，使用內容管理系統 API將限制與您的視頻資產相關聯。

  3. 有關詳細信息，請參見實施播放權文檔。

使用影音工作室

您還可以使用 Video Cloud Studio 定義資產級別限制。這些步驟可用於 Brightcove Player 或本機 SDK。

  1. 展開行政下拉菜單，然後選擇播放權。

  2. 有關詳細信息，請參見管理播放權文檔。

  3. 導航到媒體模塊，然後選擇名稱您的視頻的鏈接以查看其詳細信息。

  4. 向下滾動到播放限制部分並單擊編輯按鈕。

  5. 展開下拉菜單，然後選擇要與此視頻關聯的限制。

  6. 選擇節省 .

  7. 定義了播放限制並將其與視頻相關聯後，就可以配置播放器了。

    有關詳細信息，請參見配置播放器實施播放權文檔中的“部分"。

筆記

  • 使用播放權限，內容無需加密
  • 默認情況下，需要有效令牌來流式傳輸具有“播放權限"的資產。但是，您可以將“播放權限"設置為不要求令牌。
  • 默認情況下，所有資產都具有播放權，該播放權允許隨時隨地向所有人播放
  • 每個視頻雲帳戶最多具有100個播放權
  • 播放權與Video Cloud / CMS API限制（國家和日程安排）一起使用。資產的CMS限制在任何播放權之前均已通過驗證
  • 播放權限不適用於策略密鑰限制（域和地理位置）。在 Web 播放器或本機 SDK 中使用播放權限時，策略密鑰將被刪除。這意味著使用策略密鑰設置的限制將被忽略。
  • 播放權無法刪除；僅啟用或禁用

運行時限制

要在運行時限制流式傳輸，您將需要具有特定權限（聲明）的 JSON Web 令牌 (JWT)。此令牌通過每個流請求發送。

此功能要求您在用戶管理系統 (UMS) 和 Brightcove 的播放服務之間實施中間件解決方案。

可以在運行時設置以下限制：

  • 播放權限
  • 標籤
  • 內容清單
  • 特定用戶代理
  • 並發流限制
  • 設備限制

播放權限

您可以將播放權限分配給資產（請參閱資產級別限制 )，或者您可以在運行時使用prid宣稱。

運行時限制會覆蓋任何資產級別的播放權限。

標籤

在運行時，您可以標識具有特定標籤值的資產以允許流式傳輸。這些標籤定義為tags宣稱。您可以使用標籤對視頻進行分組媒體模塊在 Brightcove 的 Video Cloud Studio 中。

播放限制服務將檢查與每個視頻關聯的標籤。如果至少有一個標籤與JWT令牌中的列表匹配，則可以觀看視頻。

JWT 令牌中的標籤將作為標籤數組列出。

您只能在 JWT 令牌中使用 40 個標籤的數組。

內容清單

使用vids聲明，您可以在運行時指定資產列表以允許流式傳輸。

權利檢查

權利檢查在運行時根據視頻 ID 和/或標籤設置最終用戶訪問權限。這涉及在 JWT 中設置以下聲明：

  • prid
  • vids
  • tags

如果一個prid（播放權限ID）在JWT中設置，然後資產的播放權限將被覆蓋。

特定用戶代理

要允許來自特定用戶代理的流式傳輸，請使用ua宣稱。要使用它，您的內容需要加密，並且您需要設置一個帳戶標誌，該標誌需要在許可證上使用 JWT。請與您的客戶經理聯繫以啟用此帳戶標誌。

使用特定用戶代理時需要啟用許可證密鑰保護。

並發流限制

限制每個用戶的並發流會阻止查看者與沒有帳戶的朋友共享其憑據。通過並發流限制，您可以定義特定用戶在任何給定時間可以觀看的視頻流數量。

對於此功能，需要將內容加密為DRM或HLSe。

有關詳細信息，請參見限制每個查看器的並發流文檔。

將並發流限制與 HLSe 一起使用時，需要啟用許可證密鑰保護。

設備限制

在運行時，可以為用戶限制流設備的數量。為此，需要將內容加密為DRM或HLSe。

設備註冊是在發出DRM許可請求時發生的，這時為每個設備分配了唯一的ID。對於每個許可證請求，都會檢查並強制執行設備限制。

例如，發布者必須遵守內容所有者的要求，以將與單個查看器關聯的設備數限制為2個設備。達到該限制後，如果觀眾嘗試在第三台設備（如連接的電視）上流式傳輸內容，則播放將被拒絕。此功能使內容髮布者能夠實現貨幣化目標並最大限度地減少未經授權的訪問。

有關詳細信息，請參見實施設備限制文檔。

將設備限制與 HLSe 一起使用時，需要啟用許可證密鑰保護。

運行時用例

用例

以下是一些可能的用例：

允許付費訪問

  • 目標

    Liz（活動策劃者）只想允許付費用戶訪問活動的現場直播和VOD內容。她還想阻止鏈接共享。

  • 解決方案

    對於此用例，從平台請求內容的每個登錄用戶都需要具有以下聲明的令牌：

    • uid = "{unique user-id}"
    • climit = 1
    • cbeh = “BLOCK_NEW_USER”

    所有內容（實時或VOD）都需要使用DRM加密。

    有了這些權限，每個登錄的用戶都可以流式傳輸直播或 VOD 內容，只要他們在同一設備和網絡上即可。如果用戶在令牌有效的時間段內共享其平台憑據，則其他嘗試流式傳輸任何內容的用戶將無法執行此操作。通過該功能的中游檢查，Liz 可以確保整個內容流的這種行為。

阻止新流

  • 目標

    Harry（OTT客戶）希望確保其平台用戶能夠查看內容，但一次只能查看一項資產。

  • 解決方案

    對於Harry的用例，每個從其平台請求內容的登錄用戶都需要具有以下聲明的有效令牌

    • uid = "{unique user-id}"
    • climit = 1
    • cbeh = “BLOCK_NEW”

    所有內容（實時或VOD）都需要使用DRM加密。

    此案例的行為類似於 Liz 的用例。主要區別在於，當令牌有效時，Harry 的用戶將無法同時觀看不同的資產。通過該功能每 5 分鐘進行一次中游檢查，Harry 可以確保這種行為。

限制並發流

  • 目標

    Shari（Sports League 的網絡製作人）希望將並發流限制為 1，以針對特定內容。

  • 解決方案

    對於這種情況，每個登錄用戶都需要具有以下聲明的令牌：

    • uid = "{unique user-id}"
    • vids = ["asset 1","asset 2","asset 3"]
    • climit = 1
    • cbeh = “BLOCK_NEW”

設置運行時限制

要使用運行時限制，請執行以下操作：

  1. 實施中間件

    此功能要求您在用戶管理系統 (UMS) 和 Brightcove 的播放服務之間實施中間件解決方案。

  2. 創建一個JWT

    創建具有特定權限（聲明）的有效JSON Web令牌（使用您的私鑰簽名的JWT）。有關詳細信息，請參閱創建一個JSON 網絡令牌 (智威湯遜)文檔。

  3. 配置播放器

    令牌隨播放請求一起傳遞。如果令牌無效或已過期，則將限制對內容的訪問。玩家必須配置為使用令牌。

    網絡播放器

    有關配置Brightcove Web播放器的詳細信息，請參閱Brightcove Player的密鑰/許可證保護文檔。

    本機Android或iOS播放器

    有關為Android或iOS配置本機播放器的詳細信息，請參見使用本機SDK進行密鑰/許可證保護文檔。

    你自己的播放器

    如果您僅將JWT與播放器一起使用，則不需要額外的配置。如果您正在與播放器一起使用“播放權限"，請參閱配置播放器實施播放權文檔中的“部分"。

筆記

  • 要使用ua , maxip和maxu索賠，您需要以下內容：
    • 內容需要加密
    • 您的帳戶需要具有許可請求所需的 JWT（啟用許可密鑰保護）。有關詳細信息，請聯繫您的客戶經理。
  • 要使用climit和dlimit索賠，您需要以下內容：
    • 內容需要使用DRM加密
    • 您的帳戶需要啟用 CSL 並啟用檢查權限標誌。有關詳細信息，請聯繫您的客戶經理。
  • 每個令牌都有一個到期時間，並且在創建令牌時需要對其進行設置。您的集成負責保持續訂過程。
  • 默認情況下，流中檢查每5分鐘進行一次。

許可證密鑰保護

通過動態交付交付受DRM保護的內容時，許可證密鑰保護可提供更高級別的安全性。這對於想要控制對其內容的訪問並防止未經授權的內容共享的客戶特別有用。

DRM或HLSe內容保護使用許可證/密鑰請求，可以使用JSON Web令牌（JWT）保護每個流請求。

用例

當您的內容需要公開但來自受信任的請求時，您可以使用此功能。例如，具有使用HLSe加密的專有內容的區域廣播公司希望確保只有來自其平台的請求才能訪問該內容。這意味著觀眾只能從您的平台流式傳輸內容。即使使用流 URL，他們也無法將其流式傳輸到平台之外。

請注意，為具有 DRM 和/或 HLSe 保護的視頻實施播放權限將阻止視頻在 Studio 媒體模塊的預覽播放器中播放。為了啟用預覽，Brightcove 實施了明確的來源 CMS API 的端點。

如果內部或第 3 方用戶擁有有效的 API 憑據來檢索未受保護的 URL，則這些端點還可用於使他們能夠預覽視頻。但是，請注意：

  • 一旦不受保護的 URL 可用，Brightcove 和客戶就無法控制誰可以訪問內容或如何使用內容。
  • 訪問源時，將對您的帳戶收取出口帶寬費用。

啟用許可證密鑰保護

要為您的帳戶配置許可證密鑰保護，請聯繫您的客戶經理以啟用許可證密鑰保護標誌。

使用許可證密鑰保護

要使用許可證密鑰保護，請執行以下操作：

  1. 實施中間件

    此功能要求您在用戶管理系統 (UMS) 和 Brightcove 的播放服務之間實施中間件解決方案。

  2. 創建一個JWT

    啟用此功能後，解密內容的每個密鑰或許可證請求都需要一個有效的 JSON Web 令牌（用您的私鑰簽名的 JWT）。有關詳細信息，請參閱創建一個JSON 網絡令牌 (智威湯遜)文檔。

  3. 配置播放器

    令牌隨播放請求一起傳遞。如果令牌無效或已過期，則將限制對內容的訪問。玩家必須配置為使用令牌。

    網絡播放器

    有關配置Brightcove Web播放器的詳細信息，請參閱Brightcove Player的密鑰/許可證保護文檔。

    本機Android或iOS播放器

    有關為Android或iOS配置本機播放器的詳細信息，請參見使用本機SDK進行密鑰/許可證保護文檔。

    你自己的播放器

    要將 JWT 與您的播放器一起使用，不需要額外的配置。

筆記

  • 無需使用播放權限即可使用此功能。
  • JWT 令牌不需要特定權限（聲明）。只需使用客戶的私鑰進行簽名即可。
  • 您的內容需要加密 HLSe 或 DRM。

局限性

以下限制適用於此功能：

  • 對於HLSe（AES-128），令牌將附加到主清單請求中，並將傳播到加密密鑰URL。由於這些URL被附加到主清單中，因此客戶應考慮限制使用次數，以最大程度地減少其內容的顯示。
  • 對於HLSe（AES-128），播放器在播放視頻時會發出多個請求，通常每個再現一個。這馬蘇索賠需要設置得足夠高以解決這些請求。您應該考慮其他聲明，以盡量減少您的內容的曝光。切換演繹將請求新的許可證。
  • 跟踪maxu 和 maxip 聲明最終是一致的。這意味著如果使用這些聲明的令牌被快速連續重複使用多次，則可能會看到該令牌被接受的次數比這些聲明聲明的次數要多。跟踪在幾秒鐘內變得一致，在該窗口之後，令牌將按預期被阻止。
