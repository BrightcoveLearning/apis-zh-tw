錯誤訊息

HTTP代碼 錯誤代碼 描述 解析度
400 BAD_REQUEST 此錯誤表示發出的請求存在客戶端錯誤。該消息通常包含有關錯誤內容的信息。 這message響應的字段包含有關導致錯誤的原因的信息，例如`invalid value for sort範圍。
400 UNSUPPORTED_FIELD_COMBINATION_ERROR 此錯誤表明fields報告的參數包括一個或多個對該報告類型無效的字段。 這message響應的字段包含有關指定了哪些無效字段的信息。
400 TIME_RANGE_UNAVAILABLE 當歷史/批處理系統沒有您要求的數據時，會發回此響應。
401 UNAUTHORIZED 此錯誤表示用於發出請求的授權（OAuth2 或 LA 令牌）無效。 檢查正在使用的授權令牌。所使用的令牌可能超出了其有效的時間範圍，因此需要更新的令牌。通常，每個令牌會在5分鐘後過期。授權令牌可以指定為 url 參數access_token或在Authorization標題。參考管理API憑證更多細節。
404 NOT_FOUND 當請求的端點不存在時，通常會發回此錯誤。 檢查要求的端點。
405 METHOD_NOT_ALLOWED 當使用非 HTTP 方法發出 api 請求時會發生此錯誤GET . 向API發出所有請求HTTP GET。
500 SERVER_ERROR 當服務器上出現內部錯誤時會發生此錯誤。 當它不可用或面臨臨時問題時，服務器可能會將其發回。過一段時間再試一次，並進行一定程度的指數退避。如果問題仍然存在，請創建支持記錄。
503 PROCESSING 這意味著服務器已收到請求並正在處理它。 這分析 API如果遇到長時間運行的查詢，可能會發回此消息。查詢完成後，它將存儲在服務器的緩存中最多5分鐘。因此我們建議在收到此錯誤後 4 分鐘查詢 API
