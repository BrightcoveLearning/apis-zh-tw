參數

下表總結了分析 API .參數的使用將在以下各節中進行更詳細的討論。

範圍 必要 描述 價值觀 默認

帳目

您想要報告的 Video Cloud 帳戶是使用accounts範圍。例如：

  https://analytics.api.brightcove.com/v1/data?accounts={account1_id,account2_id}

哪裡過濾器

過濾器的一般語法是：

where=dimension1==value1;dimension2==value2

例如：

https://analytics.api.brightcove.com/v1?accounts=account_id(s)&dimensions=device_type&where=video==video_id;device_type==tablet

逗號被視為邏輯“或"，分號被視為邏輯“與"。例如，where=video==1234,5678;player==9876解釋為“其中視頻= 1234 或者 5678 和玩家= 9876“

空格和特殊字符

字符串值應該是 URI 編碼的。您還可以使用 "" 轉義特殊字符：

where=search_terms==boston,%20ma

您可以使用任何維度作為過濾器，但 只要 如果該維度也包含在dimensions你在請求。

在播放器上進行篩選時，請注意以下事項：

  • Brightcove玩家 -輸入以下格式的播放器網址：players.brightcove.com/account_id/player_id _default
  • 如果您使用了應用程式編號以識別播放器，然後以以下格式輸入播放器網址：players.brightcove.com/account_id/player_id_default/ application-id - 例如：players.brightcove.com/1752604059001/BJFYWrtUl_default/AlltimeViewsPlayerSample
  • 智能玩家 - 輸入玩家 ID

按視頻屬性過濾

使用特殊where=video.q=={property}:{value}過濾器，您可以根據各種屬性將報告限制為一組特定的視頻，包括：

  • 標籤
  • reference_id
  • custom_fields [1]
  • {a_specific_custom_field}
  • created_at

筆記

[ 1] 基本語法是where=video.q==custom_fields:value（匹配任何自定義字段中的值）或where=video.q==myfield:value（匹配特定自定義字段中的值myfield）。如果您要搜索特定的自定義字段，請注意，您必須在內部名稱，而不是顯示名稱：

快速檢查一下您是否使用了正確的名稱：內部名稱為全部小寫且不包含空格。

例子

這是一些例子where用於搜索標籤和自定義字段的過濾器：

單標籤
where=video.q==tags:foo
多個標籤：
where=video.q==tags:foo,bar
自定義字段
where=video.q==custom_fields:foo
標籤和自定義字段
where=video.q==tags:foo,bar+custom_fields:fish

請注意video.q搜索功能包括AND，OR和NOT邏輯如下：

  • 一種+搜索詞前的加號（加號）表示結果必須包括這個詞。
  • 一種--搜索詞前的（減號）符號表示結果禁止包括這個詞。
  • 如果沒有加號或減號，則結果可能包含也可能不包含該術語。

以下示例說明了此邏輯的使用。

搜索示例
where篩選 結果
where=video.q==tags:red%20tags:blue%tags:green 帶有標籤的視頻red或者blue或者green將被退回
where=video.q==+tags:red%20tags:blue%tags:green 返回的視頻必須具有標籤red AND可能有標籤blue或者green
where=video.q==+tags:red%20tags:blue%-tags:green 返回的視頻必須具有標籤red AND可能有標籤blue，但不能帶有標籤green
筆記：%20在上面的例子中是一個 URI 編碼的空間。根據您用於發出請求的客戶端，您可能還需要對+簽名為%2B .

有關此查詢語法的完整說明，請參見使用CMS API：搜索視頻 .

使用此過濾器獲取的數據會緩存 24 小時，因此不會反映過去一天所做的更改

過濾器和允許值摘要

下表顯示了用作過濾器的每個維度的允許值：

尺寸過濾器 允許值

日期範圍

日期範圍，在from和to所有類型的報告的參數可以用不同的格式表示：

  • 文本值：
    • to=now（可用以及所有請求的默認值）
  • 紀元時間值（以毫秒為單位），例如1377047323000
  • 以ISO 8601標準國際日期格式表示的日期：YYYY-MM-DD格式，例如2013-09-12。對於以這種格式表示的日期：
    • 將解釋指定的任何日期範圍 在為帳戶設置的時區中
    • 日期給出的時間將被解釋為午夜 ( 00:00:00 ) 在指定的日期 在為帳戶設置的時區中
  • 相對日期：您可以表達to和from相對於另一個值的天數（天）或小時（小時）。例如：
    • from=2015-01-01&to=31d
    • from=-48h&to=now
    • from=-2d&to=now（將提供與前面的示例相同的結果）
    • from=-365d&to=2014-12-31

    請注意，負數（-2d）被解釋為“在"之前（另一個值），正數（48h）被解釋為“從"（另一個值）

要生成某一天的某個維度（如“視頻”）的報告，請將 to 和 from 值設置為該日期：

...&dimensions=video&from=2013-11-01&to=2013-11-01
筆記：
  • 您可以使用端點獲取任何報告的可用日期範圍/data/status - 見API 參考詳情

限制和偏移

這limit是要返回的項目數（默認值：10）。要返回所有項目，請使用limit=all . offset是要跳過的項目數（默認值：0）。您可以使用limit和offset一起創建一個應用程序來瀏覽結果。

對帳數據

這reconciled參數是一個布爾值。如果設置為true，結果將僅限於已對帳的數據。如果false，結果將僅限於實時（每小時未對帳）數據。

實時數據僅適用於前 32 天。如果你設置reconciled=false和from date 早於 32 天前，API 將不會返回任何數據，因為此時沒有未協調的數據。

地理報告

地理分析的維度

  • country -作為ISO-3611-1國家/地區代碼。例如：'我們'
  • region -作為ISO-3611-2區域代碼。例如：'美國西澳'
  • city - 城市名。例如：西雅圖

筆記：對於未知的國家或地區，API返回“ZZ"作為代碼（根據ISO-3611-alpha2）。

字段和排序

使用fields參數指定要返回的字段。默認，video_view返回，並且對應於您要報告的維度的字段（例如，destination_domain）返回。看維度和字段更多細節。

使用sort參數指定使用哪個指標字段對返回的項目進行排序；例如：sort=video_view .您可以通過否定排序字段來反轉排序順序： sort= -video_view

可用於返回或排序的字段取決於指標針對您報告的維度返回。這些參數經過嚴格驗證，如果出現以下情況，您的請求將返回錯誤fields或者sort參數指向對請求無效的字段。

對於視頻尺寸報告，有效的排序值包括視頻元數據字段：video.name，video.reference_id， 和video.tags。

video_name 與 video.name

  • video_name是在分析數據中記錄的視頻名稱
  • video.name是視頻元數據中的視頻名稱

在大多數情況下，這兩個值是相同的，但在某些情況下它們將不相同：

  • 在某些情況下，您可能會看到video.name，但不適用於video_name -在這種情況下，由於某種原因，視頻名稱未發送到分析數據收集器
  • 在某些情況下，值可能會有所不同-在這種情況下，視頻已在某些時候重命名：video.name是您的視頻雲媒體庫中視頻的當前名稱；video_name將是將數據發送到Analytics（分析）數據收集器時的視頻的視頻名稱（如果您查詢名稱更改的時間段，則API返回的值將是與大多數視圖關聯的名稱在此期間）。

計算字段

您可以使用以下語法向 API 請求添加計算字段：

fields=calulated_field_name:expression

您可以使用計算字段根據現有指標創建自己的自定義字段，或重命名現有字段。

請注意，您可以不是將計算字段用作另一個計算字段的表達式的一部分。

計算字段的名稱可以是任何與 URI 兼容的字符串。該表達式可以包括常規字段名稱和以下算術運算符：

  • + （添加）
  • - （減法）
  • * （乘法）
  • / （分配）
  • ^ （指數）
  • () （括弧）

例子

fields=avg_seconds_viewed:video_seconds_viewed/video_view,video.name
fields=avg_incomplete_ads:(ad_mode_begin-ad_mode_complete)/video_view,video.name
fields=Video%20Views:video_view,video.name
