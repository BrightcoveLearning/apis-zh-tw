日期範圍

如何獲取特定日期的數據？
設置兩個from和to值到那個日期。例如：from=2013-05-12&to=2013-05-12。另外，您可以按date尺寸，可以與其他尺寸組合使用。
如果分析模塊和分析 API給我相同日期範圍的身份數據？
是的，但如果您比較的是相對較新的數據，並且您在不同時間從 Studio 和 API 獲取數據，則它們可能會有所不同：
  1. 數據不到 3 天。少於3天的Analytics（分析）數據是臨時的，隨時可能更改。
  2. 在小於32天的時間裡查看同一時期的數據，在小於32天的時候再查看一次。32天后，數據被移至單獨的“歷史"存儲庫，此時我們丟棄了一些數據，尤其是詳細的參與度數據。這意味著參與度分數等計算字段可能會略有變化，因為計算是基於較細粒度的數據進行的。

數據粒度

最近的 Analytics 數據的粒度是多少？
最近 32 天（包括當天），分析 API報告小時粒度，因為它將值保存到每小時存儲桶中。但是，當前時間仍在填充中，因此，當您要求輸入當前時間中的數據時，API看上去比一個小時具有更細的粒度。

例如：

  1. 如果您要求3小時前提供9:15和9:20的數據，並且它們屬於同一小時時段，則可能會獲得相同的值；但是，它們可能有所不同，因為所有分析數據都是臨時的，並且在對帳前可能會更改。
  2. 如果您要求10分鐘前的數據，然後等待5分鐘，然後再次詢問，即使它在同一存儲桶中，您也可能會獲得不同的值，因為該存儲桶仍在更新中。
分析數據的刷新間隔是多少？
/data 端點當前在查詢之間緩存了 5 分鐘，因此對於過去一小時內有流量的視頻列表，5 分鐘間隔是最小延遲，可以為您提供工作增量。
歷史分析數據的粒度是多少？
對於早於過去 32 天的日期，分析 API報告全天值。這意味著，如果您要求from=1368334306919&to=1378446336919（從格林威治標準時間 2013 年 5 月 12 日星期日 04:51:46 到格林威治標準時間 2013 年 9 月 6 日星期五 05:45:36），您將獲得與請求相同的結果from=2013-05-12&to=2013-09-06

項目和領域

判斷一個或多個視頻在一段時間內有多少流開始的最佳方法是什麼？
只是看看video_view指標-這將始終等於流開始。您可以在所有報告中看到此指標，摘要將顯示查詢日期範圍內所有視頻的流開始時間。要查看您帳戶中所有視頻的流開始，您只需運行以下請求： 
    https://analytics.api.brightcove.com/v1/data?accounts={account_ids}&dimensions=video&limit=all&offset=0&fields=all
我如何退回一個報告？
設置limit參數為報告等於all .
什麼數據fields默認情況下為項目返回？
默認情況下，僅video_view以及與所請求的維度相對應的字段（例如destination_domain）返回。要獲取其他字段，請設置fields=field_name1,field_name2返回選定的字段。
如果一個字段值是什麼意思null ?
一種null數據字段的值表示所請求的數據尚未處理。最可能的原因是：
  • 您請求的數據是最新的，尚未處理
  • 您請求的數據很舊，尚未導入當前的分析系統
我更改了其中一個視頻的名稱-為什麼我在Google Analytics（分析）數據中看到舊名稱？
任何新的分析事件都會記錄新名稱，但我們不會更改歷史數據中的名稱 - 返回的視頻名稱將是觀看時的名稱。
為什麼我不總是看到參與值的整數？
訂婚號可能會返回小數點。原因是因為參與度是標準化的，這意味著它是“video_percent_viewed * (video_engagement_25 / video_engagement_sum)”的比率，因此根據為查詢選擇的時間範圍，您可能會看到浮點數，如果它們沒有精確劃分。
如何按玩家過濾結果？

使用時請注意以下幾點where播放器上的過濾器：

  • Brightcove玩家 -輸入以下格式的播放器網址：players.brightcove.com/account id/playerID_default
  • 智能玩家 - 輸入玩家 ID

正在匯報

為什麼在查看播放器使用的帶寬時會看到 0 GB？
過去，Brightcove曾按播放器報告帶寬，因此對於想要按播放器細分事物的客戶，他們可以使用這些指標來分配帶寬成本。但是，隨著我們朝著清單驅動的交付方式發展（今天是HLS，將來是DASH），分段視頻的性質將使播放器的跟踪帶寬不准確。因此，我們將在利用率報告中按玩家維度棄用帶寬。因此，我們建議在性能報告中使用玩家查看的秒數來按玩家分配成本。

各種各樣的

我可以從分析報告中排除員工的意見嗎？
沒有簡單的方法可以做到這一點，對於高流量視頻，員工觀看次數可能很小，以至於它們對分析的影響可以忽略不計。但是，對於您認為這樣做很重要的低流量視頻，最簡單的解決方案是複制您的製作播放器，並讓員工測試/觀看副本上的視頻。然後，您可以使用過濾器僅使用 Analytics 模塊中的自定義報告功能或 Analytics API 創建關於您的生產播放器的報告。
