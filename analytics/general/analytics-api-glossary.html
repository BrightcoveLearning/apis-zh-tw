筆記：有關Analytics（分析）模塊用戶界面中顯示的值的說明，請參見Google Analytics（分析）模塊用戶界面詞彙表。

一種

account
視頻雲帳戶 ID。
active_media
帳戶中當前有效（可播放）的視頻數。請注意，有關的數據active_media僅在數據為reconciled。
ad_mode_begin
Brightcove 播放器或 SDK 切換到廣告模式以開始播放廣告內容的次數。此事件將發生在video_view在預卷場景中或之後的事件video_view中或後滾動配置中的事件。該指標並不直接對應於實際投放的廣告數量，因為顯示的廣告數量可能因廣告服務器配置而異。
ad_mode_complete
Brightcove Player 或 SDK 從播放廣告內容返回的次數。

乙

bytes_delivered
傳輸的字節數，包括視頻數據和元數據以及播放器代碼。請注意，有關的數據bytes_delivered如果數據是reconciled。

C

city
觀看視頻的城市。
country
觀看視頻的兩個字母的國家/地區代碼。遵循ISO 3166標準。
觀看視頻的國家/地區的英文簡稱。遵循ISO 3166標準。

d

daily_unique_viewers
這是帳戶在特定日期擁有的唯一身份訪問者的估計數量。此數據僅適用於 1-2 天前觀看的視頻 並且僅在帳戶級別。您可以通過以下方式提高此指標的準確性這些方向
date
觀看視頻的當月日期。這個維度的值是ISO 8601格式為 YYYY-MM-DD 的日期
date_hour
這date_hour維度類似於date維度，但可用於訪問過去 32 天內視頻的每小時數據。看date_hour 維度了解更多詳情。
destination_domain
域所在的域video_impression發生（對於網絡視圖）。這在 UI 中稱為“顯示域”。
destination_path
發生視頻觀看展示的頁面的路徑（對於網絡觀看）。這在 UI 中稱為“顯示 URL”。
device_os
設備操作系統。可能的值為：
  • android
  • bada
  • ios
  • linux
  • mac_os
  • os_x
  • rim（黑莓）
  • symbian
  • web_os
  • windows
  • other（對於未映射到上述任何一個的用戶代理）
device_type
用於觀看視頻的設備類型。可能的值為：
  • desktop -包括台式機和筆記本電腦
  • tablet -包括iPad，Android平板電腦等。
  • mobile -手機
  • tv -包括智能電視，機頂盒和遊戲機
  • other -對於未映射到上述任何內容的用戶代理
dimensions
維度是您可以為報告指定的分析數據類別。當前可用的尺寸為：

看維度和字段並使用交互式工具查看各個維度返回哪些字段，然後選擇多維查看您可以在報告中組合哪些維度。

DMA
DMA（指定市場區域）地區是美國的地理區域，尼爾森公司（Nielsen Company）測量了本地電視收視情況。

E

engagement_score
每次視頻觀看的平均觀看百分比（通過添加所有觀看百分比除以視頻觀看次數計算得出）。有關如何計算的更多信息，請參見計算視頻參與度得分。

F

fields
用於指定應返回哪些度量標準字段的參數-請參閱Analytics API概述更多細節。

G

地理尺寸
方面可提供有關以下地理位置的信息：video_impression發生。跟踪的地理維度是：
  • city
  • country
  • region

一世

item_count
符合API請求條件的項目數。

大號

limit
要返回的數據項數 - 正整數或“全部”。

中號

指標
跟踪不同的數據項方面。另請參閱領域

Ø

offset
如果您正在使用限制只返回一部分數據項，offset指定開始的項目編號。

P

play_rate
播放率是視頻播放次數（video_view）與播放機會（video_impression）的比率。
play_request
用戶按下播放按鈕或自動播放事件請求播放的次數。這通常發生在video_impression和video_view事件。但是，play_request如果用戶暫停並按播放以繼續觀看視頻，也會發送。
player
視頻雲播放器 ID。
player_load
播放器加載的次數。
player_name
視頻雲播放器的名稱。

[R

分析數據來自多個來源：一些數據是由播放器發送的，而其他數據則是從CDN和視頻雲系統收集的。為了盡快提供分析，我們開始盡快提供“實時"數據，然後在收集並處理了所有來源的數據後再調整分析。完全處理的數據稱為reconciled
客戶端（瀏覽器或其他）報告的引薦域。
referrer_domain
客戶端（瀏覽器或其他）報告的引薦域。
region
觀看視頻的兩個字母的省或州代碼。遵循ISO 3166標準。
region_name
觀看視頻的省或州名稱。遵循ISO 3166標準。
報告
一個或多個的詳細分析數據方面。

秒

search_terms
如果觀看者通過搜索找到了視頻，則這些是使用的搜索詞。
sort
用於指定用於對分析數據進行排序的指標的參數-請參閱Analytics API概述更多細節。
source_type
指示觀看者如何到達播放器頁面；可能的值為：
  • referral（來自其他頁面的直接鏈接）
  • direct（書籤，鍵入URL或來自電子郵件的鏈接）
  • organic_search（使用某些搜索引擎的常規搜索）
  • paid_search（點擊搜索結果中的讚助商鏈接）
  • secure_search（通過已登錄的查看器進行搜索）

伏

視頻
視頻的視頻雲 ID。
video_duration
視頻長度（以秒為單位）。
video_engagement
衡量每個視頻觀看的平均觀看百分比（通過將所有觀看百分比與視頻觀看次數相加得出）。有關如何計算的更多信息，請參見計算視頻參與度得分。
video_engagement_1
在視頻的 1% 點記錄的觀看次數。
video_engagement_25
在視頻的 25% 點記錄的觀看次數。
video_engagement_50
在視頻的 50% 點記錄的觀看次數。
video_engagement_75
在視頻的 75% 點記錄的觀看次數。
video_engagement_100
在視頻的 100% 點記錄的觀看次數。
視頻印象
視頻加載到播放器並準備好進行交互的次數。
視頻名稱
視頻名稱（當時video_impression發生）。
video_percent_viewed
這是每個觀眾觀看的百分比的總和。例如，如果觀看者A觀看了0.50，而觀看者B觀看了0.75，那麼總值將為1.25。該指標本身並無用處。該指標與 video_view 一起用於計算參與度分數。參與度得分是video_percent_viewed除以video_view。video_percent_viewed 可用於計算多個視頻的參與度分數。
video_seconds_viewed
觀看視頻的總秒數。
視頻視圖
視頻開始播放的次數，在流開始時記錄（不包括倒帶或重播）。在 Studio 中觀看視頻時不計算視頻觀看次數。

寬

where
用於過濾有關特定尺寸值的報告的參數-請參閱Analytics API概述有關詳細信息。
