介紹

這unique_users字段，該字段為客戶提供帳戶級別每天的唯一身份訪問者的估計數量。數據在一個名為的字段中返回daily_unique_viewers .此字段在帳戶或視頻級別可用。看概述：資料收集API有關如何收集和使用用戶數據的更多信息。

請注意，唯一用戶數據僅針對和解的數據。

跨越多天是平均的

如果您的請求跨越了幾天，例如：

    https://analytics.api.brightcove.com/v1/data?accounts=789967572001&dimensions=account&from=2014-07-09&to=2014-07-22&limit=10&format=json&fields=daily_unique_viewers,video_view

響應將包含該時間段內 daily_unique_viewers 的平均值。

    {
      "item_count": 1,
      "items": [
        {
          "account": "789967572001",
          "daily_unique_viewers": 99,
          "video_view": 2334
        }
      ],
      "summary": {
        "video_view": 2334
      },
      "account": "789967572001"
    }

數據可用性

唯一性估計發生在批處理系統中，這意味著此 daily_unique_viewers 值不是實時可用的。Daily_unique_viewers 的值將在被測量的日期過後的 3-5 天內不可用。如果調用API並獲得NULL響應，則意味著該數據尚不可用。

JSON 返回 NULL，CSV 返回“”。CSV 沒有顯示空值的方法。

識別用戶

Video Cloud Analytics 使用由以下內容組成的字符串Source IP address + the User-Agent String作為用戶的唯一標識符。

唯一身份用戶總數

由於為所有用戶存儲唯一標識符是不可行的，因此我們使用一種算法來估算一個帳戶的總用戶數。

這意味著如果您知道確切的觀眾人數並將其與我們的人數進行比較，您會看到一個小的差異。該錯誤率隨著唯一觀看者數量的增加而增加。
