概述

筆記：獲取客戶端憑據的首選方法是使用Studio 管理界面 .本文檔適用於選擇使用 OAuth API 的用戶。

要了解本文檔中的信息，最好的第一步是查看OAuth 概述如果你還沒有這樣做。

您的BC_TOKEN是 Brightcove 用戶的唯一匿名標識符。除了在後端Brightcove系統中存在之外，您在登錄Studio時還將其設置為cookie，以便Studio知道您的角色以及您有權訪問的帳戶。這一切都是透明的，不需要您採取任何行動。

使用BC_TOKEN因為您正在驗證 OAuth API 請求以創建客戶端憑據。

筆記：
  • 這BC_TOKEN value 與您在 Studio 中處理的 API 管理令牌無關。
  • 如果你使用Studio管理員創建客戶端憑據，則無需檢索您的BC_TOKEN -Studio會幫您解決這個問題。
  • 這BC_TOKEN cookie 是會話 cookie，每次退出 Studio 時都會過期（也會在 24 小時後自動過期）。每次一個新的BC_TOKEN當您登錄 Studio 時，cookie 將被刪除，它將具有一個新值。

要求

  • 具有創建客戶端憑據所需的管理員權限的有效Brightcove帳戶
