概述

筆記：獲取客戶端憑據的首選方法是使用 Studio 管理 UI。本文檔適用於那些選擇使用 curl 的人。

要了解本文檔中的信息，最好的第一步是查看OAuth 概述如果你還沒有這樣做。

筆記：文檔中的步驟涉及curl和其他 Mac 和 Linux 系統原生的命令行工具，但不是 Windows 操作系統的一部分。如果您使用的是Windows，則可以通過安裝以下工具來獲得這些工具西格溫。在安裝過程中，將提示您安裝軟件包。打開網包，然後檢查捲曲選項，然後完成安裝。完成此操作後，您將可以按照分步操作進行操作：使用cygwin命令行成功編寫了播放器管理文檔。可以使用工具在普通的 Windows 命令行中運行 curl，但在本文檔中沒有詳細說明。

還有一個用於獲取的用戶界面client_id和client_secret：

  • 管理API身份驗證憑據

要求

  • 有效的Video Cloud帳戶
  • 有效的Perform帳戶
  • 跑步能力curl從命令行（請參閱注意curl以上）

概述

得到client_id和client_secret只是對OAuth服務的POST調用，但是您需要輸入帳戶ID和BC_TOKEN，當您登錄Studio時可以在臨時Cookie中找到。

筆記：這BC_TOKEN value 是打開 Studio 時設置的 cookie。它與您在Studio中處理的API管理令牌無關。

得到client_credentials

現在，我們準備調用OAuth服務以檢索客戶端憑據。我們必須指定我們為其請求憑據的客戶端應用程序名稱 - 該名稱是任意的，旨在幫助您跟踪憑據的用途 - 這裡我們將僅使用“AnalyticsClient”。我們還必須指定我們想要訪問的操作範圍，這裡我們將使用視頻雲/播放器/所有表演/球員/所有（用於播放器管理 API）。看客戶端憑據請求的 API 操作獲取當前支持的所有操作的列表。

  1. 編輯以下curl命令，然後將其粘貼到命令行中並按返回。您必須為以下三個值提供特定值：
    • 你的 BC_TOKEN
    • 您的憑證名稱
    • 你的賬號
            curl \
          --include \
          --header "Authorization: BC_TOKEN YOUR_BC_TOKEN" \
          --data {'name=YOUR_CREDENTIAL_NAME_GOES_HERE&maximum_scope=[{
              "identity": {
                "type": "video-cloud-account","type": "perform-account",
                "account-id": YOUR_ACCOUNT_ID
              },
              "operations": [
                "LIST_OF_OPERATIONS"[1]
              ]
          }]'} \
        https://oauth.brightcove.com/v4/client_credentials

    [ 1] 看API 操作對於可用的操作。
