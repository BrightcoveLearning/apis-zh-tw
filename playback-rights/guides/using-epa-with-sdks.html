此功能適用於有權訪問播放權限管理服務的有限可用性階段的一組特定客戶。

介紹

默認情況下，本機SDK與Brightcove Playback API對話，以檢索您的視頻和播放列表內容。一個管理播放權限和限制的新系統位於Playback API的前面，並使用DRM許可證提供播放授權。

如果您不熟悉此功能，請參閱概述：播放權限管理服務文檔。

使用“播放權限"進行請求：

  • Android實施
  • iOS實施

Android實施

注意：使用此實現，將忽略使用策略鍵配置的任何內容限制。

要發出檢查播放權限和限制的播放 API 請求，請執行以下步驟：

  1. 從基本示例應用程序 .

    筆記：
  2. 此實現需要適用於 Android 版本的 Brightcove Native SDK 6.11.0或更大。
  3. 在Android平台上，當前僅支持DRM DASH。

  4. 默認情況下，適用於Android的Native SDK如果具有策略密鑰，則會向Playback API發出請求。要使用回放權限，請不要隨請求一起發送策略密鑰。

    創建一個實例com.brightcove.player.edge.Catalog，而不提供策略密鑰。這是一個例子： 

    Catalog catalog = new Catalog.Builder(eventEmitter, accountId).build();

    如果您關注的是Brightcove示例應用程序，它將獲得帳戶ID，如下所示： 

    getString(R.string.account)
    </br/>

  5. 對於用戶級別的限制，您需要創建一個JWT令牌並將其與目錄請求一起傳遞。

    要創建您的 JWT 令牌，請按照以下步驟操作定義用戶級限制實施回放權限管理服務文檔的部分。

    要通過目錄請求傳遞您的令牌，請按照安卓實現將播放授權 DRM 與本機 SDK 文檔一起使用的部分。

iOS實施

注意：使用此實現，將忽略使用策略鍵配置的任何內容限制。

要發出檢查播放權限和限制的播放 API 請求，請執行以下步驟：

  1. 從基本示例應用程序 .

    此實現需要適用於 iOS 版本的 Brightcove Native SDK 6.7.0或更大。

  2. 默認情況下，如果iOS的本地SDK具有策略密鑰，則會向Playback API發出請求。要使用回放權限，請不要隨請求一起發送策略密鑰。

    創建一個實例BCOVPlaybackService，將策略密鑰設置為nil。這是一個例子： 

    let playbackService = BCOVPlaybackService(accountId: kViewControllerAccountID, policyKey: nil)

  3. 對於用戶級別的限制，您需要創建一個JWT令牌並將其與目錄請求一起傳遞。

    要創建您的 JWT 令牌，請按照以下步驟操作定義用戶級限制實施回放權限管理服務文檔的部分。

    要通過目錄請求傳遞您的令牌，請按照iOS 實現將播放授權 DRM 與本機 SDK 文檔一起使用的部分。
