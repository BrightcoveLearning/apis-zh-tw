此功能適用於有權訪問播放權限管理服務的有限可用性階段的一組特定客戶。

介紹

Brightcove的播放權限管理服務使您可以設置視頻播放的設備限制。提出DRM許可請求時，每個用戶的設備都用唯一的ID註冊。您可以控制每個用戶的設備數量。

每個許可證請求都會檢查並強制執行設備限制。要使用此限制，您將通過JSON 網絡令牌 (JWT)使用當前用戶 ID uid到播放 API。

如果您不熟悉此功能，請參閱概述：播放權限管理服務文檔。

設備註冊

當有效的播放請求包括JSON Web令牌（JWT）和uid和dlimit索賠已收到。從那時起，該設備將佔據一個插槽。

執行

要啟用設備限制，請按照下列步驟操作：

  1. 創建一個JSON Web令牌（JWT）與一般要求的索賠以及uid和dlimit索賠。

    提出DRM許可請求時，每個用戶的設備都用唯一的ID註冊。

    這dlimit聲明可讓您控制可以與指定用戶關聯的設備數量uid .該值必須是 > 0 .

    如果dlimit聲明值在以後的請求中被刪除，先前允許的設備將繼續運行。

    例子
    如果dlimit值設置為3，用戶可以在設備 A、B 和 C 上玩遊戲（全部都被允許）。嘗試在設備 D 上播放將被拒絕。
    如果dlimit值更改為1，用戶仍然可以在所有 3 台設備 A、B 和 C 上玩遊戲，除非通過使用設備 API 管理設備手動撤銷這些設備。為此，請參閱管理設備部分。

  2. 管理用戶設備使用Devices API。此 API 允許您獲取用戶的設備、刪除設備和更新設備的名稱。

    設備註冊時，由 id 和暱稱標識。設備 API 可讓您編輯設備暱稱，從而輕鬆管理您的設備列表。

  3. 配置您的播放器使用JSON 網絡令牌 (JWT) 在上一步中創建。

管理設備

您可以使用設備 API .

設備API

使用 Devices API，您可以獲取和刪除用戶設備。

基本網址

API 的基本 URL 是： 

https://playback-auth.api.brightcove.com/v1

帳戶路徑

在任何情況下，都將請求特定的視頻雲帳戶。因此，您將始終將術語帳戶後跟您的帳戶 ID 添加到基本 URL： 

https://playback-auth.api.brightcove.com/v1/accounts/{accountID}

授權

請求的訪問令牌是必需的，並且必須存在於Authorization標頭中： 

Authorization: Bearer {access_token}

訪問令牌是必須從 Brightcove OAuth 服務獲取的臨時 OAuth2 訪問令牌。有關如何獲取客戶端憑據並使用它們來檢索訪問令牌的詳細信息，請參閱Brightcove OAuth 概述 .

權限

對設備API的請求必須來自客戶憑證具有以下權限：

  • video-cloud/playback-auth/devices/read
  • video-cloud/playback-auth/devices/write

API方法

設備 API 支持以下請求。有關詳細信息，請參見設備API參考。

獲取所有用戶設備

GET /v1/accounts/{accountID}/users/{userID}/devices
反應體

響應正文包含一組設備，應類似於以下內容： 

[
    {
        "id": "12e12345-11e1-1bd1-d123-1234567890",
        "created_at": 1612646018,
        "updated_at": 1612646018,
        "name": "my device1",
        "user_agent": "PlayReadyClient"
    },
    {
        "id": "WVCf123456789abcdefghijkl1234567890abcdefg",
        "created_at": 1612385697,
        "updated_at": 1612385697,
        "name": "my device2",
        "user_agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:85.0) Gecko/20100101 Firefox/85.0"
    }
]

更新設備名稱

這PATCH request 允許您為設備設置描述性名稱。 

PATCH  /v1/accounts/{accountID}/users/{userID}/devices/{deviceID}
  Content-Type: application/json
  Body: {devices object}
要求正文

請求正文包含設備名稱。 

{
    "name": "my device1"
}

以下是字段詳細信息：

場地 類型 描述
name 細繩 描述性暱稱，以幫助識別設備

刪除所有用戶設備

DELETE /v1/accounts/{accountID}/users/{userID}/devices

刪除特定的用戶設備

DELETE /v1/accounts/{accountID}/users/{userID}/devices/{deviceID}
