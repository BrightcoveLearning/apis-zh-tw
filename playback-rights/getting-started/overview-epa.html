此功能適用於有權訪問播放權限管理服務的有限可用性階段的一組特定客戶。

介紹

Brightcove的播放權限管理服務提供了一種可擴展的表達方式來管理視頻播放。此服務包括以下功能：

  • 按地理位置限製播放
    • 國家
    • 郵政編碼
    • 指定市場區域（DMA）
  • 定義代理限制
  • 定義每個視頻的IP限制
  • 定義域限制
  • 設置重複時間表
  • 限制設備觀看
  • 限制觀看者一次可以觀看的視頻數量

此圖顯示了從用戶的視頻和許可請求到播放的流程。

規格

Brightcove的播放權限管理服務提供以下功能：

播放權限

播放權限允許您指定視頻播放的限制。這些權利將存儲在播放權限 API並且每個集合都被分配了一個唯一的 id ( playback_right_id）。一組權利可以與零個或多個視頻相關聯。

當一個視頻有playback_right_id，播放權限對像中指定的限制適用於對該視頻的所有請求。

播放權包括以下類別：

地理

地理限制使您可以控制在何處查看內容。您可以根據以下內容指定限制：

  • 國家
  • 郵政編碼（僅限美國）
  • 指定的市場區域（DMA-Nielson開發的概念）

地理檢查在視頻播放開始時以及播放期間運行。如果中途地理檢查失敗，則播放器將停止視頻播放。

代理限制

代理限制使您可以阻止播放某些類型的連接。

IP地址和域名

域限制允許您將視頻的觀看位置限制為特定的域。IP限制將視頻播放限制為特定的IP地址（帳戶或視頻級別）。

時刻表

通過重複計劃，您可以限制特定時間範圍內的視頻播放。例如，您可以只在周末播放內容。另一個示例是允許在星期一，星期二和星期五的上午8:00和晚上8:00之間播放。

用戶級限制

您可以為每個查看者的每個帳戶創建一個唯一的 ID。對於用戶級別的限制，您將需要在每個請求中使用JWT令牌。有了這個，您可以設置以下限制：

裝置限制

提出DRM許可證請求時，將進行設備註冊。唯一的ID被分配給每個設備。對於每個許可證請求，都會檢查並強制執行設備限制。要使用此限制，您需要使用當前用戶 ID (uid) 生成 JWT。

並發流限制

限制每個用戶的並發流會阻止查看者與沒有帳戶的朋友共享其憑據。通過並發流限制，您可以定義特定用戶在任何給定時間可以觀看的視頻流數量。

標籤

限制觀看者可以根據訂閱權利觀看的內容。您生成的 JWT 令牌將指定允許查看的標籤。播放權限服務確保用戶只能觀看帶有指定標籤的視頻。

假設您具有以下訂閱級別：

  • 銀級 - 觀看除首映電影和直播之外的任何視頻
  • 黃金-觀看直播以外的任何視頻
  • 白金級-觀看任何視頻或實時流

首先，將代碼與您的視頻相關聯，以區分標準，首播和實時。然後，在您為用戶生成的 JWT 令牌中指定適用的標籤。

視頻限制和播放權限

可以通過以下方式對視頻設置以下限制媒體模塊或者內容管理系統 API：

  • 地理過濾（允許/禁止在某些國家/地區查看）
  • 日程安排（設置視頻可用性的開始和/或結束日期時間）

這些設置獨立於播放權限管理系統，但視頻必須通過視頻級別限制才能應用播放權限管理規則。
