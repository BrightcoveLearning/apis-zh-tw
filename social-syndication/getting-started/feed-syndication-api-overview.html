Brightcove Syndication Feed API 是一個公共 API，用於檢索使用聯合配置 API .

Syndication Configuration API 也可用於通過簡單地打開syndication_url在聯合對像中返回到瀏覽器中，但 Feed API 的不同之處在於它不需要身份驗證 - 任何擁有正確 URL 的人都可以檢索提要。

使用一種 API 或另一種 API 檢索提要的主要用例如下：

  • 如果您只想自己託管供稿，例如對於訂閱者，則可以使用Syndication Configuration API來檢索供稿。
  • 如果您有聯合合作夥伴，希望他們以任何希望的方式來使用和託管供稿，則可以通過以下方式提供供稿API URL：syndication_id直接檢索提要。

API參考/基本URL

這供稿 API 參考包含有關可用請求和響應的完整詳細信息。

基本網址是：

https://social.feeds.brightcove.com/v1

驗證

可以訪問Feed API 不是需要進行身份驗證，因為提要旨在公開訪問。

運作方式

Feed API 只有一種操作：根據syndication_id。

要求網址

    https://social.feeds.brightcove.com/v1/accounts/{account_id}/mrss/syndications/{syndication_id}/feed

方法：得到

查詢參數

所有查詢參數都是可選的。

limit
供稿響應中包含的最大項目數。允許範圍為1到100。默認：100。
offset
生成返回的提要時要跳過的項目數。默認：0。
updated_after
ISO-8601日期時間字符串（例如，2016-03-16T15：42：06.000Z或2016-03-16T10:47:29-05:00），將供稿結果限制為給定日期之後更新的資產。

請求正文：無

這個很重要！如果包含請求正文，則會返回錯誤。

回應類型：XML
