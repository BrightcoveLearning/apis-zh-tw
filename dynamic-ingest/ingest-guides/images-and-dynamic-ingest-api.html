介紹

您可以攝取兩類圖像：

  • 主要/默認海報和縮略圖圖像，它們在請求正文中定義為海報和縮略圖對象（字段和語法詳細信息如下）
  • 多語言變體圖像，定義在圖片對像數組（字段和語法詳細信息如下）

多語言變體圖像可以有六種類型：

海報
在用戶播放視頻之前，海報圖像會顯示在 Brightcove 播放器中。它通常是從視頻捕獲的幀，但不是必須的。
縮略圖
縮略圖顯示在播放列表、片尾畫面等中。它通常是從視頻捕獲的幀，但不是必須的。
肖像
通常在電話和其他設備上以人像模式用作海報的圖像。需要布萊頓灣燈塔。規範的詳細信息在桌子以下。
正方形
通常用作頻道徽標或網格顯示的圖像。需要布萊頓灣燈塔。規範的詳細信息在桌子以下。
寬的
通常在手機和其他設備上以橫向模式用作海報的圖像。需要布萊頓灣燈塔。規範的詳細信息在桌子以下。
超寬
通常在電視等超寬顯示器上用作海報的圖像。需要布萊頓灣燈塔。規範的詳細信息在桌子以下。

有兩種不同的方法可以使用 Dynamic Ingest API 將主要海報和縮略圖添加到您的視頻中：

  • 拍攝影像在轉碼過程中位於視頻的中點（當前適用於poster和thumbnail圖片。必須攝取其他類型）。主要海報和縮略圖的捕獲圖像可以與多語言變體的攝取圖像一起使用。
  • 在攝取視頻的同時或分別攝取您自己的圖像。請注意，如果您在攝取視頻後攝取了圖像，您必須等待視頻處理完成才能攝取圖像。

設置

動態攝取請求的設置是相同的，無論您是攝取視頻、圖像、WebVTT 文件還是所有這些：

請注意，所有文件名都應該是 URI 編碼的。
要求網址
    https://ingest.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/ingest-requests
驗證
身份驗證需要以令牌形式傳遞的訪問令牌Bearer令牌中Authorization標頭： 
    Authorization: Bearer {access_token}

關於S3的注意事項

如果您的圖像是從受保護的S3存儲桶中提取的，則需要設置存儲桶策略以允許Video Cloud訪問文件。看在 S3 中使用動態攝取詳情。

多語言變體

您可以提供用於不同語言的不同版本的圖像。這是由處理明灣燈塔適用於 Brightcove Beacon 應用程序。Brightcove Player中的常規實現當前需要自定義開發。

圖像的語言在language屬性在下一節中詳述。

自動調整大小

當您通過 Studio 或 Dynamic Ingest API 添加自定義海報和/或縮略圖圖像時，默認情況下 Video Cloud 將調整這些圖像的大小以匹配帳戶的默認攝取配置文件中定義的圖像大小。

請注意，調整大小和填充是Video Cloud或布萊頓灣燈塔。圖像永遠不會被裁剪。

拍攝影像

在轉碼過程中，Video Cloud可以從視頻的中點捕獲快照並將其另存為海報和縮略圖圖像。

動態攝取創建的海報和縮略圖的數量各為一個。即使您的個人資料中有多個圖像格式，您也不能創建多個海報和縮略圖格式。

對於自定義攝取配置文件，圖像捕獲要求向您使用的攝取配置文件添加兩個特殊格式。看攝取配置文件中的圖像以下。

您還應該設置capture-images在您的請求數據中標記true如果您希望在轉碼期間從視頻中捕獲海報和縮略圖，但這是默認值 如果 所選配置文件包括圖像再現。（如果配置文件中沒有圖像副本，則默認為capture-images是false）

示例請求數據：

      {
        "master": { "url": "http://learning-services-media.brightcove.com/videos/mp4/Bird_Titmouse.mp4" },
        "profile": "multi-platform-standard-static",
        "capture-images": true
      }
請注意，您必須包括profile字段並指定帶有圖像再現的配置文件（所有標準配置文件都有它們）。

提取配置文件中的圖像

為了poster和thumbnail僅圖像，您必須在攝取配置文件中包含圖像副本。這是用於自動捕獲，無論您是在攝取視頻時使用它還是決定稍後在轉碼時使用它。圖像格式轉換包含在所有標準攝取配置文件中，因此本節僅適用於自定義攝取配置文件。

以下是有關如何在提取配置文件中指定圖像再現的詳細信息。

財產 價值
media_type "image"
label “海報"或“縮略圖"（每個都需要翻譯）
format “png"或“jpg"
width 數量（像素）
height 數量（像素）
