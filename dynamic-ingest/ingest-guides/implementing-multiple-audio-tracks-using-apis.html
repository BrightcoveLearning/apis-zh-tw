介紹

將多個音軌與單個標題相關聯的能力在以下幾個關鍵場景中非常有用：

  • 播放不同語言的同一視頻，以在全球範圍內擴大覆蓋範圍 
  • 為視力障礙者提供音頻和描述

請注意，添加音頻軌道也可以在Studio中完成-請參見使用媒體模塊將音軌添加到視頻更多細節。

支持的視頻格式

支持多個音軌HLS V4+和DASH -有無DRM保護。

筆記：
  • 多個音軌只能與為以下內容設置的視頻一起使用動態投放。
  • 必須為多個音軌啟用該帳戶。要啟用您的帳戶，請聯繫您的客戶經理。

工作流程

有兩個主要用例：

  • 攝取具有多個音軌的新視頻
  • 將多個音軌添加到現有視頻

我們將在下面的部分中查看 API 請求的詳細信息。

音軌元數據

有一些元數據字段附加到音軌 - 其中一些是在您攝取曲目時設置的，而另一些則是由視頻雲 .其中一些字段是只讀的，而其他字段可以由您更新。下面將在相關 API 請求的上下文中詳細介紹元數據字段，但此處需要解釋兩個特定字段，因為它們對於確定 Brightcove 播放器如何處理多個音軌至關重要。

language

這language在攝取期間為每個軌道設置的字段指定軌道的語言。如果曲目是視頻中的口語配音，這一點很重要。此字段的值將是一個兩個字母的代碼，如en或者hi .有關語言代碼的完整列表，請參見Subtag值http://www.iana.org/assignments/language-subtag-registry/language-subtag-registry。

variant

這variant字段描述音軌的種類。可能的值，具有標準含義是：

  • main -主音軌，通常是混音到視頻文件中的那一條
  • alternate -備用音軌
  • commentary -音頻軌道，可提供有關視頻軌道的評論
  • dub -包含另一種語言的配音版本的曲目
  • descriptive -軌道以某種方式描述了視頻內容

帳戶默認值

您可以設置帳戶默認值language和variant確定 Brightcove 播放器將哪些音軌視為默認音軌（也可以通過更新音軌元數據來覆蓋默認值，我們將在下面的部分中看到）。要為您的帳戶設置默認值，聯繫 Brightcove 支持 .

攝取音軌

現在，我們將研究用於提取前面描述的兩個用例的音軌的API調用。

具有多個音軌的新視頻

創建視頻對象（內容管理系統 API )

  1. 您可以在創建視頻對象時添加多項視頻元數據，但這裡我們只添加最少的：name對於視頻： 
          {
          "name": "YOUR_VIDEO_NAME"
      }
  2. 提交上面的 JSON（用您的視頻名稱替換佔位符文本）作為請求正文POST請求https://cms.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/videos
  3. 您將在響應中獲得大量視頻元數據，但這裡的重要部分是id（視頻 ID），這是下一步所需的。

攝取視頻和音頻軌道

接下來，我們將攝取視頻和音頻軌道（我們可以添加其他資源，例如圖像和文本軌道，但是在這裡我們將使其保持簡單）。你可能會覺得有點困惑的一件事是audio_tracks在請求正文的 JSON 中出現兩次：

  • 一個audio_tracks內的對象master對象包含視頻文件中包含的音頻軌道的元數據（如果有的話-也稱為音頻混合）-由於音頻軌道已包含在其中，因此該對象將僅包含元數據，而沒有音頻文件的URL。視頻文件。請記住，只要符合以下規則，就可以將編碼後的音軌壓縮為“立體聲"，“5.1"，“7.1"。音訊格式並且，如已知的問題，除了設置默認音頻外，沒有其他方法可以訂購音軌
  • 頂級audio_tracks描述您正在攝取的其他音軌的對象-這些將包括音頻文件的URL以及其他元數據
  1. 請求體中要發送的JSON數據如下： 
          {
        "master": {
            "url": "https://learning-services-media.brightcove.com/videos/Great_Blue_Heron.mp4",
            "audio_tracks": [
                {
                    "language": "en",
                    "variant": "main"
                }
            ]
        },
        "audio_tracks": {
            "merge_with_existing": true,
            "masters": [
                {
                    "url": "http://learning-services-media.brightcove.com/audio/celtic_lullaby.m4a",
                    "language": "en",
                    "variant": "alternate"
                },
                {
                    "url": "http://learning-services-media.brightcove.com/audio/audio1.m4a",
                    "language": "en",
                    "variant": "commentary"
                }
            ]
        },
          "profile": "BoltIngestProfile",
          "capture-images": true,
          "callbacks": [
              "http://solutions.brightcove.com/bcls/di-api/di-callbacks.php"
          ]
      }
  2. 發送上面的 JSON，用你自己的 URL 代替佔位符，並調整language和variant值，在一個POST請求https://ingest.api.brightcove.com/v1/accounts/ACCOUNT_ID/videos/ID/ingest-requests（這ID這是從創建視頻對象的請求返回的視頻 ID）

將音軌添加到現有視頻

要將其他音軌添加到現有視頻，步驟相同，只是您不需要向內容管理系統 API創建視頻，因為它已經存在。並在請求中動態攝取 API，您不需要包含視頻文件的 URL 或audio_tracks在下面master為混入音頻提供元數據。但是，您確實希望在現有視頻中包含混入音軌的元數據。同樣重要的是要記住，音頻軌道必須遵循音訊格式。因此，攝取請求的 JSON 將如下所示：

      {
        "audio_tracks": {
            "merge_with_existing": true,
            "masters": [
                {
                    "url": "http://learning-services-media.brightcove.com/audio/celtic_lullaby.m4a",
                    "language": "en",
                    "variant": "alternate"
                },
                {
                    "url": "http://learning-services-media.brightcove.com/audio/audio1.m4a",
                    "language": "en",
                    "variant": "commentary"
                }
            ]
        },
          "profile": "BoltIngestProfile",
          "capture-images": true,
          "callbacks": [
              "http://solutions.brightcove.com/bcls/di-api/di-callbacks.php"
          ]
      }

通知

如果您指定一個或多個打回來 URL（如上面攝取請求的示例 JSON 中所示），視頻雲將為您攝取的每個音軌發送通知。通知將如下所示：

      {
        "entity": "default/audio128",
        "entityType": "DYNAMIC_RENDITION",
        "version": "1",
        "action": "CREATE",
        "jobId": "0f703adb-0f17-4a35-8395-21c7fcdd2649",
        "videoId": "5298468208001",
        "dynamicRenditionId": "default/audio128",
        "accountId": "1910141565001",
        "status": "SUCCESS",
        "language" : "en",
        "variant" : "alternate"
      }

要識別音軌通知，請查找language和variant通知中的字段。這"action": "CREATE"和"status": "SUCCESS"字段表示軌道已成功攝取。

管理音軌

攝取音軌後，您可以通過內容管理系統 API .

獲取視頻的所有音軌元數據

獲取一個音軌的元數據

要檢索與視頻關聯的一個音軌的元數據，請創建一個GET請求：

      https://cms.api.brightcove.com/v1/accounts/account_id/videos/video id/audio_tracks/audio_track_id

響應將是一個包含每個音軌元數據的對象。有關詳細信息，請參閱下面的響應字段表。

更新音軌元數據

您可以通過創建一個音軌來更新任何可寫的元數據字段PATCH請求：

      https://cms.api.brightcove.com/v1/accounts/account_id/videos/video id/audio_tracks/audio_track_id

在請求正文中，包括要修改的字段-例如：

      {
          "language": "es",
          "is_default": true
      }

刪除音軌

要刪除音軌，請將 DELETE 請求發送至：

      https://cms.api.brightcove.com/v1/accounts/account_id/videos/video id/audio_tracks/audio_track_id

注意，成功響應代碼可能是202（接受），而不是204（無內容），因為刪除過程是異步的，並且可能不會立即完成。

已知的問題

音頻主機未存儲
  • 視頻雲將要不是存儲音頻母帶
  • 如果您重新轉換視頻原版中的視頻，則其他音軌將丟失，並且必須使用將音軌添加到現有視頻上述方法
必須使用純音頻文件
音軌必須是不帶視頻音軌的純音頻文件
HLSv3，HLS同一段中的音頻和視頻
  • 播放 API 不會返回HLSv3體現
  • 全部HLS清單中將包含解混合的視頻/音頻
流暢的流媒體
平滑流 URL 將不可用。
社會分佈
無法選擇將用於分配的音軌。將始終使用視頻源中包含的軌道（音頻中的多路復用）。
工作室
  • Studio 將顯示有關音軌的信息
  • 要使用 Studio 添加音軌，請參閱多個音軌
音頻軌道的順序超出一個“默認"軌道
  • 您可以選擇默認音軌內容管理系統 API每個標題通過設置is_default場到true
  • 還有一個賬號默認，可以通過支持設置
  • 這只會影響 HLS 清單中的“默認”軌道
  • 不能進行其他訂購
從單一來源攝取多個音軌
我們只支持每個源一個音軌 .每個音軌必須分別攝取。
僅包含音頻的視頻的DRM保護
添加視頻軌道後，將啟用DRM保護。
最終用戶友好標籤
我們不支持音軌的自定義標籤。如果需要，您將需要通過Player API在客戶端進行更改。
在某些情況下，切換軌道可能會導致Brightcove Player變得不穩定
  • 在下載所有音頻段之前切換軌道
  • 當使用 Silverlight 插件播放視頻時（IE 版本小於 10，或任何 IE 版本的 Window 版本小於 8） - 多個音軌不是 Silverlight 支持。
  • 如果音頻和視頻的持續時間不同，則較短的播放器用完時播放器可能會停止播放。
視頻 ”duration”
該視頻duration如果音軌具有不同的持續時間，由目錄/播放 API 報告可能不正確。
