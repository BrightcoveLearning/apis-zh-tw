概述

DRM（數字版權管理）通過根據許可策略對視頻數據進行加密和解鎖來保護您的視頻內容。當視頻內容加載到 Brightcove 播放器時，播放器會回調許可服務器並獲得播放視頻的權限。此交易對觀眾是透明的。

有關DRM和Video Cloud支持的一般概述，請參閱概述：視頻雲中的數字版權管理。

筆記：
  • 要使用 DRM 或 HLS 加密，您的帳戶必須啟用這些功能。請與您的客戶經理聯繫以獲取更多信息。
  • 為視頻啟用 DRM 後，MP4 源/格式會在 Studio、CMS API 和 Playback API 中自動隱藏（意味著它們存在，但不會作為源發送到播放器）。

  • 以下移動操作系統瀏覽器當前支持DRM：

    • iOS Safari（11.2+）
    • Android Chrome瀏覽器

    Android 應用程序支持 Widevine，iOS 應用程序支持 FairPlay。有關創建移動應用的詳細信息，請參見Brightcove本機播放器SDK索引頁。

  • 布萊特灣目前不支援僅限視訊或僅限音訊的 DRM 播放。

動態交付的DRM

如果您的帳戶已啟用動態交付並且您的客戶經理已為其啟用 DRM，所有新上傳的視頻都將受到保護。如果您希望對現有產品進行保護動態投放視頻，請聯繫Brightcove支持。

如果您在 Dynamic Delivery 帳戶中有使用遺留攝取系統（不是動態傳遞），在應用 DRM 之前，它們需要使用動態傳遞配置文件重新轉碼。已為動態交付攝取的現有視頻不需要重新轉碼，但如上所述，您確實需要聯繫支持並要求他們對這些視頻應用 DRM 保護。

如果您的帳戶啟用了 DRM 並且您想提供一些沒有 DRM 的標題，請設置drm_disabled標誌通過內容管理系統 API到true使用更新視頻請求 .
