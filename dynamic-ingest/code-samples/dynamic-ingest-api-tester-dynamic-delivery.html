獲取憑證

與 Brightcove 的大多數 RESTful API 一樣，動態攝取 API需要 OAuth 訪問令牌來授權請求。為了獲得訪問令牌，您首先需要獲得具有要請求的各種操作權限的客戶端憑據。此快速入門在 Brightcove 學習服務示例帳戶上發出 API 請求，因此您無需在此處獲取憑據，但您應該了解發生了什麼。CodePen 將 API 請求信息發送到類似於這個 .代理從私有位置訪問客戶端憑據，並使用它們生成臨時訪問令牌。 OAuth API然後進行API請求並返迴響應數據。

要在您自己的帳戶上嘗試請求，請獲取您的客戶憑據並使用我們的其中一個API測試工具

  1. 要獲取您的憑據，請按照以下說明操作管理 API 憑證 .這是獲取客戶端憑據的推薦方式。請記住，使用動態攝取需要製作一個內容管理系統 API POST 請求也是如此，所以除了攝取權限之外，您至少需要video/write權限：

    使用工作室管理工具獲取客戶端憑據。您還可以按照下面列出的指南之一中的說明使用 OAuth API 獲取憑據。

  • OAuth：使用CURL獲取客戶端憑據
  • OAuth：使用郵遞員獲取客戶憑證
