概述

這動態攝取 API call 與它的調用完全相同攝取新視頻 - 唯一的區別是您無需事先致電內容管理系統 API在Video Cloud系統中創建視頻對象並獲取id，請求體略有不同。在重新轉碼完成之前，所有視頻都可以使用現有的副本播放。

動態攝取不是有一種處理多個視頻的機制——這個示例只是執行一個循環來為每個視頻發出動態攝取請求。
提供此示例和相關代碼作為生產開發的指南。該代碼未經認證可用於生產用途，也未經官方支持。
