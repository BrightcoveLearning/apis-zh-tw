概述

提供此示例和相關代碼作為生產開發的指南。該代碼未經認證可用於生產用途，也未經官方支持。

當您將新視頻添加到您的帳戶時，您可能不希望在未來某個時間發布視頻以供查看。但是，一旦創建了可播放的再現，該視頻就有可能可播放。除了有人錯誤地發布視頻外，它可能還會顯示在智能播放列表中，因為它符合為列表選擇視頻的搜索條件。

防止意外過早發布的最佳方法是安排視頻在您希望它在您攝取時可用的時間。您只需包含一個schedule請求正文中的對象內容管理系統 API創建視頻的調用：

              {
                "name": "greatblueheron.mp4",
                "schedule": {
                  "starts_at": "2017-03-10",
                  "ends_at": "2018-12-31"
                }
              }

這starts_at和ends_at值可以是完整的 ISO 8601 日期時間戳，或者您可以只包含日期部分，如下所示。這兩個字段都是可選的：省略starts_at字段表示該視頻將立即可用；省略ends_at字段表示該視頻將無限期提供。
