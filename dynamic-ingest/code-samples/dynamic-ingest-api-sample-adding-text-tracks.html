概述

提供此示例和相關代碼作為生產開發的指南。該代碼未經認證可用於生產用途，也未經官方支持。

在此示例中，我們一次顯示了一個帳戶20的視頻（使用CMS API），並允許用戶將WebVTT字幕添加到一個或多個顯示的視頻中。可以輕鬆修改示例以添加縮略圖和/或海報圖像，或重新轉碼視頻。該示例的重點是超越基本的 API 請求，向您展示如何將它們組裝成一個真正的應用程序。

筆記：動態傳遞在清單中提供標題和字幕，在這種情況下，與目錄中發送的曲目相比，與清單中發送的曲目相比，我們更喜歡清單內的曲目。清單中的曲目類型只能是“SUBTITLES”而不是“CLOSED-CAPTIONS”，因為“CLOSED-CAPTIONS”是為 608/708 字幕保留的。Brightcove 播放器將字幕顯示為 SUBTITLES。

該應用程序還旨在供多個用戶在一個或多個帳戶上使用。它實際上並沒有執行此操作，但是在下面的說明中，您將看到掛鉤從某些後端系統中提取用戶信息的位置。（或者，您可以像大多數API示例一樣向用戶詢問每次使用的帳戶ID和客戶端憑據。）

動態攝取不是有一種處理多個視頻的機制 - 此示例只是迭代一組視頻，為每個視頻發出動態攝取請求。
