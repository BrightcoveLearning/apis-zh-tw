介紹

這動態攝取 API功能強大且靈活，但要最有效地使用它，您需要遵循一些最佳實踐，詳情如下。

最佳實務

以下是動態提取的規則和準則。

  • 你應該不是對同一視頻進行重複的作業處理，儘管在同一作業中攝取其他資產（如圖像和文本軌道）以及視頻是可以的。
    如果您為視頻攝取圖像、文本軌道或其他資產 前 視頻已完成處理，他們將 不是 被添加到視頻中。

    如果您使用自動化流程來創建提取作業，並有可能在同一視頻ID上創建多個作業，則需要檢查通知以確保在創建新視頻之前，同一視頻上沒有其他作業要處理。

  • 這profile攝取請求的字段是可選的 .避免硬編碼profile 在應用和集成中-而是將最常用的配置文件設置為默認該帳戶，並忽略該字段，或者獲取該帳戶的可用配置文件並強迫用戶選擇一個。
  • 每個帳戶每秒 20 個請求的限制，用於總請求數動態攝取和內容管理系統 API .
  • 每個賬戶有 100 個並發普通優先級 DI 作業的限制
  • 如果要提取大量文件，請考慮使用低優先級隊列這樣您就可以將100個以上的作業排入隊列。
  • 攝取內容通常限制為每個文件100GB（如果您需要攝取較大的文件，請與您的客戶經理聯繫）。
  • 如果要分批提取文件，請將並發普通優先級作業限制為100，並等待一項作業完成處理後再添加另一項-請參閱後備/重試策略想要查詢更多的信息。或者，您可以設置“優先級”：“低” - 使用低優先級，您不必擔心攝取隊列的大小，但處理可能需要更長的時間。
  • 驗證攝取配置文件更改以確保您在攝取完整批次之前獲得所需的結果。
