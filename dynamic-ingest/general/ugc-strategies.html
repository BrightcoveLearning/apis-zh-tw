概述

處理用戶生成內容（UGC；也稱為消費者生成媒體，或 CGM）的關鍵問題是如何將用戶的視頻放入您的 Video Cloud 帳戶。顯然，您不能授予訪問您網站的所有人訪問您的Video Cloud Studio帳戶的權限。您需要將用戶生成的視頻定向到您控制的服務器，然後使用動態攝取以受控方式將這些視頻攝取到您的 Video Cloud 帳戶。

主要步驟

  1. 計劃您的UGC活動。制定行動號召和政策。為您的用戶提供有關文件格式和視頻編碼問題的指導。考慮版權和商標問題。
  2. 設置中間服務器託管您的上傳表單並接收視頻。
  3. 創建一個上傳表單您的用戶可以在其中上傳他們的視頻和元數據。
  4. 查看視頻在它們發布之前。將所需的任何編輯過濾器應用於用戶生成的視頻。
  5. 將用戶生成的視頻從中間服務器攝取到視頻雲媒體庫。您可以使用動態攝取將您的視頻拉入視頻雲，或使用用於動態攝取的源文件上傳 API如果您不能使用 Dynamic Ingest 可以直接從中提取的存儲庫。
  6. 發布您在網頁上選擇的用戶生成的視頻。您可以將智能播放列表用於動態內容。

此圖顯示了推薦的工作流程：

計劃您的UGC活動

您需要根據您的業務目標來規劃您的 UGC 活動。計劃如何將您的UGC廣告系列傳達給您的受眾。您可以圍繞您現有的視頻內容和網絡資產以及您的附屬公司和業務合作夥伴的視頻內容和網絡資產使用促銷活動。許多UGC活動都有競賽元素，在這種情況下，您需要考慮獎項，評判和活動持續時間。

在您的UGC廣告系列中，您需要將政策傳達給用戶。需要考慮的一些因素：

  • 接受的文件格式
  • 源文件編碼建議
  • 最大文件大小和視頻長度
  • 法律要求，例如分發上傳視頻的所有權和權利，以及對用戶的限制' 使用版權和商標（您的和第三方的）。

你還需要計劃你想要的方式發布視頻 .

發布前審查視頻

在公開可用的UGC廣告系列中，某些用戶可能提交了無法發布的視頻內容-侵犯版權或商標，或淫穢，令人反感或毫無意義的視頻內容。計劃您要選擇或主持和發布用戶生成的視頻的方式。您可以採用以下幾種方法：

  • 查看中間服務器上所有提交的視頻，然後將可接受的視頻移動到單獨的目錄中，從該目錄可以將它們上傳到您的視頻雲媒體庫。
  • 將所有提交的視頻直接攝取到您的視頻雲媒體庫中，但是將其置於非活動狀態，直到對其進行審核和批准。

設置中間存儲庫

您不希望用戶生成的內容直接進入您的視頻雲媒體庫，除非您真的可以信任所有用戶。相反，您應該使用中間存儲庫：用戶將他們的視頻上傳到中間存儲庫，然後使用Dynamic Ingest將視頻從中間服務器移到您的Video Cloud Media Library中。

我們建議為您的中間存儲庫使用 S3 存儲桶，因為從 S3 設置動態攝取非常容易，無論存儲桶是公共的還是受保護的。

創建一個上傳表單

上傳表單為用戶提供了一種將視頻上傳到您的中間服務器的方法。上傳表單通常具有用戶可以為視頻提供名稱、描述和標籤的字段，但您可以為要由用戶修改的視頻的任何屬性添加一個字段。（如果需要，您還可以在攝取視頻時添加其他元數據。）

攝取視頻

準備就緒後，您可以使用動態攝取將 UGC 內容攝取到您的視頻雲帳戶。涉及兩個 API 請求：

  1. 一種建立影片向CMS API發送請求，該CMS API在視頻雲系統中創建視頻對象，並添加元數據（例如說明和標籤）。
  2. 這攝取請求，它會拉入視頻文件並對其進行轉碼。

見動態攝取 API 概述想要查詢更多的信息。

發佈內容

考慮您要如何呈現用戶生成的視頻。您將在哪些頁面上發布它們？您想要定制的播放器嗎？

您可以使用智能播放列表自動對用戶生成的視頻進行分組。您可以考慮以下兩種方法：

  • 製作智能播放列表觀看用戶提交的觀看次數最多的視頻。為此，請使用通用標籤標記所有用戶提交的視頻，並創建由以下人員定義的新智能播放列表"search": "tags:\" {YOUR_UGC_TAG}\"並使列表類型PLAYS_TOTAL
  • 定義基於主題的標籤，並讓您的用戶選擇最佳標籤。
