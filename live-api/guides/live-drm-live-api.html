介紹

可以通過以下方式將DRM添加到實時作業中：DRM創建作業時，請求主體中的對象。這DRM對像有兩個屬性：

  • modes設置為數組或以下一項或多項：“all"，“fairplay"，“playready"，“widevine"，“widevine：dash"或“widevine：hls"
  • token_expires_in設置為43200（默認值）和31622400（令牌過期前的秒數）之間的數字

環境mode to all 將導致所有可用的 DRM 類型應用於輸出。默認情況下，DRM應用於輸出，如下所示：

  • FairPlay：應用於HLS輸出
  • 隨時可用：應用於DASH輸出
  • Widevine：應用於 HLS 輸出

Widevine 可應用於 DASH  輸出而不是 HLS。要使 Widevine 與 DASH 成為默認值，需要帳戶級覆蓋。如果您希望將此替代應用於帳戶，請與支持部門聯繫。在下一節中，您將看到如何指定widevine:hls或者widevine:dash對於特定作業，無論您帳戶的默認設置如何。請注意，根據您的帳戶/職位的配置方式，API響應（如下圖所示）將包括widevine:hls或者widevine:dash輸出，但不能同時輸出。

請注意，您與 Microsoft 的服務協議中定義了 PlayReady 的並發限制。看這個微軟文檔有關管理並發的信息。如果超過影片的並行限制，觀眾將無法播放超過最大數目。

所有DRM模式

Widevine-HLS工作

創建 Widevine 打包的 DASH  輸出（沒有獲得帳戶覆蓋以使其成為默認值），指定widevine:dash 在裡面modes如下例所示。同樣，如果您的帳戶默認設置為將 Widevine 與 DASH 一起使用，您可以通過指定來獲取特定作業的 Widevine-HLS 輸出widevine:HLS在裡面modes .

樣品請求

  {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 300,
    "ad_insertion": false,
    "static": false,
    "drm":{"modes":["fairplay","widevine:dash"]},

        "label": "hls720p",
