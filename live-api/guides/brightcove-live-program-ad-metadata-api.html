介紹

廣告元數據 (ad_server_data) 架構支持沒有嵌套元素（僅限鍵值對）的平面 JSON 語法。通過API設置的廣告服務器數據用於更新後收到的所有廣告間隔，直到數據被覆蓋或重置為止。客戶可以在直播期間隨時更新和/或重置數據。

此元數據不會在 Steam 激活之間持續存在。在流激活之間需要保留的廣告元數據應在SSAI應用程序廣告配置中設置。通過API設置的數據可以配置為用作廣告服務器上的宏以及服務器端信標替換。

廣告程序元數據通過自定義播放列表標記傳遞到SSAI系統，並遵循SSAI系統中現有的動態廣告參數（DAP）優先級。在提示點上發送的廣告服務器數據優先於分配給作業的元數據。如果提示點數據與作業上的廣告元數據之間發生鍵衝突，則提示點值被認為具有較高的優先級，並在合併的集合中使用

提示點上提供的廣告元數據僅用於該提示點，而不會保留用於後續的提示點插入。只有通過程序廣告元數據API設置的廣告元數據將在以後收到的所有廣告間隔中保持不變，直到更新或重置為止。

終點

有關端點的完整詳細信息，請參見實時API參考。

添加/更新

要添加或更新直播節目廣告元數據，請發送POST請求：

https://api.bcovlive.io/v1/jobs/{job_id}/cuepointdata

添加/更新字段
場地 類型 描述 例子
ad_server_data 物件 包含鍵/值對的對象。 
{
  "ad_server_data": {
  	"one":"won",
  	"two":2,
  	"three":false
  }
}

刪除

要刪除直播節目廣告元數據，請發送DELETE請求：

https://api.bcovlive.io/v1/jobs/{job_id}/cuepointdata
