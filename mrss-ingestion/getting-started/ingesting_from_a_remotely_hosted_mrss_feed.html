介紹

您可以使用媒體 RSS (MRSS) 讓 Brightcove 從您發布的 MRSS 提要中從您自己的系統中提取您的內容。Video Cloud 從 MRSS 提要中提取元數據，然後創建遠程資產或下載（從託管它們的位置）並處理視頻和任何相關資產（縮略圖、隱藏式字幕文件等）。

如果您有興趣參與從遠程託管的 MRSS 源中提取的測試版，請聯繫您的客戶經理。

MRSS攝取概述

關於MRSS

媒體RSS（MRSS）是RSS的擴展，它使您可以在RSS feed中聯合多媒體文件。有關技術細節，請參見媒體RSS規範。

設置 MRSS

請與您的客戶經理聯繫，並指出您要使用MRSS。您的 MRSS 提要需要託管在公共 URL 上，並且其中包含的視頻和資產也必須位於公共 URL 上。URL 可以包含 http:// 或 https://。

使用 MRSS

要使用 MRSS，您需要在 Web 服務器上託管元數據和/或視頻和其他文件（縮略圖、隱藏式字幕），並創建一個XML 清單文件引用該內容。

請注意，Brightcove將每五分鐘提取一次MRSS提要。

使用 MRSS 僅提取元數據

  1. 如中所述創建XML清單文件XML清單文件。您必須使用ooyala:remoteasset僅用於元數據攝取的標記。
  2. 在您提供的 MRSS 位置之一上託管 XML 清單文件。確保文件名和路徑與您提供給 Brightcove 的路徑之一完全匹配.

使用 MRSS 提取視頻、縮略圖和隱藏字幕文件

  1. 在Web服務器上託管一個或多個文件（視頻，縮略圖和隱藏式字幕文件）。URL 必須可公開訪問。
  2. 創建一個XML清單文件。確保文件中的條目引用服務器上的文件。特別是，價值<media:content>元素的href屬性必須是服務器上文件的 HTTP URL，如下例所示： 
    <media:content url="http://mysite.com/upload/lacrosse_70.mov" />
  3. 在您提供的 MRSS 位置之一上託管 XML 清單文件。確保文件名和路徑與您提供給 Brightcove 的路徑之一完全匹配.
    筆記：不幸的是，“客戶端日誌記錄”的概念不適用於基於 MRSS 提要的處理。驗證您的 Feed 是否成功處理的最佳方法是檢查您的資產的外觀，或者使用影雲工作室或視頻雲內容管理系統 API .
