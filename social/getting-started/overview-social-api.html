介紹

Social API 允許獲取您視頻的社交共享狀態和歷史記錄。它主要用於幫助CMS集成，以使您的用戶可以在自己的CMS系統中查看此信息。

有關請求的完整詳細信息，請參見API參考。

基本網址

API 的基本 URL 是：

  https://edge.social.api.brightcove.com/v1

驗證

請求的身份驗證需要授權頭：

        Authorization: Bearer {access_token}

這access_token是一個臨時的 OAuth2 訪問令牌，必須從Brightcove OAuth服務。有關如何獲取客戶端憑據以及如何使用它們來檢索訪問令牌的詳細信息，請參閱Brightcove OAuth概述話題。

運作方式

當您請求客戶端憑據時，您需要指定所需的帳戶訪問或操作類型。以下是當前支持的操作列表社交API：

  • video-cloud/social/status/read

請注意，儘管社交API處於測試版中，但其權限為不是可在Studio管理員部分中獲取，用於管理API憑據。相反，您將必須使用OAuth API創建憑據。但是，Brightcove學習服務已創建了一個簡單的在線工具為此。

請求摘要

在當前版本的API中，僅GET支持請求。有兩組請求，在以下部分中進行了總結。

狀態請求

狀態請求返回視頻的當前社交共享狀態。您可以獲取帳戶中所有視頻或特定視頻的狀態。

歷史記錄請求

歷史記錄請求返回視頻的社交共享歷史記錄。您可以獲取帳戶中所有視頻或特定視頻的歷史記錄。
