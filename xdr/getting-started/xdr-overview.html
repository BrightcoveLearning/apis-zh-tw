概述

Brightcove的跨設備恢復（XDR）功能使觀看者可以在一個設備上開始觀看視頻，暫停視頻，然後稍後再在另一台設備上繼續觀看。當觀眾恢復播放時，他們將繼續從上次中斷的地方觀看。這為一段內容提供了無縫的觀看體驗。

XDR 服務存儲觀看活動，包括最近的播放位置。檢索此信息以在任何設備上恢復播放。

Brightcove的XDR功能：

  • 與您的客戶網站和移動設備應用程序集成
  • 在桌面和移動設備上使用基於 REST 的 API 和本地語言調用
  • 處理存儲在 Video Cloud 庫中的內容
  • 保留播放位置信息90天
  • 符合GDPR和COPPA的要求
請注意，提交 XDR 數據與通過 API 提供該數據之間存在長達 30 秒的延遲。

要求

使用跨設備恢復需要滿足以下條件：

  • 您必須使用最新版本的 Brightcove Player 和支持跨設備恢復的 Native SDK。
  • 您將需要一個服務器端應用程序來發出 REST API 請求以獲取播放位置。

帳戶設定

請與您的客戶經理聯繫，以使您的帳戶能夠跨設備恢復。

工藝流程

此功能允許觀眾在一台設備上開始觀看、暫停並繼續在另一台設備上觀看。

第1步

觀眾觀看視頻時，播放器會連續記錄正在觀看的視頻的播放頭位置。此信息存儲在 XDR API 中。

第2步

當查看者繼續在其他設備上觀看視頻時，將從 XDR API 檢索播放頭位置。在觀眾離開的地方繼續播放。

執行

要實現跨設備恢復，請按照下列步驟操作：

  1. 即使查看器數據已自動發送到Brightcove Analytics，您仍需要在應用程序中設置查看器標識符。有關詳細信息，請參閱步驟3中的鏈接文檔。

    XDR API 中發送和存儲的信息將包括：

    • account_id -視頻雲帳戶ID
    • viewer_id -正在保存播放頭的查看器的ID
    • video_id -觀看者觀看的視頻的視頻雲ID

    • playhead_seconds -當前的播放頭位置（以秒為單位）

  2. 向XDR API發出請求，以獲取查看者的播放位置。您將需要一個服務器端應用程序來為播放位置發出 REST API 請求。

    對 XDR API 的請求以秒為單位返回播放位置。

    有關詳細信息，請參見以下內容：

    • 從XDR API獲取播放位置
    • 跨設備簡歷（XDR）API參考

  3. 在您的應用程序中恢復播放。有關詳細信息，請參見以下內容：

    • Brightcove Player的跨設備簡歷
    • 使用本機SDK進行跨設備恢復
