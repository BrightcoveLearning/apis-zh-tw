介紹

Playback API 是低延遲 API，用於客戶端從網頁或移動應用程序獲取視頻或播放列表數據。這是不是用於將 Video Cloud 與您的 CMS 或其他系統集成的通用媒體管理 API - 為此，您應該使用內容管理系統 API .

Playback API 結果被過濾為僅返回可播放的視頻（狀態 = ACTIVE，攝取完成標誌 = true，如果安排了正確的數據/時間）。這意味著播放器將能夠在一個再現存在時立即播放視頻，即使其他再現仍在處理中。如果您需要獲取當前無法播放的視頻，則應再次使用CMS API反而。

還請注意，為了最大程度地提高性能，播放API所訪問的視頻數據會緩存一小段時間。特定數據集ID緩存的時間長短會有所不同，但可能長達20分鐘。

另請參閱API參考。

如果您只需要有關播放器中當前視頻的信息，則可以使用mediainfo對象，而不是進行Playback API調用。

這mediainfo對象包含有關當前加載到播放器中的視頻的信息，而 Playback API 可以檢索有關任何視頻的信息。有關詳細信息，請參見來自mediainfo的視頻元數據文檔。

用例

這Brightcove 播放器和Brightcove SDK 播放器包括與 Playback API 的方法相同的目錄方法，並且在大多數情況下，如果您只是在運行時檢索視頻或播放列表以進行播放，則可以簡單地使用這些方法。

但是，您可能希望在沒有播放器的移動應用程序屏幕中提供視頻或播放列表的視圖。

另外，例如，有時您可能想在不讓播放器出現在屏幕上的情況下檢索視頻或播放列表以顯示有關它們的信息，例如，作為視頻門戶的登錄頁面。

Playback API 在這些情況下很有用。

一般信息

Playback API 當前不支持來自使用 IPv6 協議的 IP 地址的請求。

基本網址

Playback API 的基本 URL 是：

https://edge.api.brightcove.com/playback/v1
  • 你必須使用https作為協議。
  • 使用IP限制的客戶必須用https://edge-elb.api.brightcove.com/playback/v1而不是標準網址；有關在北美以外受IP限制使用的播放器的更多詳細信息，請務必查看播放器配置指南以防止可能出現的問題。

帳戶路徑

在任何情況下，都將請求特定的視頻雲帳戶。因此，您將始終添加術語accounts後跟您的帳戶 ID 到基本 URL：

https://edge.api.brightcove.com/playback/v1/accounts/{account_id}

驗證

請求通過策略密鑰進行身份驗證，可以通過以下三種方式之一進行傳遞：

  1. 在爭論Accept標頭： 
    Accept: application/json;pk={policy_key}

    這是基於瀏覽器的客戶端的推薦方法，因為作為瀏覽器 CORS“飛行前”檢查的一部分，它允許請求在沒有額外請求的情況下繼續進行。這可以節省第一次發出瀏覽器請求時的延遲。

  2. 在Authorization使用realm關鍵字的標頭BCOV-Policy： 
    Authorization: BCOV-Policy {policy_key}
  3. 在一個BCOV-Policy標頭： 
    BCOV-Policy: {policy_key}

獲取策略密鑰

您可以通過三種方式獲取策略密鑰：

  1. 每個Brightcove播放器都會自動分配一個。您可以找到有關獲取玩家策略密鑰的說明這個文件 .由於策略密鑰適用於整個帳戶，因此無論該播放器是否嵌入到頁面中，您都可以使用它。
  2. 您可以使用以下命令生成基本策略密鑰策略密鑰快速入門
  3. 您可以向政策API

令牌認證

支持 HLS 和 DASH 內容的令牌認證。如果您需要令牌認證，請與您的客戶經理聯繫，以為您的帳戶啟用它。

避免使用硬編碼的網址

縮略圖、海報、視頻文件和其他媒體的 URL 不應在您的頁面或應用程序中進行硬編碼。這播放接口將返回媒體文件 URL 的最新緩存版本，但 URL 本身可能會發生變化。你應該使用內容管理系統 API（ 或者播放接口 ) 請求在每次頁面加載時檢索這些 URL，或將它們緩存不超過 6 小時。

緩存視頻和圖像URL

您可以緩存視頻和圖像的 URL 以提高頁面性能，但必須定期刷新緩存。如果您緩存檢索到的URL以提高頁面性能，請確保通過每六個小時至少重複一次API調用來刷新緩存。

方法

API 僅支持以下類型的 GET 請求：

  • 通過ID獲取視頻
  • 通過參考ID獲取視頻
  • 取得影片
  • 通過SSAI獲取視頻
  • 通過ID獲取播放列表
  • 通過參考ID獲取播放列表

搜索視頻

你必須使用一個啟用搜索的策略密鑰 , 使用 Playback API 搜索視頻時。有關詳細信息，請參見播放API參考文檔。

Playback API 搜索限制為每個請求 100 個視頻，上限為 1000 個視頻。當您需要返回超過 1000 個視頻時，您必須使用內容管理系統 API .

參數

請注意，所有參數都是可選的。他們適用於GET Videos僅請求。

獲取視頻請求參數
範圍 描述
q 查詢字符串以獲取搜索-有關詳細信息，請參見搜索指南
limit 要返回的視頻數量-必須為1到100之間的整數。默認：20
offset 要跳過的視頻數量（用於分頁結果）。必須為正整數。默認：0
sort 一個字符串，指定要作為排序依據的字段。
從...開始-降序排序。
如果為q提供，則默認排序是按“分數"（搜索結果與原始查詢的相關性）。
如果沒有價值q提供，則默認排序為updated_at下降。
以下字段對排序有效：name , reference_id , created_at , published_at , updated_at , schedule_starts_at , schedule_ends_at , state , plays_total， 和plays_trailing_week
有關詳細信息，請參閱獲取視頻 Playback API 參考的部分

使用搜索參數

布賴特科夫的播放接口提供了一種在您的視頻中搜索視頻的程序化方式視頻雲圖書館。

要對您的視頻數據執行基本和復雜的搜索，您將使用q範圍：

https://edge.api.brightcove.com/playback/v1/accounts/1752604051111/videos?q={search terms}

有關如何搜索視頻的詳細信息，請參見搜尋影片文檔。

分頁結果

使用limit參數來指定您要在請求中返回多少個項目 - 最多 100 個。然後您可以使用offset用於翻閱大於的結果集的參數limit .這offset是要跳過的項目數。

例如，以下搜索返回總結果集中的視頻51-75，假設總結果集中至少有75個視頻：

/videos?q=updated_at:2014-01-01..2014-06-30&limit=25&offset=50

這limit和offset參數只能用於視頻。

分頁最佳做法

從 Playback API 分頁瀏覽結果集時，建議按照以下步驟操作：

  1. 使用limit和offset從結果集中返回數據組的參數。 
    /accounts/578380111111/videos?q=tags:nature&limit=20&offset=50
  2. 對於每個響應，您將看到count字段始終顯示結果集中的最大視頻數。 
    {"count": 171,
  "videos": [ ... ]
}
    請記住，Playback API 搜索的上限為 1000 個視頻。
  3. 總數count您第一次請求的視頻數量可用於確定您還需要請求多少頁。

    或者，您可以詢問count每個響應的值並繼續直到count小於或等於頁面大小乘以limit . 
    count <= page-number * limit

排序視頻結果

使用參數sort=field_name指定結果的排序方式。您可以對以下視頻字段進行排序： [ 1-1]

使用減號 ( sort=-field_name ) 為降序。
  • 名稱
  • reference_id
  • created_at
  • Published_at
  • 更新時間
  • schedule_starts_at（注意：這是種類場 - 的搜索字段是schedule.starts_at )
  • schedule_ends_at（注意：這是種類場 - 的搜索字段是schedule.ends_at )
  • 狀態
  • plays_total [ 1-2]
  • play_trailing_week [ 1-2]

筆記

  • [ 1-1] 如果您沒有為視頻搜索呼叫提供排序值，結果將按相關性排序。如果您不提供GET視頻通話，結果將按以下順序排序updated_at下降。
  • [ 1-2] 你可以排序plays_total或者plays_trailing_week，但這些字段不包含在結果中

所有視頻和大數據集

如果您要檢索帳戶中的所有視頻或大量視頻，則必須注意以下幾點：

  1. 您可能想使用允許的最大limit (100)，但最好以 25 個或更少的批次檢索視頻，以盡量減少 API 請求超時的可能性
  2. 在翻閱大型數據集時，可能會在操作過程中更新視頻數據，這可能導致項目的響應發生偏移：
    • 您可能會在連續頁面上看到重複的項目
    • 某項可能已丟失，因為它已轉移到先前的響應集

    為了解決第一種可能性，您的應用程序應該在您完成檢索視頻後刪除完整的項目列表。要處理第二種可能性，您需要將檢索到的項目總數（去重後）與您期望的數量進行比較，然後重新運行請求，按 last_modified_date（降序）對結果進行排序 - 您不應該需要檢索多於一批以撿拾遺漏的物品。

  3. 您可以通過對返回的結果進行適當排序來降低上一項中出現場景的可能性。默認排序方式關聯 for search 基於一種複雜的算法，該算法看起來是關鍵字、標籤和自定義字段值的組合。如果您要搜索基於多個關鍵字，標籤和/或自定義字段的視頻，則按相關性排序正是您想要的。但是，如果您只是嘗試檢索所有或大量視頻，請設置sort參數顯式將使您可以更好地控制返回項目的順序。

錯誤回應

如果您嘗試將“獲取視頻"端點與未啟用搜索的策略密鑰一起使用，您將從Playback API收到以下錯誤消息：

[
  {
    "api_call": "search",
    "error_subcode": "API",
    "error_code": "ACCESS_DENIED",
    "message": "Access to this resource is forbidden by access policy."
  }
]

查找相關視頻

Playback API 允許您在您的帳戶中搜索與指定視頻相關的視頻。基於name和short description在指定視頻中，API將在以下字段中搜索任何部分匹配項：

  • name
  • short description
  • long description
  • tags

如果地理限制生效，則可能不會收到整頁的結果。

筆記：
  • 你必須使用一個啟用搜索的策略密鑰 , 使用 Playback API 查找相關視頻時。
  • 隨著我們改進查找相關視頻的算法，此端點的響應結果可能會發生變化。如果您不希望更改結果，或者想要精確控制查找相關視頻，則應使用搜尋影片功能。

參數

以下 URL 參數可用於 GET 相關視頻請求：

GET相關視頻請求參數
範圍 描述
limit 要返回的視頻數量（默認 = 20；最大 = 100）
ad_config_id 附加您的ad_config_id以返回帶有SSAI的視頻

默認情況下，響應最多包含20個視頻。您可以使用limit參數最多可返回 100 個視頻，但會增加延遲。

請注意，沒有offset查找相關視頻時使用該參數，因此您將無法翻閱視頻結果。

使用 SSAI 請求視頻

即使您在應用程序中未使用Brightcove Web播放器或Brightcove Native SDK，您仍然可以在自己的播放器中使用服務器端廣告插入（SSAI）。

筆記：
  • 要使用 SSAI，您的 Video Cloud 帳戶需要配置為動態交付並為 SSAI 啟用。請與您的客戶經理聯繫以開始使用此功能。
  • 目前，SSAI不支持多個音軌。

使用SSAI獲取視頻

要返回帶有 SSAI 的視頻，請將您的廣告配置 ID 作為 URL 參數附加到您的 Playback API 請求中。對於此請求，您可以使用視頻 ID 或視頻參考 ID。

要使用服務器端廣告插入 (SSAI)，您的視頻需要為動態交付攝取 .

帶有URL參數的端點

追加您的ad_config_id到Playback API請求的末尾：

https://edge.api.brightcove.com/playback/v1/accounts/{account_id}/videos/{video_id}?ad_config_id={ad_config_id}

例子

https://edge.api.brightcove.com/playback/v1/accounts/1752604059001/videos/5565662146001?ad_config_id=1234abcd
