介紹

這是一個簡單的示例，說明如何使用播放列表 API 獲取視頻雲播放列表的數據並將其呈現為 JSON 或 MRSS 數據，您可以將這些數據傳遞給聯合合作夥伴。

請注意，與聯合提要一樣，此應用返回的提要數據包括僅視頻數據，而不是播放列表元數據 .
提供此示例和相關代碼作為生產開發的指南。該代碼未經認證可用於生產用途，也未經官方支持。

局限性

為了確保您的內容安全，Brightcove API返回的視頻和資產的URL為簽，這意味著它們包含僅在短時間內（通常是幾個小時）有效的令牌，但這可能會因帳戶配置而異。這意味著此工俱生成的 MRSS 代碼中的 URL 只會在一段時間內有效。

您可以通過以下兩種方式之一處理此問題：

  • 您可以要求 Brightcove 支持增加附加到您帳戶的媒體 URL 的令牌的“生存時間”。該值可以設置為最多一年的任何時間段。但是，請注意，這是帳戶級別的設置，更改將應用於您的所有視頻。
  • 另外，您可以創建自己的該應用程序版本，並讓其在令牌有效期內定期創建MRSS Feed的新版本。

要求

要自己構建它，您需要做兩件事：

  • 您可以從中獲取的視頻雲播放列表ID 工作室
  • 您的Video Cloud帳戶的策略密鑰，您可以從任何Brightcove播放器在您的帳戶中，或從政策API

可選的

您不需要出於任何功能原因執行此操作，但為了美化 MRSS 代碼並使其更易於閱讀，我們包含了美化 JavaScript 庫。
