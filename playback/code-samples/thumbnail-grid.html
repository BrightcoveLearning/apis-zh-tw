概述

儘管布賴特科夫畫廊提供用於構建視頻門戶的強大系統，您還可以構建一個簡單的 DIY 門戶。如果您不想在用戶選擇視頻之前在網頁上加載播放器，則可以顯示此示例中顯示的視頻縮略圖網格。選擇縮略圖後，Brightcove 播放器會加載到包含所選視頻的頁面上。

你需要什麼

構建該門戶所需的全部是：

  • 視頻雲帳戶
  • 啟用搜索的策略密鑰，用於驗證對Playback API的請求
  • 一些 Web 開發技能（HTML、CSS 和 JavaScript）

設計注意事項

獲取視頻數據

雖然無論是內容管理系統 API或者 Playback API 可用於獲取視頻數據，Playback API 是這裡顯而易見的選擇，原因如下：

  • 由於響應數據已緩存（最多20分鐘），因此速度更快
  • 它返回播放視頻所需的所有數據，而內容管理系統 API需要兩個單獨的請求來獲取視頻元數據和視頻源
  • 它啟用了CORS，這意味著您不需要通過服務器端代理應用程序路由API請求

但是，Playback API提供了兩種獲取視頻數據的方法。有一種通過 id（或參考 id）獲取視頻的基本方法，我們可以輕鬆地遍歷一組 id 以請求每個。然而，這不是很有效。

更好的選擇是使用視頻搜索方法，該方法使我們可以按ID查找所有視頻，並通過一個請求將其返回。請求的搜索參數如下所示：

q=id:6543217890%20id:1234567890%20id:987654321...

這%20在這個搜索字符串中是一個 URI 編碼的空間。有關搜索語法的更多信息，請參見CMS /播放API：視頻搜索 .

玩家

對播放器沒有特殊要求 - 默認的 Brightcove 播放器可以正常工作。

一些用戶可能根本不觀看任何視頻，因此在需要之前使用帶寬來加載播放器是沒有意義的。我們將使用Brightcove 播放器加載程序在第一次選擇視頻進行播放時動態加載播放器。為了最大限度地提高性能，我們還將預加載 Brightcove 播放器 JavaScript，但如果播放器加載器尚未存在，播放器加載器將自動執行此操作。

獲取策略密鑰

將為所有Brightcove播放器自動生成策略密鑰。但是出於安全原因，標準策略密鑰不允許使用Playback API搜索視頻。對於此示例，我們確實需要啟用搜索的鍵。您可以使用政策API，但最簡單的方法是使用Brightcove 學習示例應用程序並確保選中獲取啟用搜索的密鑰的選項。
