狀態代碼和錯誤

地位 錯誤代碼 描述
200 SUCCESS 發送正常響應。
400 BAD_REQUEST 格式錯誤的請求，即提供了重複的查詢參數
401 INVALID_POLICY_KEY 必須提供帶有合法策略密鑰的BCOV-Policy標頭。
403 ACCESS_DENIED，VIDEO_NOT_PLAYABLE，PLAYLIST_NOT_PLAYABLE，JWT_AUTH 提供的策略密鑰不允許此帳戶或視頻，或者請求的資源處於非活動狀態。VIDEO_NOT_PLAYABLE可以通過單個視頻請求從 Roebuck 返回。它指示視頻未通過可播放的檢查（已按計劃進行了已攝錄，活動，檢查）。
404 VIDEO_NOT_FOUND，PLAYLIST_NOT_FOUND，RESOURCE_NOT_FOUND，ACCOUNT_NOT_FOUND 未找到指定的資源。
405 METHOD_NOT_ALLOWED 此api僅允許GET，HEAD和OPTIONS使用。
500 SERVER_ERROR 內部Wedge服務器錯誤，通常在日誌中會有更多信息。
502 SERVER_ERROR，ACCOUNT_RETRIEVE_FAILURE，VIDEO_RETRIEVE_FAILURE，VIDEO_URLS_RETRIEVE_FAILURE，PLAYLIST_RETRIEVE_FAILURE，PLAYLIST_VIDEOS_RETRIEVE_FAILURE 來自後端服務器的響應很差，通常Roebuck或Nomic返回了5xx錯誤。
503 SERVICE_UNAVAILABLE 楔不返回。如果負載均衡器中沒有活動的服務器，則可能發生。那會很糟糕。（Playerback API不會進行速率限制，也不會調用它的服務。）
504 SERVER_TIMEOUT 對後端服務器或它們所依賴的服務器之一的調用超時。

錯誤回應

根據API設計標準，錯誤響應（4xx和5xx）由一個或多個映射的錯誤響應列表組成，其中包括error_code，一個可選的error_subcode，以及非本地化（可選）message詳細信息。

錯誤代碼

這error_code這意味著人類可以理解並由客戶端代碼使用來選擇不同的操作和可翻譯的用戶消息。錯誤代碼是 UPPER_SNAKE_CASE。

地位 錯誤代碼 錯誤源
403 ACCESS_DENIED 名義政策
403 VIDEO_NOT_PLAYABLE 羅巴克
404 VIDEO_NOT_FOUND 羅巴克
其他 VIDEO_RETRIEVE_FAILURE如果無法檢索視頻 羅巴克
其他 VIDEO_URLS_RETRIEVE_FAILURE如果無法檢索來源或圖像 羅巴克

筆記

[ 1] ACCESS_DENIED / POLICY_ERROR指評估策略時出現的任何意外錯誤（包括獲取地理限制帳戶的視頻信息）。可能值得檢查此錯誤的日誌中的異常。

當您請求播放列表時，如果由於地理限製而無法播放某些視頻，服務器將返回 200 OK 狀態。響應將包含播放列表數據和錯誤數組，指示某些視頻被省略。
