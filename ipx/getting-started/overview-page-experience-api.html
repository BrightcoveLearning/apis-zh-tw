介紹

有兩個與頁內體驗相關聯的 API，但它們的用途截然不同：

頁面內體驗 API (REST)

頁內體驗 API 是一種 REST API，允許您以編程方式管理頁內體驗。它主要用於將視訊雲端平台與您的 CMS 應用程式整合。

另請參閱完整API參考有關可用請求的完整詳細信息。

頁面內體驗用戶端 API

頁內體驗客戶端 API 是一個 JavaScript 庫，可幫助您在運行時管理頁內體驗的行為，類似於使用 Brightcove Player API 控製播放器行為的方式。API 可在運行一種或多種 Brightcove 嵌入式體驗的任何頁面上使用。見概述：頁內體驗客戶端API想要查詢更多的信息。

本概述的其餘部分涉及用於在後端管理頁內體驗的 REST API。

有關頁內體驗的一般介紹，請參見概述：建立、編輯和發佈頁面內體驗。

以下部分概述了 4 組 REST API 請求：

  • 體會
  • 模板
  • 主題
  • 字型

體會

體驗要求分為三個子類別：

管理體驗
通過這些請求，您可以獲得帳戶所有體驗的列表。您還可以創建新體驗，以及獲取、更新或刪除特定體驗。最後，您可以複製體驗，以建立新體驗的起點。
發佈/取出發佈體驗
這些請求都是為了獲取帳戶或特定模板的所有模板的列表。
管理即時體驗的狀態
這兩個請求允許您獲取實時體驗的當前狀態或更改該狀態（從preEvent 到live， 例如）。

模板

模板控制體驗的整體外觀和感覺。無法建立自訂樣板，但有數個標準樣板，隨著時間的推移可能會加入更多樣板。API 允許您獲取可用於您的帳戶的模板列表，或檢索特定模板。

主題

主題允許您自定義模板以創建您想要的外觀和感覺。API 提供了完全管理您的主題、創建、獲取、更新或刪除它們的方法。

字型

您可以使用自己的字體。這些方法允許您添加字體、更新它們、獲取它們或刪除它們。這些是通過 CSS 加載的 Web 字體。
