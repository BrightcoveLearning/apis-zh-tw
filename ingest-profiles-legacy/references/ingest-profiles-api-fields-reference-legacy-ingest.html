個人資料欄位

如果用戶以編程方式拉取一個配置文件，目的是使用檢索到的內容更新該配置文件，則用戶將需要清理該配置文件，刪除諸如“id " 用於演繹，而不是配置文件，以及 " version "應該完全刪除。
財產 類型 描述 默認
id 細繩 個人資料 ID（只讀） 沒有任何
version 整數 配置文件版本（只讀） 沒有任何
account_id 整數 這視頻雲帳戶ID 沒有任何
name 細繩 個人資料名稱 - 在帳戶中必須是唯一的 沒有任何
description 細繩 個人資料說明 沒有任何
date_created 整數（紀元時間戳（毫秒） 日期配置文件已創建（只讀） 沒有任何
date_last_modified 整數（紀元時間戳，以毫秒為單位） 日期配置文件上次修改（只讀） 沒有任何
brightcove_standard 布爾值 只讀 - false用於所有自定義配置文件 false
digital_master 目的 定義是否以及如何歸檔數字母版 false
numRenditions 整數 配置文件中的演繹次數，包括圖像演繹（只讀） 沒有任何
renditions 大批 再現對像數組 []
packages 大批 DRM軟件包數組 []

筆記

  • [ 1-1] 重現舊人的行為視頻雲攝取系統使用值 10 - 請參閱內容安全（DRM 和 HLSe）更多細節
  • [ 1-2] 更高的質量意味著更高的比特率，反之亦然。可能的值：
    1. 高度壓縮。平庸的視覺質量，但文件很小
    2. 可接受的質量
    3. 高品質。比大多數網絡視頻更好
    4. 很棒的品質。看起來很棒
    5. 幾乎無損。大文件。不建議使用，除非您打算再次對該輸出進行編碼
  • [1-3] 你不應該低於 16 歲；對於大多數文件，16 幾乎是無損的。通常24點左右看起來不錯。大約35個看起來非常壓縮
  • [1-4] 更多的參考幀會導致更高的壓縮質量，但會增加解碼複雜度。實際上，超過5幾乎沒有太大好處。我們默認為 3 作為壓縮和解碼複雜性的良好折衷。設置為“自動”，讓我們的速度設置自然選擇這個數字
  • [1-5] 較低級別更容易解碼，但較高級別提供更好的壓縮。例如，iPhone 3gs僅支持Baseline配置文件。Main 和 High 配置文件在壓縮方面是一個明確的步驟，並且可以很好地用於 Web 播放，但要小心 Flash Player 中的 HD High Profile 視頻
  • [1-6] 主要用於設備兼容性。例如，iPhone支持H.264 3級，這意味著視頻的解碼器_bitrate_cap不能超過10,000 kbps。通常，只有在針對需要它的特定設備時才應更改此設置。
  • [1-7] B 幀僅受 H.264 Main 或 High profile 支持，不支持 Baseline。使用 B 幀可以在相同比特率下提高質量，但對 CPU 的要求更高。這在舊計算機或設備上可能會出現問題，但在現代硬件上通常可以流暢播放。Video Cloud 默認為 0 以獲得最廣泛的兼容性。我們建議將值設為 3 以進行壓縮/質量改進。高於 5 或 6 的值很少提供太多好處，並且會大大增加編碼時間
  • [1-8] 對於'破折號' 輸出，當前支持的配置文件是：
    • live -將DASH配置文件設置為“urn:mpeg:dash:profile:isoff-live:2011並使用一個SegmentTemplate對於每個AdaptationSet。對於DRM，它將使用8字節的IV，對於PlayReady將使用1.1版，其中包括PSSH媒體中的原子
    • hbbtv_1.5 -將DASH配置文件設置為“urn:hbbtv:dash:profile:isoff-live:2012"，並包括音頻和視頻表示形式所需的屬性。對於DRM，它將使用16字節的IV，對於PlayReady將使用1.0版，但不包括PSSH原子數
  • [1-9] 必須在再現級別添加水印 - 到您想要加水印的每個再現

數字主場

這digital_master object 定義是否以及如何存儲數字母版，您可以稍後使用它進行重新轉碼。看存檔大師詳情。

財產 類型 描述 默認
rendition 細繩 存檔的再現 - 要么passthrough存檔原始源視頻或reference_id對於其中之一演繹在配置文件中定義或none如果您不想存檔母版 passthrough
distribute 布爾值 是否digital_master應推送到 CDN 以供播放 false

包裝領域

軟件包用於DRM。如果您的帳戶未啟用DRM，只需離開packages數組為空。

注意：包引用格式為reference_id。要使用包，您的再現必須具有 reference_ids，並且再現必須在配置文件的 JSON 中定義 前 包被定義。

財產 類型 描述 默認
drm 大批 DRM 類型（僅適用於 MPEG-DASH） [2-2] 沒有任何
package_type 細繩 DRM 包類型 [2-2] 沒有任何
renditions 大批 一個字符串數組，每個字符串都是reference_id用於在配置文件中呈現[2-3] []

筆記

  • [2-1] 當前支持的值drm數組是marlin , playready， 和widevine
  • [2-2] 看內容安全（DRM 和 HLSe）詳情
  • [2-3] 請注意，配置文件中的再現必須在包之前的 JSON 中定義，以便 reference_ids 存在

水印字段

水印圖像被刻錄到右下角的每個視頻幀中。

財產 類型 描述 默認
url 細繩 圖片的網址 沒有任何
width 細繩 [ 3-2] 水印寬度作為像素或幀寬度的百分比 - 例如20% [ 3-1] 沒有任何
height 細繩 [ 3-2] 水印的高度作為像素或幀高度的百分比 - 例如10% [ 3-1] 沒有任何
x 細繩 [ 3-2] 從視頻幀的左邊緣到水印中心的距離，以像素或幀寬度的百分比表示 沒有任何
y 細繩 [ 3-2] 從視頻幀的頂部邊緣到水印中心的距離，以像素或幀寬度的百分比表示 沒有任何

筆記

  • [ 3-1] 如果只指定寬度，圖像縱橫比將自動保持
  • [ 3-2] 雖然width , height , x， 和y字段是字符串值，它們包含的數字（像素或百分比）必須是整數 - 像“5.2%”這樣的十進制值將被拒絕。
