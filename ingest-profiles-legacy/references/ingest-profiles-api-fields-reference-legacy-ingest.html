    內嵌設定檔欄位參考

    這是為舊式內嵌系統內嵌設定檔的欄位參考。
    請注意，本指南適用於舊版內嵌系統；大部分的客戶都位於動態傳遞系統上。

    個人資料欄位

    如果用戶以編程方式提取配置文件，目的是使用檢索到的內容更新該配置文件，則該用戶將需要清理配置文件，並刪除“id“用於演繹，但不用於個人資料，”version”應完全刪除。
    屬性 類型 描述 預設
    id 串 設定檔 ID (唯讀) 無
    version 整數 設定檔版本 (唯讀) 無
    account_id 整數 的Video Cloud帳戶ID 無
    name 串 個人資料名稱-在帳戶中必須唯一 無
    description 串 資料描述 無
    date_created 整數（紀元時間戳記（毫秒） 建立設定檔的日期 (唯讀) 無
    date_last_modified 整數（以毫秒為單位的紀元時間戳） 上次修改設定檔的日期 (唯讀) 無
    brightcove_standard 布林 唯讀- false適用於所有自訂設定檔 false
    digital_master 對象 定義是否以及如何歸檔數位主版 false
    numRenditions 整數 描述檔中的轉譯數目，包括影像轉譯 (唯讀) 無
    renditions 陣列 轉譯物件的陣列 []
    packages 陣列 DRM 套件陣列 []

    渲染字段

    屬性 類型 描述 預設
    id 串 轉譯 ID (唯讀) 無
    aspect_mode 枚舉（字符串） 如何處理源和目標寬高比之間的不匹配- preserve，stretch，crop，請注意-如果您選擇的值不是preserve，寬度和高度都可以修改以實現所需的寬高比 preserve
    audio_bitrate 整數 kbps 中的音訊位元率 匹配來源
    audio_sample_rate 整數 採樣率Hz-（AAC：8000、11025、12000、16000、22050、32000、44100、48000、96000；FLV：11025、22050和44100） 匹配來源
    max_audio_sample_rate 整數 最大音頻採樣率（以Hz為單位），優先audioSampleRate（AAC：8000、11025、12000、16000、22050、32000、44100、48000、96000；FLV：11025、22050和44100） 匹配來源
    audio_constant_bitrate 布林 力CBR，與VP6不兼容 false
    audio_channels 整數 音頻通道數（null（不要更改），1要么2） null
    audio_codec 串 音訊轉碼器- aac | mp3 由格式或配置文件確定
    max_aac_profile 串 最大 Aac 配置文件：AAC-LC，他-Aac，或他-AAC-v2 我們建議使用aac-lc當再現的音頻比特率小於48kbps時
    constant_bitrate 布林 是否CBR應該被強制引渡 無
    crf 整數 位元控制設定，並設定常數速率因子。有效值為0-51，值越小質量越好。 [1-3] 無
    decoder_bitrate_cap 整數 以kbps為單位，通過緩衝區饋送到解碼器的最大比特率 無
    decoder_buffer_size 整數 以kbps為單位，當使用bitrate_cap 無
    encryption_method 串 使用的加密方法，例如aes-128 [1-1] 無
    encryption_key_rotation_period 整數 為每組細分使用不同的鍵，在這麼多的細分之後旋轉到新的鍵[1-1] null
    fixed_keyframe_interval 布林 防止場景剪輯之類的東西放置額外的內容keyframes false
    forced_keyframe_interval 整數 強制每 X 幀一個關鍵幀，但仍然允許其他關鍵幀用於場景等。 無
    forced_keyframe_rate 浮動 強制關鍵幀速率，h264僅，如果忽略forced_keyframe_interval用來 無
    format 串 影片格式- ts，mp4，flv 無
    frame_rate 浮動 每秒影格數 與來源相同
    h264_reference_frames 整數 要使用的參考框架數目。部分決定於speed。 [1-4] 3
    h264_profile 串 H.264具有三種常用的配置文件：Baseline（最低），Main和High。 [1-5] baseline
    h264_level 浮動 限制比特率和宏塊。 [1-6] 計算
    h264_bframes 整數 最大連續B幀數（0-16） [1-7] 0
    hls_optimized_ts 布林 針對HLS優化的時間段-僅適用於HLS輸出 無
    keyframe_interval 整數 關鍵幀之間的最大幀數 250，覆蓋keyframe_rate
    keyframe_rate 浮動 每秒最大關鍵幀 keyframe_interval一百五十
    label 串 圖像再現的圖像類型- poster要么thumbnail 無
    live_stream 布林 這是否將是實時視頻流 無
    live_dvr_sliding_window_duration 整數 流的持續時間（以秒為單位）可用於LiveDVR交付 300
    max_video_bitrate 整數 以kbps為單位，123 = 123kbps-僅適用於h264 無
    max_frame_rate 浮動 限制c frame_rate而不是設置它，用作替代frame_rate 無
    max_hls_protocol_version 整數 設定要使用的最大 HLS 通訊協定版本。特殊功能將視情況提供使用 (2-5) 3 用於即時串流，5 表示 AES 加密輸出，否則為 2
    media_type 串 video，audio， 要么image 無
    one_pass 布林 強制一次通過編碼 false
    reference_id 串 帳戶內唯一轉譯的識別碼-DRM 需要 無
    segment_seconds 整數 要包含在區段中的內容秒數 10對於HLS，2對於DASH
    skip 對象 跳過再現格式的一組條件，請參見條件輸出詳情 null
    skip_video 布林 不包括視頻流-用於僅音頻轉譯 false
    speed 整數 1-5-目標轉碼速度。轉檔速度較慢，可以進行更進階的檔案壓縮，而跳過一些進階壓縮功能可以加快轉碼速度 3

    注意事項

    • [1-1]重現舊的行為Video Cloud攝取系統使用值10-請參閱內容安全性（DRM和HLSe）更多細節
    • [1-2]更高的質量意味著更高的比特率，反之亦然。可能的值：
      1. 高度壓縮。平庸的視覺品質，但小文件
      2. 可接受的品質
      3. 品質良好。比大多數網絡視頻更好
      4. 高品質。看起來很棒
      5. 幾乎無損大型檔案。不推薦，除非您打算再次編碼此輸出
    • [1-3]您不必低於16歲；大多數文件16幾乎是無損的。大約 24 通常看起來相當不錯。35 左右看起來很壓縮
    • [1-4]更多參考幀會導致壓縮質量稍高，但會增加解碼複雜度。在實踐中，超過 5 很少有很多好處。我們默認為 3 是壓縮和解碼複雜性的一個很好的折衷。設定為「自動」，讓我們的速度設定自然選擇此號碼
    • [1-5]較低的級別更容易解碼，但是較高的級別提供更好的壓縮。例如，iPhone 3gs 只支援「基線」描述檔。主要和高設定檔在壓縮中是一個明確的一步，並且可以正常播放網頁，但要小心 Flash Player 中的高清高檔視頻
    • [1-6]主要用於設備兼容性。例如，iPhone 支援 H.264 等級 3，這表示影片的解碼器不能超過 10,000 kbps。一般而言，如果您的目標是特定裝置需要此設定，您才應該變更此設定。
    • [1-7] B幀僅受H.264 Main或High配置文件支持，而不受Baseline支持。使用 B 幀可以以相同的位元速率提高品質，但在 CPU 上的要求更高。這在舊電腦或裝置上可能會有問題，但通常在現代硬體上順暢播放。Video Cloud 預設為 0，以獲得最大的相容性。為了改善壓縮/品質，我們建議使用 3 的值。高於 5 或 6 的值很少提供很多好處，並且大大增加編碼時間
    • [1-8]對於“破折號”輸出，當前支持的配置文件為：
      • live -將DASH配置文件設置為“urn:mpeg:dash:profile:isoff-live:2011並使用一個SegmentTemplate每個AdaptationSet。對於DRM，它將使用8字節的IV，對於PlayReady將使用1.1版，其中包括PSSH媒體中的原子
      • hbbtv_1.5 -將DASH配置文件設置為“urn:hbbtv:dash:profile:isoff-live:2012”，並包括音頻和視頻表示形式所需的屬性。對於DRM，它將使用16字節的IV，對於PlayReady將使用1.0版，不包括PSSH原子數
    • [1-9]必須在演繹級別上添加水印-要添加水印的每個演繹版本
