用戶功能

下面是相關的函數視頻雲您可能希望為您的 CMS 用戶提供：

  • 將新視頻添加到視頻雲
  • 替換一個視頻雲帶有新版本的視頻
  • 更新視頻的元數據，例如標題、描述和標籤
  • 刪除影片
  • 創建播放列表
  • 更改播放列表中的視頻
  • 刪除播放列表
  • 創建視頻播放器
  • 修改視頻播放器屬性，例如尺寸或樣式
  • 通過插件為視頻播放器添加特殊功能
  • 發佈單個視頻或播放列表
  • 提供有關視頻負載，觀看次數，播放率，參與度等的分析數據。

您可能不希望向最終用戶公開所有這些功能 - 例如，您可能不想讓他們刪除視頻。整合的優勢之一視頻雲使用您的 CMS，而不是讓用戶直接訪問視頻雲 Studio 是您可以選擇通過 Brightcove API 向用戶公開哪些功能。

驗證

對於所有Brightcove API請求，身份驗證均基於OAuth2訪問令牌。獲取訪問令牌有兩個步驟：

  1. 創建具有所需API操作權限的客戶端憑據
  2. 使用客戶端憑據創建臨時訪問令牌以對 API 請求進行身份驗證

創建客戶端憑證

創建客戶端憑據是一項一次性操作，可以通過視頻雲工作室或者OAuth API .但是，只要您這樣做，client_id和client_secret返回，必須將其保存以請求訪問令牌。

這client_secret僅在創建客戶端憑據時顯示 - 永遠無法再次檢索。如果丟失客戶端機密，則必須創建新的客戶端憑據。

創建訪問令牌

臨時訪問令牌是使用OAuth API .這client_id和client_secret必須是 BASE64 編碼的並作為Basic授權字符串。

這access_token返回的是通過 API 調用在 Authorization 標頭中傳遞的：

    Authorization: Bearer your_access_token

訪問令牌的有效期為5分鐘。除非您正在執行某種將進行數百個連續 API 調用的批處理操作，否則為每個 API 調用請求一個新的而不是試圖跟踪超時是有意義的。

新增中影片

如果您希望讓用戶將視頻添加到視頻雲從您的 CMS，您可以使用動態攝取 API .我們建議您讓用戶將視頻上傳到您的存儲庫，可以是 S3 存儲桶或只是面向公眾的服務器。動態攝取系統可以提取視頻並將它們添加到視頻雲系統通過下面概述的兩步過程。

將視頻對象添加到視頻雲

第一步是在視頻雲系統通過製作郵政向內容管理系統 API：

    https://cms.api.brightcove.com/v1/accounts/:account_id/videos

請求正文將在一個JSON對象 - 至少，視頻name，但您也可以包含其他元數據，例如description和tags：

    {
    "name": "Woodpecker",
    "description": "A bird that hunts insects inside wood",
    "reference_id": "Bird_Woodpecker.mp4",
    "tags": ["bird", "air", "nature"]
    }
一種reference_id可以添加到任何視頻。參考ID在帳戶中必須唯一。如果您的視頻在您的 CMS 系統中有 ID，最好使用該 ID 作為參考 ID 視頻雲系統。

攝取視頻

創建視頻對象時，內容管理系統 API將返回一個JSON包含視頻屬性的對象。您將提取視頻id來自JSON，並使用它來調用動態攝取 API請求視頻的攝取和轉碼：

    https://ingest.api.brightcove.com/v1/accounts/ACCOUNT_ID/videos/VIDEO_ID/ingest-requests

你會再次發送JSON在指定視頻文件位置的請求正文中：

    {
      "master":{
        "url":"http://learning-services-media.brightcove.com/videos/mp4/Bird_Woodpecker.mp4"
      },
      "profile":"multi-platform-extended-static",
      "capture-images": true
    }

這profile這是指定在轉碼過程中應創建哪些再現的攝取配置文件。在大多數情況下，以下標準配置文件之一應足夠：

動態投放配置文件

  • multi-platform-extended-static
  • multi-platform-standard-static

舊版攝取檔案

  • videocloud-default-v1 (the default)
  • screencast-1280
  • smart-player-transition
  • single-bitrate-high
  • audio-only
  • single-bitrate-standard
  • high-resolution

但是，如果需要，您可以使用攝取配置文件 API或使用視頻雲工作室 .

管理視訊

這內容管理系統 API允許您取回帳戶的視頻數據。（如上所示，作為視頻攝取過程的一部分，它也用於創建視頻對象。）最基本的請求如下：

    https://cms.api.brightcove.com/v1/accounts/account_id/videos

默認情況下，此請求返回一個JSON包含大量元數據的 20 個視頻對像數組，包括名稱、描述、標籤、自定義字段、創建和上次修改日期、海報和縮略圖的 URL 等等。

您可以通過向請求添加以下一個或多個參數來優化請求的結果：

limit
這決定了要返回的視頻對象的數量，可以設置為 100 以內的任意數字 - 默認為 20
offset
這決定了要跳過的項目數，因此與limit翻閱視頻目錄 - 默認為 0
sort
這決定了對結果進行排序的視頻元數據字段 - 默認情況下，結果的排序方式updated_at（降序，首先顯示最近更新的視頻）

看內容管理系統 API概述 - 參數有關這些參數的詳細信息。

搜索視頻

您還可以使用q範圍。您可以按名稱、描述和標籤等特定字段以及視頻的日期和狀態進行搜索：

    https://cms.api.brightcove.com/v1/accounts/account_id/videos?q=tags:sea,mammal

有關詳細信息和所有搜索選項，請參見搜索視頻。

獲取和更新特定視頻

要通過其 id 或參考 id 檢索特定視頻：

    https://cms.api.brightcove.com/v1/accounts/account_id/videos/id
    or
    https://cms.api.brightcove.com/v1/accounts/account_id/videos/ref:reference_id

一種得到請求返回視頻對象。要更新它，請修改JSON並使用返回它修補請求到相同的 URL。

在向用戶顯示視頻時，最好在顯示中包含縮略圖，以減少選擇錯誤視頻進行發布或更新的機會。

播放清單

播放列表信息也使用內容管理系統 API與視頻信息的方式大致相同。注意視頻雲支持八種類型的播放列表，分為兩類：

手動（或EXPLICIT）播放列表
包含一組指定的視頻-最多可以包含100個視頻
智能播放列表
根據搜索條件在運行時動態生成-共有七種智能播放列表，它們與列表中視頻的排序方式相對應：
  • ACTIVATEDOLDESTTONEWEST
  • ACTIVATEDNEWESTTOOLDEST
  • ALPHABETICAL
  • PLAYSTOTAL
  • PLAYSTRAILINGWEEK
  • STARTDATEOLDESTTONEWEST
  • STARTDATENEWESTTO_OLDEST
請注意，您可以不是創建播放列表後，將手動播放列表更改為智能播放列表（反之亦然）。

視頻數量限制可以設置為最多 100 個。

與視頻一樣，您可以使用limit和offset如果該帳戶包含大量的播放列表，則可以分頁瀏覽結果：

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists

返回的播放列表對像數組將包括播放列表的元數據，包括type對應於上述類型之一。如果類型是EXPLICIT，還會有一個video_ids包含所包含視頻的ID的數組。如果類型是智能播放列表類型之一，則將有一個search包含用於檢索視頻的搜索字符串的屬性，如下所示：

    q=tags:fish,birds

您還可以通過其檢索單個播放列表id：

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id

如果您需要檢索播放列表的完整視頻對象（以在頁面上顯示有關視頻的信息），只需添加/videos到該URL：

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id/videos

請注意，對於智能播放列表，請求將返回當前符合搜索條件但可能會更改的視頻。

創建玩家

Brightcove 播放器可以通過播放器管理 API .該 API 允許您創建玩家、更新他們的屬性並以 URL 的形式獲取嵌入代碼，iframe標記，或要嵌入頁面的 HTML 塊。

每個帳戶最多可以有 200 名玩家，但根據您的絕對需要擁有盡可能少的玩家通常不會讓用戶感到困惑。你應該有單獨的播放器來播放單個視頻或播放列表，否則你只需要不同的播放器，當它們的樣式不同或通過插件添加不同的功能時。

要創建播放器，您只需創建一個郵政向播放器管理 API：

    https://players.api.brightcove.com/v2/accounts/account_id/players

在請求的正文中，包括播放器配置 -唯一需要的是name：

    {
    "name": "Single video player for blog posts"
    }

響應將為您提供玩家 ID，以及多種形式的嵌入代碼：

    {
    "embed_code": "<iframe src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "embed_in_page": "http://players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/in_page.embed",
    "id": "de055fa4-4f09-45af-8531-419c6794ad04",
    "preview_embed_code": "<iframe src='//preview-players.brightcove.net/v1/accounts/57838016001/players/de055fa4-4f09-45af-8531-419c6794ad04/preview/embeds/default/master/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "preview_url": "http://preview-players.brightcove.net/v1/accounts/57838016001/players/de055fa4-4f09-45af-8531-419c6794ad04/preview/embeds/default/master/index.html",
    "url": "http://players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html"
    }

要獲得完整的播放器配置，您可以向/players端點，但添加在上述響應中返回的玩家 ID：

    https://players.api.brightcove.com/v2/accounts/account_id/players/de055fa4-4f09-45af-8531-419c6794ad04

你可以做一個修補請求同一端點更新播放器配置。

您會在上面的回復中注意到，preview_embed_code和preview_url .為了允許測試新玩家或玩家更新，新創建或更新的玩家設置為預覽模式，以便您在將更改推送到現有玩家之前查看它。要將更改推送到生產中，您需要發布有此請求的玩家：

    https://players.api.brightcove.com/v2/accounts/account_id/players/de055fa4-4f09-45af-8531-419c6794ad04/publish

定制玩家

這Brightcove 播放器使用標準網絡技術構建：HTML、CSS 和JavaScript .您可以使用這些相同的技術自定義播放器。這可以在發布播放器的頁面中完成，但最佳做法是通過播放器將您的自定義添加到播放器本身配置 , 通過一個更新播放器修補向播放器管理 API如上一節所述。

您還可以通過以下方式向播放器添加其他特性和功能JavaScript插件 , 並且有一個廣泛的播放器接口幫助您將代碼與播放器集成。Brightcove提供了許多現成的插件來實現諸如啟用廣告，自定義最終屏幕以及添加覆蓋圖之類的功能。

發布視頻

在裡面創建玩家部分上面我們看到，當你使用播放器管理 API，返回的數據包括用於將播放器嵌入 HTML 頁面的 iframe 標記，如果您想將播放器直接嵌入頁面，還包括完整 HTML 的 URL。

我們建議您盡可能使用 iframe 嵌入，因為這將防止播放器的 CSS 和JavaScript以及頁面中使用的其他樣式表和腳本。看這個文件有關差異的詳細信息以及關於在不同情況下使用哪些嵌入的更具體建議。

無論您選擇哪種嵌入，您都需要添加一個視頻雲將視頻 id 或播放列表 id 添加到嵌入代碼以向播放器添加內容。iframe 嵌入代碼如下所示：

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html'
    allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

到播放器的 URL，您需要附加參數videoId={}video_id，以便完整的嵌入代碼如下所示：

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html?videoId=4483119716001'
    allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

如果這是播放列表播放器，則使用參數playlistId={playlist_id}反而。頁內嵌入代碼的修改類似。

除非在播放器配置中固定播放器尺寸，否則您還需要通過添加寬度和高度來調整播放器的大小style屬性：

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html?videoId=4483119716001'
    allowfullscreen webkitallowfullscreen mozallowfullscreen
    style=width:640px;height:360px;></iframe>

API 摘要

以下是可用於集成的 API 摘要視頻雲 .

OAuth API
用於創建客戶端憑據和訪問令牌以訪問其他 API。
媒體管理

用於創建自定義攝取配置文件，指定要為添加到的視頻創建的再現視頻雲
動態攝取 API
用於將視頻和相關媒體資產添加到視頻雲
內容管理系統 API
用於創建用於攝取的視頻對象，以及管理視頻和播放列表
Brightcove玩家
Brightcove 播放器
播放器包括一個JavaScript在運行時與播放器交互的 API
播放器管理 API
用於創建和配置播放器，以及獲取播放器嵌入代碼
分析 API
用於獲取有關視頻性能的分析報告
