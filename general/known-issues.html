分析 API

在fields而不要求它們作為尺寸

如果您使用video 或者player 字段（例如video ，video_name ，player 或者player_name ）返回的字段中，但不要將其包括在dimensions ，響應項目將包括與這些項目無關的值，應將其忽略。項目中的其他值將有效

解決方法：不要在響應字段中包含視頻或播放器字段，除非您也將它們包含為dimensions 。

數量眾多的“其他"結果device_manufacturer
在某些情況下，您可能會看到大量的“其他"結果device_manufacturer場地。當數據收集器無法識別user_agent與分析數據一起交付給它。我們為此確定的最常見原因是 Instagram 創建的自定義 user_agent 字符串，但可能還有其他原因。

內容管理系統 API

在某些情況下，original_filename 損壞
在某些情況下，original_filename 視頻的字段已損壞，將不包含真實的文件名。
標籤不能包含逗號
如果您嘗試將標籤添加到包含逗號的視頻中（例如“SomeCo，Inc"），請求將失敗，並顯示“非法標籤"錯誤
預定視頻
由於Playback API和商品目錄最多可緩存視頻10-15分鐘，因此要求播放器計劃在接下來的幾分鐘（最多20分鐘）內可用的視頻的播放器可能無法獲取可播放的視頻，直到刷新緩存為止。
has_digital_master

如果您使用此字段測試視頻是否可以重新轉碼，則對於共享視頻來說是不可靠的，因為如果原始視頻具有主視頻，則將是正確的，但附屬帳戶仍無法重新轉碼。

解決方法：如果視頻已共享到該帳戶，則需要同時測試has_digital_master = true 和sharing.by_external_account = false 。示例代碼（JavaScript）：

var video={some_video_object}
if (video.sharing !== null) {
  if (video.sharing.by_external_account === false) {
    if (video.has_digital_master) {
    // video can be retranscoded
    }
  }
}
使用更新/ PATCH時，播放列表允許使用重複的參考ID
當您更新播放列表時reference_id使用內容管理系統 API（或 Media 模塊，因為它使用相同的 API 請求），一個已經用於另一個播放列表的引用 ID 將被允許。
