介紹

這個應用程序創建了一個播客iTunes 的 RSS 提要來自您的 Video Cloud 帳戶中的一個或多個純音頻標題。

提供此示例和相關代碼作為生產開發的指南。該代碼未經認證可用於生產用途，也未經官方支持。

注意：如果您輸入所有必填信息，但未生成Feed，或者其中不包含視頻，請參見故障排除下面的部分。

使用此應用程序需要什麼：

  • 搜索中返回的所有視頻都將包含在供稿中。因此，您需要使用標籤或自定義字段來準確標識要包含的視頻。
  • 提要中的網址必須指向 MP4 格式。如果您選擇的視頻沒有MP4再現，則該應用程序將不是將它們包括在提要中。
  • 確保您滿意蘋果的要求關於圖像和其他東西。此應用程序將使用海報圖像作為播客藝術作品。您可能需要創建自定義攝取配置文件使您的海報尺寸符合圖像要求。
  • 另請參閱播客最佳做法（此應用將在輸入允許的範圍內跟踪它們）。
  • 您的播客作為純音頻內容添加到視頻雲中 - 請參閱音頻內容有關如何執行此操作的詳細信息。
  • 播客的標題。如果沒有標題，則name視頻（或一組情節中的第一個視頻）中的視頻將被使用。
  • 播客的描述。
  • 您的站點 URL（例如 https://yoursiteurl.com）
  • 將託管此提要的 URL（例如 https://yoursiteurl.com/feeds/podcast.rss）
  • 作者（個人或組織名稱）。
  • 播客的聯繫電子郵件地址
  • 廣播摘要（如果您忽略此內容，我們將使用description來自視頻雲）。

下面的應用程序包含您需要提供的所有信息的輸入字段。要查看應用程序如何將您的數據和視頻元數據映射到播客提要字段，請參閱字段映射頁面底部的部分。

URL標記化

除了上述要求之外，您還需要修改帳戶的URL標記。視頻雲增加了一個ttl用於再現的 URL 的令牌，以防止您的內容被盜。默認情況下，這些令牌的壽命很短-對於播客來說太短了。你需要聯繫 Brightcove 支持讓他們將令牌的生存時間值設置得更高。允許的範圍是一小時到一年。要求他們將值設置為適合您希望播客可用的時間。請記住，這是一個帳戶設置。請勿將值設置得過高，以免損害其他視頻的安全性。

有關該應用程序工作方式的註釋

  • 填寫供稿所需的信息後，您可以從您的帳戶中提取（最多20個）視頻
  • 如果您為播客視頻命名，標記或具有自定義字段值以使其易於查找，則將更加容易
  • 然後，您為提要選擇一個或多個視頻
  • 如果您選擇多個視頻，則該應用會假定這些是播客集，並且視頻的激活日期反映了這些集的順序

故障排除

如果您在使用此應用程序時遇到問題，請檢查以下事項：

  • 如果生成的提要中不包含視頻，請確保要包含的視頻具有MP4格式。您可以使用 CMS API 進行檢查，但最簡單的方法是查看 Studio Media 模塊中的視頻文件列表，並確保至少可以下載一個節目：

    如果無法下載任何格式的摘要，則需要使用包含MP4（逐行）格式的摘要配置文件替換或重新轉換視頻。

  • 如果您的視頻確實具有MP4格式，但是您的供稿中仍然不包含視頻，請檢查以確保您使用的搜索字符串返回了正確的視頻。一種方法是在下面複製請求網址，然後替換{account_id}和{search_string}使用您的帳戶ID和搜索字符串，然後在我們的網站中嘗試該請求API測試儀確保返回您期望的視頻：

            https://cms.api.brightcove.com/v1/accounts/{account_id}/videos?q={search_string}
