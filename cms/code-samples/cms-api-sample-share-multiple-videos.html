介紹

在此示例中，您將學習如何使用CMS API將多個視頻從一個主帳戶共享到一個或多個附屬帳戶。

提供此示例和相關代碼作為生產開發的指南。該代碼未經認證可用於生產用途，也未經官方支持。

局限性

  1. 對共享視頻的 API 請求的響應將包括全部視頻的共享，而不僅僅是請求中指明的關聯公司。當您查看響應時，請注意這一點。
  2. 這個應用程序不是檢查關聯公司是否已批准共享合同。否則，共享將失敗。
  3. 這個應用程序不是檢查選定的視頻是否已與選定的附屬機構共享。如果您轉載已經與會員共享的視頻，則該會員所做的任何元數據更改都將被覆蓋。
  4. 這個應用程序不是檢查共享錯誤。由於主帳戶和附屬帳戶之間的自定義字段不匹配或其他原因，共享可能會失敗。查看對視頻共享請求的響應，看是否有任何錯誤。

可以通過向您自己的此應用程序版本添加附加代碼來消除上述最後三個限制。我們沒有添加代碼是為了使示例應用程序邏輯盡可能簡單易懂。

使用此應用程序的要求

為了使用此應用程序，您將需要準備一些先決條件：

  1. 這掌握必須為媒體共享啟用帳戶 - 如果沒有，請聯繫您的客戶經理或 Brightcove 支持
  2. 您必須擁有啟用媒體共享操作的主帳戶的客戶端憑據（請參閱獲取憑據以下部分）
  3. 你必須有一個default頻道（當您的帳戶啟用媒體共享時會自動設置此頻道）
  4. 你必須有一個或多個附屬公司添加到頻道 - 如果你還沒有這樣做，你可以在工作室或者通過使用這個用於添加多個附屬公司的配套應用程序 .
