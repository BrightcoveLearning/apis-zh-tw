介紹

在本主題中，您將學習如何執行以下操作：

  • 使用q範圍
  • 對搜索結果進行排序
  • 使用必需和排除的術語進行搜索
  • 使用帶引號的搜索來匹配精確的術語和多個單詞
  • 搜索自定義字段
  • 搜索具有特定日期和範圍的日期字段
  • 組合搜索條件
請注意，本主題涵蓋視頻搜索 - 另請參閱播放列表搜索 .

接口使用

搜索功能可用於內容管理系統 API或播放 API。

任一 API 的搜索字符串必須至少包含 3 個字符。

內容管理系統 API

將搜索與 CMS API 結合使用時，以下內容適用：

  • 所有請求（包括搜索）都需要一個包含您的訪問令牌的授權標頭。有關如何獲取客戶端憑據並使用它們來檢索訪問令牌的詳細信息，請參閱Brightcove OAuth 概述 .
  • 從搜索返回的最大視頻數沒有限制，但速率限制確實適用。
  • 搜索結果包括您帳戶中的所有視頻，無論它們是否可播放，和/或受地域限制。

有關 API 請求/響應的詳細信息，請參閱獲取視頻的部分CMS API 參考 .

播放接口

在 Playback API 中使用搜索時，以下內容適用：

  • 使用 Playback API 的搜索請求需要一個 Policy Key，它是啟用搜索 .
  • 有一個限制搜索返回的最大視頻數。
  • 搜索結果只會返回可播放的視頻（帶有state:inactive將被忽略）。
  • 搜索強制執行播放策略限制，例如從結果中省略受地理限制的視頻。
  • 結果緩存提供了更高的請求率和更快的響應，並且沒有速率限制。
Playback API 搜索限制為每個請求 100 個視頻，上限為 1000 個視頻。了解如何翻頁搜索結果 .當您需要返回超過 1000 個視頻時，您必須使用 CMS API。

有關 API 請求/響應的詳細信息，請參閱獲取視頻的部分播放 API 參考 .

基本搜索

要在媒體庫中搜索術語，請使用q範圍。

      q={search terms}

您指定的搜索詞應該是用空格分隔的 url 編碼的詞列表。

搜索支持 詞乾化.詞乾化是將一個詞映射到詞根和其他來自同一詞根的詞。例如，對“run”的搜索應該匹配在指定字段中包含“running”或“ran”的視頻。它會不是匹配“rune”，因為“rune”沒有“run”作為它的根。

當您沒有為搜索詞提供限定詞時，例如q=bird，請求將在以下字段中搜索該值：

  • id [1-1]
  • name
  • description
  • long_description
  • text（不是真正的元數據字段，而是可用於搜索name , description， 和long_description - 例如q=text:bird )
  • tags
  • reference_id
  • custom_fields（搜索所有自定義字段）
  • custom_field_name（搜索特定命名的自定義字段）[1-2]

筆記

[1-1] 注意：為了一致性，包括按 id 搜索，但搜索q=id:12345將返回與請求完全相同的結果https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/12345
[1-2] 如果您有一個列表類型的自定義字段並希望返回具有多個值之一的視頻，您可以這樣做：

假設您有一個名為的字段color可以採用以下值：red , green , yellow， 或者blue .您想查找將該字段設置為該值的視頻green或者blue：

      q=color:green%20color:blue

例子：此請求返回值為bird至少在上面列出的領域之一。

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=bird
少於 3 個字符的搜索字符串將不起作用 - 相反，將返回所有視頻。

將搜索限制為特定屬性

當您為搜索詞提供限定詞時，例如q=name:bird，請求將搜索視頻name字段的值bird .

例子：此請求返回值為wildlife在裡面name場地。

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=name:wildlife

支持的搜索字段是：

支持的搜索字段
場地 法律價值
name 字符串或帶引號的字符串
文本 字符串或帶引號的字符串（搜索name , description， 和long_description )
tags 字符串或帶引號的字符串（多個標籤應以逗號分隔）
custom_fields 字符串或帶引號的字符串（搜索所有自定義字段 - 您也可以使用特定的自定義字段 內部的 姓名） [2-1]
reference_id 字符串或帶引號的字符串
state ACTIVE , INACTIVE , PENDING , DELETED [2-3]
updated_at 日期範圍
created_at 日期範圍
schedule.starts_at 日期範圍
schedule.ends_at 日期範圍
published_at 日期範圍
complete [2-2] true或者false

筆記

  • [2-1] 這是不是可以搜索具有沒有值或值的自定義字段的視頻null，因為除非該字段已被賦予值，否則它根本不包含在視頻元數據中。
  • [2-2] 當您創建新視頻時，complete屬性自動設置為false .一旦存在視頻的一種再現，complete屬性將自動設置為true .
  • [2-3] 以下限制適用於搜索已刪除的視頻：
    • 搜索已刪除的視頻是只要使用 CMS API 可用；播放 API 將不是返回已刪除的視頻
    • 僅有的將返回前 10 天（當前時間減去 10 天）刪除的視頻
使用字符串值的字段對該字段執行文本搜索。

搜索結果排序

這sort參數允許您對視頻獲取請求的結果進行排序。您可以按以下順序排序：

  • reference_id
  • name
  • created_at
  • published_at
  • updated_at
  • schedule.starts_at
  • schedule.ends_at
  • state
  • plays_total
  • plays_trailing_week

當未通過使用顯式排序結果時sort，結果將根據稱為術語頻率/逆文檔頻率的算法進行排序或TF-IDF .看這裡想要查詢更多的信息。

例如，假設您搜索條件coastal,city並且您的帳戶中有 120 個視頻在視頻元數據的某處包含這些術語（name , description , tags，等等）並且也符合結果的排序標準（例如，它們都具有相同的schedule_starts_at約會時間）。視頻在結果中出現的頻率取決於一個或兩個術語在元數據中出現的頻率，對於整個視頻庫中出現頻率最高的術語給予更大的權重。

必需/排除條款

您可以根據需要標記搜索詞（返回的視頻必須匹配）或排除（返回的視頻必須不匹配）。這是由 URI 編碼的控制+ (%2B)或者-緊接在該術語之前的標誌。

你必須編碼+作為%2B當使用它來表示所需的術語時。

必需/排除條款
例子 網址編碼 意義
+foo %2Bfoo 視頻必須包含術語foo在裡面name , description , long_description , tags , reference_id或者custom_fields
+custom_fields:foo %2Bcustom_fields:foo 視頻必須包含值foo對於一些自定義字段
+foo -bar %2Bfoo%20-bar 視頻必須包含術語foo但不得包含該術語bar在裡面name , description , long_description , tags , reference_id或者custom_fields
+name:foo -name:bar %2Bname:foo%20-name:bar 視頻必須包含術語foo但不得包含該術語bar在裡面name

例子：此請求返回值為sea但沒有價值lake在裡面tags場地。

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=%2Btags:sea%20-tags:lake

看組合搜索條件下面查看如何使用必需/排除的語法來強制執行多個搜索詞的 AND 邏輯。

結合其他參數

搜索（使用q參數）可以與其他參數組合，例如sort , limit和offset .所有 URL 參數都由& .參數順序無關緊要。

搜索參數limit有一個最大值100每個請求。請參見頁面搜索結果關於視頻分頁。

例子

例子：此請求返回必須具有值的視頻bar在裡面tag字段並且可能有一個name包含價值foo

      .../videos?q=name:foo%20%2Btags:bar&sort=updated_at

例子：此請求返回與上述相同的視頻，但另外按字段對這些結果進行排序updated_at然後將結果限制為僅 10 個視頻。

      .../videos?sort=updated_at&q=name:foo%20%2Btags:bar&limit=10

引用的搜索詞

默認情況下，搜索會將相似的詞與您的搜索詞進行匹配。如果要匹配多個單詞，只需將術語括在引號中即可。

大多數瀏覽器和其他代理會處理文字引號 ( "..." ) 正確，但如果您遇到引用的搜索詞似乎沒有返回正確結果的情況，請嘗試將引號替換為%22 ( %22...%22 )

            
              q="foo" or q=%22foo%22
              q="foo%20bar" or q=%22foo%20bar%22
請注意，引號要求匹配視頻中包含所有術語，但不一定完全匹配這些術語。例如，搜索"home run"可能會返回帶有文字的視頻homeless和running在標題中。

這在針對特定字段進行搜索時也有效：

            
              q=name:"home"
              q=name:"home%20run"

多個單詞

例子：請注意，此請求返回具有以下值的視頻sea或者mammal在裡面tags場地。

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=tags:sea,mammal

但是，以下請求僅返回具有標籤的視頻sea,mammal .

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=tags:"sea,mammal"
標籤是一種特殊情況，您可以在其中搜索多個標籤作為逗號分隔的列表 - 這不適用於名稱、描述等其他字段。對於其他類型的多個術語，將術語與+簽名，像這樣：q=tags:"bird"%20name:"owl" .

自定義字段

您可以搜索為視頻定義的任何自定義字段。

      q=my_field:foo
      q=my_field:"foo"

注意：所有自定義字段值都被視為字符串。例如，如果您有一個列表類型的自定義字段，可以採用值true或者false , search 將尋找那些字符串，而不是布爾值（在許多編程語言中，1和0可以與true和false作為布爾值，但搜索q=my_boolean_field:1不會返回具有my_boolean_field設置true）。

例子：此請求返回值為動物在裡面subject自定義字段。

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=subject:animal

日期範圍

如果您在日期字段上進行搜索，您可以指定特定日期或日期範圍，使用兩個句點來分隔開始日期和結束日期 ( q=updated_at:2018-01-01..2018-02-01）。

這將搜索所有帶有updated_at 2012 年 8 月 1 日至 2012 年 10 月 8 日之間的值。在這裡，我們以 UTC 格式指定每個日期。

      q=updated_at:2012-08-01T00:00:00Z..2012-10-08T23:59:59Z

您可以通過刪除時間組件來簡化此搜索。以下相當於上面的搜索。

      q=updated_at:2012-08-01..2012-10-08

支持的日期格式

支持的日期格式包括UTC 和relative。

日期格式示例
格式（URI 編碼格式） 意義
2015-08-01T06:15:00Z 這表示 UTC 時間。
2012-08-01 這表示 UTC 中一天的午夜。該示例相當於 2012-08-01T00:00:00Z
-1天 當前時間減1天。 （看以下 )

相對日期

對於相對日期，我們支持一個方向（+或者- ) 後跟一個數字，後跟一個持續時間。相對日期始終從當前時間開始計算。法定持續時間為：分鐘、小時、天。

這+標誌必須被 URI 編碼為%2B .

例子：

相對日期樣本
q 日期參數 意義
q=updated_at:-1day..NOW 從 1 天前更新到今天的視頻
q=created_at:-2 天 2 天前添加的視頻
q=updated_at:-4hours..NOW 視頻從 4 小時前更新到當前時間
q=created_at:-60 分鐘.. 從 60 分鐘前添加到當前時間的視頻
q=created_at:2016-01-01..-1d 2015 年 1 月 1 日至一天前創建的視頻
q=updated_at:-14d..NOW 過去兩週的視頻

開放範圍

如果您想匹配指定時間之前的所有日期，或匹配指定時間以來的所有日期，請保留範圍的一端。

例子：搜索過去 2 天內修改過的所有視頻

      q=updated_at:-2days..

例子：搜索 2013 年 8 月 11 日或之後修改的所有視頻：

      q=updated_at:2013-08-11T00:00:00Z..

NOW預定日期的運算符

為了schedule.starts_at和schedule.ends_at， 您可以使用NOW作為日期值。這是一個方便的運算符，允許您根據當前日期時間設置動態查詢。幾個例子：

調度數據示例
從/到參數 URI 編碼 意義
?q=schedule.starts_at:..NOW ?q=schedule.starts_at:..NOW starts_at 是從時間開始到這一刻
?q=schedule.starts_at:NOW ?q=schedule.starts_at:NOW starts_at 是從這一刻開始
?q=schedule.ends_at:NOW.. ?q=schedule.ends_at:NOW.. Ends_at 是從這一刻到時間的盡頭
?q=+schedule.starts_at:..NOW +schedule.ends_at:NOW.. ?q=%2Bschedule.starts_at:..NOW%20%2Bschedule.ends_at:NOW.. 在這一刻之前開始_在這一刻之後結束_在這一刻之後（視頻在這一刻排定）

組合搜索條件

您可以組合複雜搜索的條件。

例子：此請求搜索帶有name的價值閒話，其更新時間為 2010 年 8 月 1 日至 2010 年 10 月 8 日。然後它對響應數據進行排序updated_at日期按降序排列。

      q=%2Bname:gossip%20%2Bupdated_at:2010-08-01..2010-10-08&sort=-updated_at

組合詞

使用必需/排除的語法返回具有全部指定的條款。

請注意以下多個搜索詞的所有示例，文本搜索詞後跟附加詞在第一個詞之後有一個 URI 編碼的空格 (%20)，這樣附加詞就不會被解釋為第一個詞的一部分 .

例如，如果您搜索：

      q=name:foo +tags:bar (URI-encoded: q=name:foo%20%2Btags:bar)

響應將包含帶有標籤“bar”的視頻' 並且可能還有foo在名字裡。如果您只想返回那些具有foo在名稱和標籤“bar”中，您必須搜索：

      (unencoded) q=+name:foo +tags:bar (URI-encoded) q=%2Bname:foo%20%2Btags:bar

同樣，如果您只想返回具有foo以名義，但做不是有標籤“bar”，你可以搜索：

      (unencoded) q=+name:foo -tags:bar (encoded) q=%2Bname:foo%20-tags:bar

搜索特定視頻

如果您想將搜索限制為一組特定的視頻，您可以在id：

例子：此請求返回帶有 ID 的視頻123456789 , 987654321和48376387

      q=id:123456789%20id:987654321%20id:48376387

按州搜索

您可以使用以下參數按視頻狀態執行或過濾搜索：

      q=state:ACTIVE( | INACTIVE | PENDING | DELETED)[3]

筆記

  • [3] 搜索已刪除的視頻僅適用於過去 10 天（當前時間減去 10 天）刪除的視頻，並且只能通過內容管理系統 API（不是播放 API）。

詞幹

支持詞幹，但不是部分詞搜索。例如，q=name:ban應該返回名稱為“的視頻”Parking Ban Announced“ 或者 ”Parking to be Banned“ 或者 ”City Banning Parking“ 但不是 ”Bank Holiday“ 或者 ”Bandit Captured”。

空格/特殊字符

這內容管理系統 API通常處理搜索字符串中的特殊字符，但有幾個例外：

  • 不允許使用空格，必須編碼為%20 .（未編碼+符號也可以替換空格，但這可能會導致您的查詢混淆，因為+也可以表示需要一個術語。看必需/排除的語法 )

    例如，要在名稱中搜索“我最喜歡的視頻”：

    q=name:"my%20favorite%20video"

    請注意，引號確保返回的視頻包含所有條款。如果沒有它們，將返回具有三個術語中任何一個的視頻。更多關於下面的內容。
  • 搜索文字+簽署或使用+表示返回的視頻必須包括一個術語，你必須編碼+作為%2B：

    搜索必須包含的視頻"two+two"在名稱字段中

    q=name:two%2Btwo

    搜索必須包含的視頻"heron"在名稱字段中

    q=%2Bname:heron

  • 有些代理可能無法正確處理文字引號，因此編碼更安全"foo"作為%22foo%22
筆記：

對於多字標籤或字符串，將標籤/字符串括在 URI 編碼的引號 ( %22 )，並對空格進行 URI 編碼(%20)：

多字標籤示例
未編碼的搜索字符串 編碼搜索字符串
q=tags:"multi word tag" q=tags:%22multi%20word%20tag%22

引號確保返回的視頻與所有條款匹配，但不一定與確切條款匹配。詞幹仍然適用 - 例如，搜索"new house"還將返回帶有name或者description喜歡news about housing .

對於一次性請求，您可以使用 Brightcove Learning 的字符串編碼器對您的搜索字符串進行編碼。對於應用程序，您需要找到您使用的語言的 URI 編碼函數。

星號 ( * ) 被視為普通的特殊字符，不是通配符。

剪輯搜索詞

剪輯是從其他視頻的部分創建的視頻。剪輯可以通過Brightcove 社交，以及其他方式將在未來可用。您可以使用一些特殊的搜索詞來查找帳戶中生成的剪輯：

  • q=%2Bis_clip:true - 只返回剪輯
  • q=%2Bis_clip:false - 僅返回非剪輯
  • q=%2Bclip_source_video_id:video_id - 返回從指定視頻生成的剪輯

忽略的詞

搜索字符串中的某些詞會被忽略，因為它們非常常見，以至於它們可能會返回許多與您實際搜索的內容無關的結果。以下是搜索忽略的單詞列表：

“a”、“an”、“and”、“are”、“as”、“at”、“be”、“but”、“by”、“for”、“if”、“in”、“into” ", "is", "it", "no", "not", "of", "on", "or", "such", "that", "the", "their", "then", “那裡”、“這些”、“他們”、“這”、“至”、“曾經”、“將”、“與”

已知的問題

  • 重複結果：在某些情況下，搜索結果中的某些項目可能會出現多次。

    解決方法：為防止重複搜索結果，請始終使用sort搜索請求中的參數。
