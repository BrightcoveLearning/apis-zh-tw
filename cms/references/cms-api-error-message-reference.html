一般錯誤訊息

地位 名稱 信息 原因
400 BAD_REQUEST 非法路徑：不支持超過 10 個視頻的請求 基礎的GET /videos/video_ids請求不能請求超過 10 個視頻
400 INVALID_SORT 嘗試按無效屬性排序：“[屬性名稱]” 這sort參數指向無效或不存在的字段
400 INVALID_SEARCH 搜索字符串無效 搜索字符串語法無效，或者您未能對搜索字符串進行 URIEncode
400 ILLEGAL_QUERY 查詢字符串有問題 搜索字符串語法無效 - 示例：1）做一個tags以逗號結尾或引號未結束的搜索
400 BAD_VALUE 提交數據中無法識別的字段 拼寫錯誤或使用不存在的字段
400 AD_CONFIG_NOT_FOUND 未找到廣告配置 找不到SSAI請求中指定的廣告配置
400 AD_CONFIG_INACTIVE 廣告配置無效 SSAI請求中指定的廣告配置無效
400 REFERENCES_EXIST 該視頻至少被一個播放列表引用。 您正在嘗試刪除至少包含在一個播放列表中的視頻。
400 SHARED_VIDEO 刪除共享視頻失敗。 尚不支持刪除共享視頻。
401 UNAUTHORIZED 沒有權限。 缺少或無效的OAuth訪問令牌。如果您確實提供了訪問令牌，則該令牌可能沒有此請求的適當範圍
403 NOT_AVAILABLE 資源不可用。 您請求的資源暫時不可用 - 這可能是對視頻進行某種處理時的臨時情況，但如果消息仍然存在，請聯繫支持人員。
404 NOT_FOUND 資源不存在。 您請求的資源不存在 - 檢查拼寫。
405 METHOD_NOT_ALLOWED 此資源不允許用於請求的 HTTP 方法 最常見的原因是向只讀資源發送 PUT、POST、PATCH 或 DELETE。
406 NOT_ACCEPTABLE HTTP Accept 標頭具有不允許的值。 這Accept標頭必須有值application/json ;其他值，例如 application/x-www-form-urlencoded將導致請求因此錯誤而被拒絕。
409 LIVE_VIDEOS_NOT_SHAREABLE 無法分享此視頻，因為它是直播的。 無法共享實時流式視頻和由它們創建的剪輯。
409 REFERENCE_ID_IN_USE 參考 ID 已在使用中。 您嘗試使用已在使用的參考 ID 創建視頻，或將參考 ID 添加到已被其他視頻使用的視頻。
409 CONCURRENT_UPDATE 由於同時進行了另一次更新，更新失敗。請再試一次。 當前有單獨的API請求，Studio用戶或某些系統進程正在更新視頻。如果您對同一視頻異步、快速連續地發出 API 更新請求，也會發生這種情況。
409 CONFLICT 參考 ID 已在使用中。 為了確保引用 id 的唯一性，涉及引用 id 的操作將鎖定 id 長達 3 分鐘。這意味著，如果您發出創建或更改引用 ID 的請求，然後立即發出使用該引用 ID 的任何其他請求，則操作將失敗並顯示 409 錯誤。這將包括：1）刪除視頻，然後嘗試將其參考ID分配給另一個視頻； 2）嘗試創建失敗並出現503（服務不可用）錯誤的新視頻，然後再次嘗試相同的請求。

收到此錯誤後，請至少等待3分鐘，然後重試該操作。另請注意，如果您嘗試使用參考ID創建視頻，但嘗試失敗（由於與參考ID無關的原因），該參考ID將被鎖定30秒鐘，並且重複創建視頻的嘗試將失敗，直到您等待足夠長的時間以使參考ID解鎖。
409 CONFLICT 必須為媒體共享啟用兩個帳戶。 當主賬戶嘗試將附屬賬戶添加到頻道進行媒體共享時， 兩個都 必須為媒體共享啟用帳戶。
409 UNSUPPORTED_MEDIA_TYPE   該請求很可能缺少標頭：Content-Type: application/json .
415 SHARING_DISABLED 主帳戶未啟用媒體共享。 如果您嘗試從未啟用媒體共享的帳戶共享視頻，則會返回此錯誤。
422 ILLEGAL_FIELD 提交數據中無法識別的字段 拼寫錯誤或使用不存在的字段
422 VALIDATION_ERROR （JSON數據無效-錯誤消息因問題而異） 消息示例：
  • 名稱：REQUIRED_FIELD（創建一個沒有視頻名稱的視頻，將名稱更新為空字符串）
  • 帳戶ID：錯誤的帳戶 （URL 中的帳戶 ID 與 JSON 中的帳戶 ID 不匹配）
  • 無效的鍵：ID（ID不能修改，並且不應在JSON中顯示為鍵）
429 TOO_MANY_REQUESTS 請求過多 您同時提交過多請求或每秒請求過多
500 UNKNOWN 發生未知的內部錯誤 Brightcove系統中的問題-請稍後再試。
503 SERVICE_UNAVAILABLE API 暫時不可用 後端問題-請稍後再試。
504 TIMEOUT 某事花了太長時間 服務器可能太忙 - 稍後再試。

媒體共享錯誤

媒體共享錯誤不會作為對API請求的單獨錯誤響應返回，而是在error_message正常響應中的字段：

  [
  {
  "video_id" : "394872349182374",
  "affiliate_id" : "234987239487",
  "affiliate_video_id" : "30308254055202",
  "status" : "COMPLETE",
  "shared_at" : "2017-12-11T17:57:45.530Z",
  "updated_at" : "2017-12-11T18:03:32.789Z",
  "error_message" : "[{"error_code":"MISSING_CUSTOM_FIELDS","error_message":"Affiliate account is missing custom fields: [whisky]"}]"
  }
  ]

這error_message json 將是一個錯誤對像數組，對象將具有error_code和error_message .

error_code

錯誤代碼通常是特定於共享的，但它也可以是可以由視頻的 PATCH（更新）操作返回的任何有效錯誤代碼。共享特定錯誤代碼如下表所示。

媒體共享錯誤代碼
error_code 錯誤信息 原因
MISSING_CUSTOM_FIELDS 附屬帳戶缺少自定義字段：[field1, field2] 師傅已經定了enforce_custom_fields到true，但正在共享的視頻引用了聯盟帳戶中不存在的字段。
CONFLICT 附屬帳戶未配置地理限制。 主服務器已將 enforce_geo 設置為 true 並且他們的帳戶配置為地理過濾，但附屬帳戶未配置為地理過濾。
SHARING_DISABLED 附屬帳戶未啟用媒體共享。 主人已嘗試與未啟用媒體共享的附屬機構共享視頻。
NOT_FOUND 資源不存在。 有些東西不見了。通常這意味著主視頻消失了。如果在共享請求之後立即刪除主視頻，就會發生這種情況。這也可能意味著主帳戶還沒有頻道，或者如果附屬帳戶不存在或不是頻道的有效成員。如果遷移過程中出現問題，可能會發生這種情況內容管理系統 API媒體共享。
UNKNOWN 共享失敗，出現未知錯誤，error_id INSERT-UNIQUE-ERROR-ID 發生了意想不到的事情。error_message 將包含一個錯誤 id，devops 可以使用它來識別根本原因。重試共享嘗試將在許多情況下起作用。
VALIDATION_ERROR {field}：ILLEGAL_VALUE 如果{field}是“經濟學”，那麼問題是主視頻是“AD SUPPORTED”，但附屬帳戶沒有啟用廣告。如果在禁用廣告帳戶的情況下將“廣告"設置為視頻元數據（非共享視頻的）對“廣告"的支持，也會發生這種情況。如果{field}是別的，那就需要調查了。
REFERENCE_ID_IN_USE 參考 ID 無論您使用的是什麼，都已在使用中。 參考 ID 在主賬戶中有效，但已被附屬賬戶使用。
