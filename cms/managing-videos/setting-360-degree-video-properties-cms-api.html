概述

要將視頻識別為 360° 視頻，需要設置投影屬性。您可以設置此屬性在 Studio 中使用媒體模塊，或者您可以按照本主題所示以編程方式執行此操作。

此示例使用 Postman 應用程序通過 CMS API 更新視頻屬性。了解如何開始使用使用 Postman 處理 HTTP 請求文檔。

在本主題中，您將執行以下任務：

  • 獲取客戶憑證
  • 檢索視頻信息
  • 設置 360° 視頻屬性

獲取客戶憑證

要使用 Brightcove API，您將需要要使用的帳戶和 API 的客戶端憑據。

  1. 按照以下說明獲取Studio中的客戶憑據管理API身份驗證憑據。在下面的步驟中，我們將使用Postman發出Player Management API請求，因此您的憑據至少應具有以下權限：

    • 內容管理系統：視頻讀/寫

    您可以根據需要添加任意數量的額外權限，以獲得可用於更廣泛 API 請求的憑據。還要注意，如果您願意，您會獲得適用於多個帳戶的憑據。

    為了生成客戶端憑據，您必須具有 Studio 的管理員權限，或者讓組織中具有管理員權限的人員為您生成憑據。

檢索視頻信息

將視頻上傳到Video Cloud Studio之後，就可以從CMS API檢索視頻信息了。有關上傳視頻的信息，請參閱使用上傳模塊上傳視頻文檔。

  1. 打開郵遞員應用程序。
  2. 為您的視頻雲帳戶獲取新的訪問令牌，然後選擇使用令牌按鈕。
  3. 做一個得到請求到CMS API，使用通過ID獲取視頻句法。您將通過將以下 URL 輸入到輸入請求網址字段，用您的帳戶 ID 代替{account_id}以及您的視頻 ID {video_id}： 
        https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}
  4. 將方法保留為GET，然後選擇發送：
  5. 在響應正文中，您應該看到投影財產。

設置 360° 視頻屬性

要指示視頻是 360° 視頻，投影屬性應設置為等距長方形 .

  1. 返回郵遞員應用程序。您可能需要獲取新的訪問令牌。
  2. 選擇修補作為 HTTP 方法，使用與上一節中相同的 URL 語法。
  3. 點擊標頭按鈕，並提供核心價值一對內容類型-application / json。只需開始在輸入區域中輸入，您就會獲得一些選項，您可以從中選擇指定的值。
  4. 點擊身體部分位於網址下方，然後生的單選按鈕：
  5. 對於原始數據，請為主體輸入以下JSON代碼（JSON後的屏幕截圖顯示了請求的顯示方式）： 
        {
        "projection" : "equirectangular"
    }
  6. 點擊發送
  7. 在響應正文中，您現在應該看到投影屬性已設置。

    您現在可以將視頻作為 360° 視頻播放了。
