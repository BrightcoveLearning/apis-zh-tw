介紹

您可能希望在一段時間後才能播放視頻的原因有很多。例如：

  • 還在審核中
  • 這是尚未發布的電影預告片
  • 它是尚未開始或結束的營銷活動的一部分
  • 這是尚未發表的新聞報導的一部分

如果您尚未將視頻嵌入代碼放入公共頁面或應用程序中，則當然無法查看，但是有一些方法可能會意外發布視頻，最明顯的是其標籤或其他屬性適合智能已發布的播放列表。

有兩種方法可以確保無法查看視頻雲視頻：

  • 停用它
  • 安排它在未來的某個時間播放

這兩項更改都可以通過 Studio 中的 Media 模塊實現，但在這裡我們將介紹如何使用內容管理系統 API .這兩個操作都是使用一個視頻更新請求 .

注意：激活/停用和調度機制是完全獨立的，因此您應該根據您的用例使用其中一種，但不能同時使用兩者。如果視頻可根據日程，但它的狀態是不活動，視頻將無法播放。

驗證

向內容管理系統 API需要包含您的訪問令牌的授權標頭。有關如何獲取客戶端憑據並使用它們來檢索訪問令牌的詳細信息，請參閱Brightcove OAuth 概述 .

您可以使用工作室管理頁面或者OAuth API

您還可以使用OAuth API直接檢索訪問令牌，或者如果您正在使用郵差或者失眠應用程序或提出您的請求，請參閱我們的教程，了解如何設置這些以獲取訪問令牌：

  • 使用 Insomnia 進行 API 請求
  • 使用 Postman 進行 API 請求

更新視頻請求

要更新視頻，請發送修補請求

https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id

通常，請求正文只需要包含您想要更新的那些字段，但您也可以包含其他字段以及當前值。我們將在接下來的部分中查看激活/停用和調度的請求主體。

激活/停用視頻

通過設置視頻可以使視頻處於活動狀態（可見）或非活動狀態（不可見）狀態 .默認情況下，視頻是積極的一旦它們被攝取（除非在創建視頻或使用上傳模塊上傳時省略了必填字段）。

請注意，該視頻仍可在 Studio Media 模塊視頻預覽中播放（如果您在查看視頻時停用該視頻，這會很方便）。

要停用視頻，您只需發送以下請求正文和更新視頻請求：

{
  "state": "INACTIVE"
}

您可以通過檢查視頻來驗證視頻現在處於非活動狀態狀態 API 響應中的字段

  "state": "ACTIVE"

安排視頻

使視頻可播放或不可播放的第二種方法是使用日程視頻更新請求中的對象。這schedule 對像有兩個屬性：

  • 開始於視頻可播放時的 ISO-8601 日期時間（必需）
  • 結束時視頻不再可播放時的 ISO-8601 日期時間（可選）

這是一個示例請求正文，它使視頻可在 2021 年 5 月 16 日上午 6:00 (UTC) 播放，並使其在整整一年後再次無法播放：

{
  "schedule": {
		"starts_at": "2021-05-16T06:00:00.000Z",
		"ends_at": "2022-05-16T06:00:00.000Z"
	}
}
