概述

遠程資產主要是通過資產運作，以下有一些例外。請注意，資產的GET操作可用於提取的資產和遠程資產。

授權

向內容管理系統 API需要包含您的訪問令牌的授權標頭。有關如何獲取客戶端憑據並使用它們來檢索訪問令牌的詳細信息，請參閱Brightcove OAuth 概述 .

video 權限包括對應的權限asset 操作-即/video/all 授予您所有資產操作的權限，/video/read 授予您資產讀取操作等權限。

工作流程

以下是設置遠程資產的步驟：

在以下各節中，我們將研究每個步驟。

對於對資產端點的所有請求，您可以指定正在操作的視頻ID或者reference_id：

      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/...
      or
      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/ref:{reference_id}/assets/...

創建視頻

在添加遠程資產之前，必須使用POST請求在Video Cloud媒體庫中創建一個視頻對象，以執行以下操作：

      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos

在請求正文中，包括一個JSON對象，該對象至少包含視頻名稱，但可能包含其他元數據：

      {
        "name": "My Remote Video",
        "description": "A short description of the video",
        "tags": ["remote", "fall_campaign"],
        "reference_id": "myUniqueReferenceID"
      }

有關創建視頻操作的詳細信息，請參見API參考。

添加轉場

接下來，通過向以下地點發出POST請求來添加一個或多個全長遠程視頻副本：

      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/renditions

請求正文中的 JSON 必須至少包含遠程 url，但可能包含其他元數據用於再現：

      {
        "video_duration": 31487,
        "name": "RA - Great Blue Heron",
        "remote_url": "//learning-services-media.brightcove.com/videos/mp4/greatblueheron.mp4",
        "encoding_rate": 983000,
        "frame_height": 360,
        "size": 3887659,
        "frame_width": 640,
        "video_codec": "h264",
        "video_container": "MP4",
        "progressive_download": true
      }

筆記

  1. 僅將此端點用於添加全長視頻再現（通常為 .mp4 或 .mov 文件） - 若要添加分段流（如 HLS 或 DASH），請使用下一節中描述的清單端點。

  2. Video Cloud 不會檢查遠程視頻文件，因此提供額外的元數據將使 Video Cloud 能夠在 Studio（或 Brightcove 播放器，如果您將播放器配置為顯示元數據）中顯示此信息。

    有關添加再現操作的詳細信息，請參見API參考。

  3. 遠程資產的複制必須具有video_codec 和video_container 字段才能正常工作。
  4. MP4格式必須有"progressive_download": true 字段設置為在Brightcove播放器中正常工作
  5. 不允許使用混合資產-您無法向提取的視頻添加遠程演繹
  6. 如果您的CDN或其他遠程存儲庫同時支持HTTP和HTTPS傳遞，則請使用與協議無關的網址如上例所示（即使用“//my.domain/path / ..."而不是“http：//my.domain/path / ..."）

添加清單

請注意，要添加遠程分段類型（例如DASH和HLS再現），只需創建視頻，然後添加清單-您可以不是如上一節所述，需要“添加演繹"。

除了添加完整格式，您還可以通過向清單類型的端點發出POST請求，為分段格式（例如HLS，HDS和ISM）添加清單。

      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/hls_manifest
      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/hds_manifest
      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/ism_manifest
      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/ismc_manifest
      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/dash_manifests

在請求正文JSON中，提供遠程URL和名稱（可選）：

      {
        "remote_url": "http://learning-services-media.brightcove.com/avideo/hls/chickadee/chickadee.m3u8",
        "name": "chickadee_hls"
      }
您可以為每種類型添加一個清單。要播放分段類型，您必須為 Brightcove 播放器安裝相應的插件，某些瀏覽器可能需要額外的瀏覽器插件。

有關使用添加清單操作的詳細信息，請參閱API參考的相關部分：

  • 添加HLS清單
  • 添加HDS清單
  • 添加ISM清單
  • 添加ISMC清單
  • 添加DASH清單

添加海報/縮略圖

您可以通過向這些端點發出 POST 請求，為您的視頻添加遠程海報和縮略圖：

      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/poster
      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/thumbnail

在請求正文JSON中，必須包含圖像的遠程URL，並且可以根據需要添加名稱：

      {
        "remote_url": "http://learning-services-media.brightcove.com/images/great-blue-heron-poster.png"
      }

有關操作的詳細信息，請參閱《 API參考》：

  • 添加海報
  • 添加縮略圖

添加文字軌道

文本軌道可用於向視頻添加字幕、章節和其他基於時間的元數據。

與再現和圖像不同，遠程資產視頻的文本軌道與攝取到 Video Cloud 系統的視頻的處理方式相同。要將文本軌道實際添加到視頻中，您必須使用 Studio 中的媒體模塊或內容管理系統 API更新視頻操作 .

Brightcove Player 支持文本軌道（WebVTT 文件）。要為舊智能播放器添加字幕，請參閱添加字幕 .

用攝取的代替遙控器

您可以通過向 Dynamic Ingest API 發出請求，用攝取的視頻替換遠程資產視頻：

      https://ingest.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/ingest-requests

最簡單的請求正文如下所示：

      {
        "master": {
          "url": "http://host/master.mp4"
        }
      }

見動態攝取 API 參考其他選項和信息。

遠程資產視頻的常見用途之一是創建用於現場直播事件的佔位符視頻。事件完成後，您可能希望用事件的視頻點播錄製代替“實時"視頻，該視頻出現在用於實時流的同一播放器中。您可以使用上述方法執行此操作，但它也可用於將任何遠程資產視頻轉換為被攝取到 Video Cloud 中的視頻。

修改資產

除了為視頻添加遠程資產外，還存在用於檢索，更新和刪除資產信息的操作。這些操作使用上述端點，但使用 GET、PATCH 或 DELETE 請求。對於更新和刪除請求，必須將資產ID（包含在POST和GET請求的響應中）添加到請求URL：

      https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/assets/{asset_type}/{asset_id}

見API 參考有關獲取、更新和刪除操作的詳細信息。

局限性

  • 遠程資產不支持 SSAI。
  • 僅當您可以自定義播放器以與DRM許可證服務器通信時，才支持DRM。Brightcove Player 可以通過插件實現這一點。對於其他播放器，您需要查閱播放器文檔。
  • 分析：Brightcove Analytics系統依賴於系統生成的視頻元數據，例如視頻時長。除非您可以為遠程資產提供此元數據，否則遠程資產的Analytics（分析）數據將不太豐富。
  • 播放保證：提取到Video Cloud中的所有視頻都可以在Brightcove Player中播放。幾乎所有遠程資產都是如此，但在某些極端情況下，視頻可能無法播放。例如，我們尚不支持 DASH 和 HLS 的一些高級功能。對於非Brightcove播放器，您需要驗證該播放器完全支持視頻格式。
