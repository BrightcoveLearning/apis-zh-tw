免責聲明

此示例應用程序旨在讓您探索 Player Management API 的基礎知識。它並不是要成為可用於企業的，可部署的應用程序。例如，表單驗證不是詳盡無遺的，並且密碼不是使用HTTPS發送的。

要了解有關此示例應用程序架構的一般方法，特別是身份驗證和 AJAX 實現，請參閱使用 HTTPRequest 的示例 .

了解 last_viewed 信息

每個播放器都與一個數組相關聯，該數組包含一個播放器被加載了多少次的記錄。此信息存儲在數組名稱中last_viewed .您將在下面看到此類數組的示例。請注意count記錄一天中加載特定播放器的時間。這date指定特定日期。

    [
      {
        "count": 13,
        "date": "2018-10-24T05:00:00.000Z"
      }, {
        "count": 22,
        "date": "2018-10-23T05:00:00.000Z"
      }, {
        "count": 40,
        "date": "2018-10-22T05:00:00.000Z"
      }, {
        "count": 2,
        "date": "2018-10-21T05:00:00.000Z"
      }
    ]
請注意以下事項：
  • 計數每天編譯一次。
  • 計數用於播放器加載，沒有必要實際播放視頻來添加計數。
  • 該數組限制為 30 個條目。這很可能不是連續30天，而是30天的計數不為零。
  • 彙編的數據不記錄 2018 年 10 月中旬之前的計數。
  • 如果玩家沒有數據，則該數組為空。

應用架構

此應用程序的高級應用邏輯如下：

  1. 收集必要的用戶數據以檢索帳戶中所有玩家的信息
  2. 構建JSON數據結構以傳遞到API來創建播放器
  3. 單擊按鈕後，執行AJAX調用以檢索帳戶中所有玩家的信息
  4. 在AJAX調用成功後，提取last_viewed每個玩家的數組。
  5. 對於每個玩家，顯示last_viewed表中的信息
