介紹

您可以在播放器配置中設置的屬性詳述如下（並在左側導航中列出）以帳戶ID .本文檔中還包括有關如何查看和配置屬性值的信息。

你不應該使用data-setup與 Brightcove 播放器。你可能會看到使用data-setup在 API 文檔中，但這是因為該文檔是直接從 Video.js 播放器源代碼生成的，您可以將該屬性與video使用純 Video.js 標記。該屬性向播放器發送配置信息，但 Brightcove Player 使用不同的方法來執行此任務，這使得data-setup不可靠。

瀏覽使用config.json

新的Brightcove播放器配置的實例存儲在JSON文件中。您可以通過多種方式查看此 JSON 文件或其中的一部分。

查看播放器配置的一種方法是通過將播放器的URL更改為index.html至config.json。例如：

    http://players.brightcove.net/1507807800001/HJyMlHiuZ_default/config.json

播放器配置中的一些屬性定義了以下內容：

  • 播放器的名稱和版本
  • 播放器將播放的視頻
  • 玩家所屬的賬號
  • 玩家身份證
  • 播放器可以使用的任何插件，以及插件代碼位置的網址
  • 影片播放前要展示的海報

帳戶ID

與播放器關聯的 Video Cloud 帳戶 ID。這個值無法修改 .

ad_config_id

ad_config_id信息：

  • 數據類型：字符串
  • 默認：undefined
  • 動態投放SSAI廣告配置ID。除了在播放器配置上進行設置外，還可以在運行時通過iframe播放器的adConfigId查詢字符串參數或頁內播放器的data-ad-config-id屬性進行設置。這些都將覆蓋播放器配置中的任何值。

ad_failover

ad_failover信息：

  • 數據類型：布爾值
  • 默認：undefined
  • 設置為 true 以啟用廣告故障轉移行為。這是一個多部分配置，取決於：
    • 必須在播放器上配置IMA3和SSAI插件。
    • 一個ad_config_id應該可用，否則SSAI廣告不可用。

application_id

application_id信息：

  • 數據類型：字符串
  • 默認：undefined
  • 應用程序 ID 用於在指標中區分單個播放器的多次使用。用戶看不到它。

自動前進

autoadvance信息：

  • 數據類型：字符串
  • 默認：最初未啟用
  • 與播放列表一起使用並設置播放列表的自動前進行為。

有關更多信息，請參見播放清單以下部分以及播放清單API文檔。

自動播放

autoplay信息：

  • 數據類型：布爾值或字符串
  • 默認：false
  • 指示播放器應在允許的平台上立即開始播放。

如果autoplay設置為布爾值，將使用瀏覽器的本機自動播放行為。如果將其設置為以下三個有效字符串值之一，則會發生以下情況：

  • play：播放器將手動嘗試調用播放。
  • muted：播放器將手動將播放器靜音並調用播放。
  • any：玩家將首先嘗試調用播放。如果失敗，則將播放器靜音並調用播放。

如果以上任何一項失敗，則播放器將顯示“Big Play Button"（大播放按鈕），就像自動播放設置為false一樣。

base_url（用於IP限制問題）

在北美以外使用Brightcove播放器使用IP限制時，可能會出現問題。當 Brightcove 播放器被要求從 Video Cloud 帳戶加載視頻時，它會連接到此地址的目錄https://edge.api.brightcove.com/playback/v1/ .

該 Edge 端點的行為方式可能會導致 IP 限制問題：

  • 從WITHIN North America打Edge時，直接打Edge，Edge看到的是用戶的IP地址。它可以根據IP限制白名單允許或拒絕對目錄的訪問。這是所需的功能。
  • 當從 OUTSIDE North America 訪問 Edge 時，出於負載平衡/本地緩存 CDN 的原因，它會通過 Cloudfront。這很重要，因為因此，Edge 看到的是 Cloudfront IP 而不是用戶的 IP，並拒絕所有人，因為 Cloudfront 不在白名單上（而且永遠不應該在白名單上，因為那樣任何使用 Cloudfront 的人都可以訪問）。

要解決此問題，您可以將播放器配置為使用備用 Edge 端點， edge-elb.api.brightcove.com .必須對需要從北美以外訪問 IP 受限帳戶的每個玩家進行此更改。

這種方法有一個缺點。從美國以外地區檢索緩存內容的時間將稍有延遲，因為目錄請求將發送到弗吉尼亞州，而不是通過Cloudfront的本地服務器全球網絡發送。這很可能幾乎不會引起注意。

實際的配置更改涉及為 Video Cloud 設置一個值base_url .請記住，僅當北美以外的Brightcove播放器使用IP限制時，才應更改此設置。對於現有的播放器，您可以使用以下curl命令，該命令使用PATCH HTTP方法（您必須將數據替換為環境變量$EMAIL，{account_id}和{player_id}）：

    curl
    --header "Content-Type: application/json"
    --user $EMAIL
    --request PATCH
    --data '{
        "video_cloud": {
      "base_url": "https://edge-elb.api.brightcove.com/playback/v1/"
        }
      }'
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

外掛程式

這plugins屬性指定了初始化 video.js 插件的順序以及要應用於它們的任何自定義設置。該插件必須得到相應的支持scripts屬性 URL 條目，如果需要支持stylesheets網址。每個元素plugins數組是一個帶有name屬性，還可以包含可選options財產：

  • name：video.js 插件的名稱，取自支持的 JavaScript 代碼
  • options：設置的對象哈希值將覆蓋以在初始化時提供給插件

選項子

在這個options子屬性，您在初始化時傳遞要利用的數據。以下 JSON 傳入單個對象options .

    "plugins": [{
      "name": "navigateOnVideoEnd",
      "options" : {"redirectURL": "http://docs.brightcove.com"}
    }]

這使您能夠通過播放器配置將數據傳遞給插件。例如，使用redirectURL上面顯示的對象，您將在插件本身中執行以下操作：

    videojs.registerPlugin('navigateOnVideoEnd', function (options) {
      var myPlayer = this;
      myPlayer.on("ended", function () {
        window.location.href = options.redirectURL;
      });
    });

注意：您使用標準插件實現，但使用options作為匿名函數中的參數。然後，您可以使用訪問該值options.propertyName符號。

這 選項 屬性可以是所需的任何數據結構，例如對像數組。請記住，如果您傳入一個對像數組，則必須以對像數組的形式訪問數據。

Brightcove 提供的一些插件利用插件註冊表，配置與此處顯示的不同。見概述：插件註冊表有關詳細信息的文檔。

預載

preload信息：

筆記：預加載設置實際上只是 HTML5 規範中的建議，因此在不同的瀏覽器上可能會有不同的行為。將值設置為汽車可能會對帶寬消耗產生重大影響。
  • 數據類型：字符串
  • 默認：none
  • preload 屬性通知瀏覽器視頻數據是否應該在視頻標籤加載後立即開始下載。選項是汽車 , 元數據， 和沒有任何 .

  • 汽車：表示即使用戶最終沒有觀看視頻，也可以下載整個視頻文件。從正面看，如果觀眾播放視頻，它可以立即開始播放。不利的一面是，即使觀眾沒有播放視頻，此選項也會因預加載/加載而增加帶寬消耗。

  • 元數據（默認）：加載視頻的元數據，其中包括諸如視頻的持續時間和尺寸之類的信息。此設置還將加載幾秒鐘的視頻數據。

  • 沒有任何：不要預加載任何視頻數據。這將等到用戶單擊播放開始下載。

    <video-js preload ...>
    or
    { "preload": "auto" }
筆記：使用服務器端廣告插入 (SSAI) 時，Brightcove 建議設置preload=none .如果preload被設定為auto或者metadata，將請求視頻流的第一幀，即使用戶沒有開始播放，這也會將廣告計為已投放。

query_string_to_window

query_string_to_window信息：

僅支持標準 (iframe) 播放器實現。
用於從查詢字符串參數控制全局命名空間（窗口）的填充。這在嵌入式播放器的情況下尤其有用，例如 Google 的 AMP 組件或 Facebook Instant Articles。查詢字符串的解析是使用Node.js 查詢字符串模塊 .使用此配置對象至少需要以下子屬性之一。兩者可以同時使用。
    • 名稱：query_string_to_window.target
    • 數據類型：字符串
    • description:在窗口上創建的目標對象，該目標對象將填充有完整的已解析查詢字符串。例如，使用query_string_to_window.target在播放器配置中給出以下內容： 
          {
    "query_string_to_window": {"target": "queryStringParams"}
    }
      並給出以下查詢字符串： 
          ?foo=bar&nums=1&nums=2
      將產生以下結果window.queryStringParams全局對象可供播放器和插件使用： 
          {
    foo: 'bar',
    nums: ['1', '2']
    }
    • 名稱：query_string_to_window.globals
    • 數據類型：大批
    • description:窗口對象屬性名稱的數組，將從查詢字符串中找到的值填充。上已經存在的任何匹配屬性window對像或其原型鏈將不會設置。例如，使用query_string_to_window.globals在播放器配置中給出以下內容： 
          {
    "query_string_to_window": {
      "globals": [
        "foo",
        "nums",
        "self"
      ]
    }
    }
      並給出以下查詢字符串： 
          ?foo=bar&nums=1&nums=2&self=not-allowed
      將產生以下結果window添加了以下屬性的全局對象： 
          window.foo; // 'bar'
    window.nums; // ['1', '2']

    // Because `self` already exists on the `window` object, it will not be set
    // and a warning will be logged to the browser console.
    window.self; // window

技術訂單

techOrder信息：

  • 數據類型：大批
  • 默認值：['html5'，'flash']
  • 默認情況下，Brightcove Player執行科技至上在搜索源/技術組合以播放視頻時進行排序。這意味著如果您有兩個來源和兩個技術，播放器將嘗試使用第一個技術播放每個視頻techOrder option 屬性，然後繼續嘗試下一個播放技術。見指導：播放技術文檔以獲取更多信息。

您可以使用以下 curl 語句來更改techOrder從默認['html5','flash']到['flash','html5']：

    curl \
      --header "Content-Type: application/json" \
      --user $EMAIL \
      --request PATCH \
      --data '{
          "techOrder": "['flash','html']"
         }' \
      https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration
