自動播放器更新

默認情況下，Brightcove自動定期更新所有播放器，以便他們可以利用新功能和修復功能，而無需客戶付出任何努力。雖然 Brightcove 在大多數情況下不建議選擇退出更新，但 Brightcove 意識到，有時有些客戶希望您的球員保持完全不變（例如，您已經為準備大型活動而進行了嚴格的測試，而您沒有想冒任何改變的風險）。

選擇退出自動更新

如果您發現有一段時間需要退出播放器更新，最簡單的方法是將PATCH應用於播放器的配置。以下 curl 語句執行此任務：

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
        "player": {
            "template": {
              "locked": true
            }
        }
    }' \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

這會將鎖定設置添加到您的預覽播放器。PATCH 最初僅影響您的預覽播放器，因此您需要發布您的播放器。您可以在 Studio 中或使用以下 curl 語句執行此操作：

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request POST \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish

重新選擇自動更新

當您準備好選擇重新加入更新時，您只需應用另一個更新，設置locked到false然後重新發布播放器。首先，您將鎖定更改為false：

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "locked": false
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

如果您的預覽播放器與您當前擁有的不同，這會將您的預覽播放器更新為當前播放器模板版本。您應該使用此調用返回的預覽播放器 URL 進行測試，以確保您的播放器仍按預期運行。一旦滿意，您就可以發布播放器以使新播放器上線：

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request POST \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish

切換到特定版本

您可能希望將播放器切換到特定版本。您可能會選擇退出更新，但想要使用在您選擇退出更新後出現的某些特定版本。要做到這一點：使用下面的 curl 語句，當然替換你想要的播放器的特定版本：

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "version": "1.14.11"
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

請注意，您可以同時設置版本和退出更新：

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "version": "1.14.11",
          "locked": true
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

與任何PATCH更新一樣，一旦您對更改感到滿意，就必鬚髮布播放器：

    curl \
    --header "Content-Type: application/json"cp
    --user $EMAIL \
    --request POST \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish
