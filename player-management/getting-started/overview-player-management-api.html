介紹

創建和更新播放器的過程分為兩個階段。並非將所有更改自動應用到生產播放器，而是將更改分為預覽和發布。您現在可以對預覽播放器進行所有您希望的更新，而不會影響最終用戶看到的內容。您可以選擇隨時發布更改。

播放器管理 API 允許用戶：

  1. 創建一個球員
  2. 查看播放器的設置
  3. 更改播放器的設置
  4. 列出與發布者關聯的所有玩家
  5. 發布播放器以將其提供給消費者

另請參閱API參考。

如果你還沒有經歷過一步步：球員管理，強烈建議您從此處開始。您將進行安全設置，了解系統的一些基礎知識，然後準備在這裡進一步深入。

基本網址

API 的基本 URL 是：

    https://players.api.brightcove.com/v2

介紹

玩家管理系統由三個關鍵對象組成。它們是默認模板、播放器和子播放器。

Brightcove 默認模板 - 模板是一組文件和設置，用作創建呈現給用戶的觀看體驗的基礎，以及創建所有播放器的基礎。這些文件最終會使用自定義的播放器設置進行編譯，並生成在瀏覽器（播放器）中加載的實際代碼。目前只有一個模板在系統範圍內使用。因此，無法通過API看到模板。

玩家們 - 播放器是客戶特定的設置，用於捕獲要應用於模板的一般信息和自定義。這些設置被發送到 API 和播放器對象結果。定制可以包括設置，樣式和插件。有關自定義功能的完整列表，請參見播放器配置指南。

兒童玩家 - 使用 Embed API 創建的子玩家是創建多個玩家實例的結果。播放器擁有您希望播放器擁有的大部分屬性，然後您可以使用子播放器自定義不同播放器實例上的屬性子集。子播放器只能定義要添加到給定播放器中的媒體，也可以定義任何更改其所屬普通播放器的設置或樣式替代。對父級播放器設置所做的更改將應用於從父級播放器派生的所有子級播放器。看看嵌入式API指南有關更多詳細信息。

此圖顯示了默認模板、播放器和子播放器之間的關係。

首次使用播放器管理 API 時，會在您的帳戶中生成 Brightcove 默認播放器。該玩家對象供您選擇。您可以按原樣使用它、更改其設置、重新設計樣式或添加插件。您可以選擇僅使用 Brightcove 默認播放器進行自定義，或者您可以選擇創建其他播放器來捕捉您網站上不同位置的不同外觀和設置。

發布概述

播放器管理系統的功能之一是能夠將要測試的更改與公開發布的播放器版本分開。在UI和API中，當您更改播放器的配置時，這些更改將通過播放器的預覽版本立即可用。這些預覽版本完全可用且可測試。您可以共享預覽 URL 以供批准或處理播放器預覽，直到對更改感到滿意為止，所有這些都在發布播放器以使其公開可用之前。

更新瀑布

發布玩家會影響子玩家。以下是有關發布的詳細信息：

  • 玩家發布觸發從該玩家創建的所有子玩家實例的重新發布。
  • 子播放器發布僅影響該子播放器。
  • Brightcove默認模板更新（由Brightcove控制）將觸發所有玩家的重新發布，而玩家重新發布又將觸發從每個玩家創建的所有子玩家的重新發布。

設置以子贏方式應用。這意味著子級別的設置將優先於在父級別所做的更改。考慮以下：

  • 默認模板將 CSS 值定義為紅色。
  • 播放器將相同的CSS值定義為白色。
  • 子播放器將相同的CSS值定義為藍色。

由於孩子獲勝，結果將是：

  • 播放器的 CSS 值為白色。
  • 子播放器的 CSS 值為藍色。

如果屬性的數據類型是數組，則可以更改此繼承行為。這數組字段 Embed APIs Guide 的部分描述了詳細信息。

配置選項

為了創建一個比默認播放器更高級的播放器，您必須提供一個播放器配置。配置是使用 JSON 定義的。

限速

播放器管理 API 對每個啟用的唯一 IP 都有速率限制。以下是速率限制：

請求類型 限制
發布和創建（播放器和嵌入） 60個請求/分鐘
讀取（獲取） 600個請求/分鐘
寫入（POST、PUT、PATCH 和 DELETE） 300個請求/分鐘
播放器預覽 每分鐘100個請求
所有速率限制都計入其類別的每個請求。例如，嘗試在一分鐘內執行200個POST和200個PUT將超出300個請求/分鐘的寫入請求限制。

任何導致超出限制的請求都將返回狀態代碼： 429和一個響應主體{"error": "Rate limit exceeded"}。

響應狀態碼

響應和錯誤代碼/消息基於這些由W3定義。一些最常見的代碼是：

  • 200要求成功
  • 201已創建（播放器，配置）
  • 400錯誤的請求-API調用的語法可能不正確
  • 401身份驗證無效-檢查密碼輸入是否正確，或者您是否遵循了OAuth指示正確地
  • 404找不到-檢查資源是否存在以及API調用中使用的URL是否正確
  • 429超過限速
  • 500內部服務器錯誤-嘗試滿足請求時發生錯誤
