介紹

您無需了解有關 OAuth 的任何信息即可使用該系統，我們將在下面解釋您需要執行的所有操作。有關OAuth的詳細概述，請參見Brightcove OAuth API文件。

要使用 OAuth 身份驗證，請執行以下步驟：

  1. 獲取您的客戶憑證
  2. 獲取您的訪問令牌
  3. 稱呼播放器管理 API
  4. 為了簡化步驟 2 和 3，使用服務器端代理

當與播放器管理 API，您還可以使用基本身份驗證。有關詳細信息，請參見一步步：球員管理文檔。

獲取客戶憑證

要使用 OAuth，您首先需要獲取您的客戶端憑據（client_id和client secret）。這些憑據對您正在使用的帳戶和 API 操作有效。這是一次性操作，前提是您將這些憑據保存在安全的地方以備將來使用。

接下來，您將需要獲取OAuth訪問令牌。

獲取訪問令牌

您在上一步中獲得的客戶端憑據用於請求您的訪問令牌 ( access_token )，然後提供進行 API 調用的授權。由於此令牌僅在 5 分鐘內有效，因此在大多數情況下，您需要為每次 API 調用獲取一個。

請按照以下步驟使用cURL獲取訪問令牌：

  1. 在您的命令行中，使用export命令來設置您的客戶端ID和客戶端密鑰的值。

        export CLIENT_ID=your client id value
              export CLIENT_SECRET=your client secret value

  2. 將以下cURL語句複製並粘貼到您的命令行中，然後按Enter。

        curl
              -s
              --user $CLIENT_ID:$CLIENT_SECRET
              --header "Content-Type: application/x-www-form-urlencoded"
              --data "grant_type=client_credentials"
              https://oauth.brightcove.com/v4/access_token

  3. 您的回复應該類似於：

稱呼播放器管理 API

使用 OAuth 與播放器管理 API，您將需要在上一節中生成的訪問令牌。請記住，該令牌將在5分鐘後過期，因此，如果您的訪問令牌已過期，則可能需要重複前面的步驟。

請按照以下步驟使用cURL獲取播放器配置信息：

  1. 在您的命令行中，使用export命令來設置訪問令牌，帳戶ID和玩家ID的值。

        export ACCESS_TOKEN=your access token value
              export ACCOUNT_ID=your account id
              export PLAYER_ID=your player id

  2. 將以下cURL語句複製並粘貼到您的命令行中，然後按Enter。

        curl
              --header "Content-Type: application/json"
              --header "Authorization: Bearer $ACCESS_TOKEN"
              --request GET
              https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

  3. 您的回复應該類似於：
