介紹

在本分步指南中，您將執行以下任務：

  • 創建一個球員
  • 更新播放器以自動播放視頻
  • 通過添加插件來自定義玩家
  • 顯示播放器的配置
有關本文檔中使用的 API 的詳細信息，您可以查看播放器管理 API 概述文件和播放器管理 API參考。
文檔中的步驟涉及 curl 和其他 Mac 和 Linux 系統原生的命令行工具，但不是 Windows 操作系統的一部分。如果您使用的是Windows，則可以通過安裝以下工具來獲得這些工具西格溫。在安裝過程中，將提示您安裝軟件包。打開網包，然後檢查捲曲選項，然後完成安裝。完成此操作後，您將可以按照分步操作進行操作：使用cygwin命令行成功編寫了播放器管理文檔。可以使用工具在普通的 Windows 命令行中運行 curl，但在本文檔中沒有詳細說明。

開始吧

本分步文檔採用的方法是使用 curl 語句與播放器服務 API 進行通信。curl 工具用於在命令行中使用 URL 語法傳輸數據。有關捲曲的更多信息，請參見http://curl.haxx.se。

為了簡單起見，您當然不必像在此分步文檔中那樣使用 curl' 清酒。當然，您可以使用自己喜歡的語言與 API 進行通信。看看播放器管理示例的設置該示例演示瞭如何使用基本身份驗證，AJAX和JavaScript編寫一些用於執行基本任務（例如創建播放器，顯示所有播放器，刪除播放器等）的微型應用。

開始使用API之前，需要採取一些初步步驟。這些是：

  • 登錄到Studio。如果您有多個帳戶，請使用下拉菜單選擇要在其中創建新玩家的帳戶。為了使憑據系統正常運行，您需要對此帳戶擁有管理員權限。如果不確定用戶是否具有管理員權限，請轉到“用戶設置"頁面以查看列出的用戶及其角色。
  • 在Studio中，請確保已選擇家從模塊列表中。您可以在頁面左上角的帳戶名稱下方，複製您的帳戶ID。
  • 在命令行中，輸入以下內容以將“帳戶ID"值分配給環境變量： 
            export ACCOUNT_ID=YourAccountID
  • 身份驗證將通過在curl語句中提供您的帳戶電子郵件地址來處理，然後API會提示您輸入密碼。由於電子郵件地址將被頻繁使用，因此還要將該值分配給環境變量： 
            export EMAIL=YourEmailAddress
如果您想確認環境變量已被正確分配，您可以使用echo 命令，例如：echo {account_id} .

您現在已準備好開始使用 API。

創建一個球員

您現在可以通過調用播放器管理 API 來創建播放器。此 API 調用在播放器管理 API 概述文檔。建議您將以下curl語句複製並粘貼到命令行。

以下步驟可幫助您創建播放器。

筆記：如果您不習慣使用curl語句和命令行，則存在許多REST客戶端應用程序。有關使用名為 Postman 的 REST 客戶端 Chrome 瀏覽器插件的詳細信息，請參閱使用 Postman 處理 HTTP 請求文檔。
  1. 第一個 curl 語句創建播放器並分配名稱和描述。在命令行中粘貼此語句並按 進入，系統將提示您輸入密碼。 
            curl \
          --header "Content-Type: application/json" \
          --user $EMAIL \
          --request POST \
          --data '{
            "name": "My New Player Name v2",
            "description": "My new player description"
            }' \
          https://players.api.brightcove.com/v1/accounts/{account_id}/players

    您將看到服務器以 JSON 格式返迴響應數據：

            {
          "id": "S1qN4xeG7",
          "url": "http://players.brightcove.net/1507807800001/S1qN4xeG7_default/index.html",
          "embed_code": "<iframe src='//players.brightcove.net/1507807800001/S1qN4xeG7_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
          "embed_in_page": "http://players.brightcove.net/1507807800001/S1qN4xeG7_default/in_page.embed",
          "preview_url": "http://preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/index.html",
          "preview_embed_in_page": "http://preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/in_page.embed",
          "preview_embed_code": "<iframe src='//preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>"
        }
    此時的preview_url 和preview_embed_code 不會討論。本文檔稍後將介紹預覽播放器和已發布播放器之間的差異。
  2. 使用以下一個（或全部）值查看您的播放器（此時，播放器中沒有視頻，但您可以使用 Studio 使用新創建的播放器發布視頻）：
    • 複製url 進入瀏覽器以查看您的播放器。
    • 複製embed_code 進入HTML頁面，以在iframe中查看播放器。為了使此功能正常運行，必須從實際的HTTP服務器提供包含iframe的頁面。
    • 這embed_in_page 功能的使用在高級（頁內）嵌入代碼的可用選項文檔。
    您可以在同一個 HTML 頁面上使用多個播放器。

視頻雲視頻

當然，您可以使用視頻雲庫中的視頻來創建播放器。而不是使用media JSON數據中的部分，您可以使用video_cloud 部分。下面的 curl 語句顯示了使用 Video Cloud 視頻 ID 的播放器創建。

        curl \
          --header "Content-Type: application/json" \
          --user $EMAIL \
          --request POST \
          --data '{
            "name": "Video Cloud CURL 10 March",
            "configuration": {
              "video_cloud": {
                "video": "4093372393001"
              }
            }
          }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players
您可以將視頻的參考 ID 與語法一起使用，而不是使用 ID "video": "ref:1234refid" .

響應在邏輯上等同於使用創建播放器時的響應媒體/來源/來源如上圖所示。

        "url": "http://players.brightcove.net/1507807800001/HJyMlHiuZ_default/index.html",
        "embed_code": "<iframe src="//players.brightcove.net/1507807800001/HJyMlHiuZ_default/index.html" allowfullscreen="" webkitallowfullscreen="" mozallowfullscreen=""></iframe>",
        "embed_in_page": "http://players.brightcove.net/1507807800001/HJyMlHiuZ_default/in_page.embed",
        "preview_url": "http://preview-players.brightcove.net/v1/accounts/1507807800001/players/HJyMlHiuZ/preview/embeds/default/master/index.html",

您還可以將自己的策略密鑰添加到配置中，而不是生成一個。玩家和玩家子配置交互的正常規則適用於此處：在玩家子級別設置的策略鍵將覆蓋在玩家級別設置的任何策略鍵。

更新播放器

您現在已經完成了創建播放器的基礎知識。接下來，您將學習如何對播放器進行簡單的更新。在這種情況下，如果瀏覽器允許，您將設置播放器以在播放器中自動播放視頻。

  1. 在播放器創建返回的JSON中，id 值已顯示。複製到一個PLAYER_ID 環境變量。 
            export PLAYER_ID=YourPlayerID
  2. 要更新播放器，您將使用 HTTP PATCH 方法。您將發送數據以更新您的播放器。您將設置autoplay選擇true .複製並粘貼以下curl語句並執行它。 
            curl \
          --header "Content-Type: application/json" \
          --user $EMAIL \
          --request PATCH \
          --data '{
            "autoplay": true
          }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration
  3. 檢查以確保服務器返回的數據包括id ，preview_url 和preview_embed_code 以下JSON格式的值： 
            {
          "id": "S1qN4xeG7",
          "preview_url": "http://preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/index.html",
          "preview_embed_in_page": "http://preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/in_page.embed",
          "preview_embed_code": "<iframe src='//preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>"
        }
    更新後，您將獲得播放器的預覽版本。此更改不會反映在您在更新之前使用的播放器的已發布版本中。
  4. 使用preview_url 或者preview_embed_code 值以查看播放器配置的變化。你會看到自動播放現在是真的。
  5. 現在，您將發布播放器以將其推送到正確的位置，以供用戶查看。輸入以下命令： 
            curl \
        --header "Content-Type: application/json" \
        --user $EMAIL \
        --request POST \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish

    播放器發布後，您將擁有與預覽版本不同的播放器。已發布版本與預覽版本在以下方面有所不同：

    • JavaScript 和 CSS 被壓縮、連接並直接內聯到播放器中。
    • 生成海報圖像的低分辨率版本並將其內嵌到頁面中，以改善請求建立時間較長的網絡（即蜂窩數據網絡）上的感知加載時間。
    • 以前版本的播放器會被保存，以便在更新上線後發現問題時可以恢復。
    筆記：在創建播放器時（上述步驟1），播放器也會自動發布。這就是為什麼在創建播放器時，JSON 響應包含已發布和預覽信息。
  6. 檢查以確保來自發布播放器的JSON響應如下所示： 
            {
        "id": "S1qN4xeG7",
        "url": "http://players.brightcove.net/1507807800001/S1qN4xeG7_default/index.html",
        "embed_code": "<iframe src='//players.brightcove.net/1507807800001/S1qN4xeG7_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
        "embed_in_page": "http://players.brightcove.net/1507807800001/S1qN4xeG7_default/in_page.embed"
        }
  7. 使用播放器實現之一來確保您發布的播放器正常運行。

自定義玩家

本部分假設您已通過 Studio 或播放器管理 API 將視頻與播放器相關聯。根據您的方法，生成的播放器配置可能與顯示的有所不同。

您可以使用插件自定義您的播放器。您可以學習如何在播放器中包含預先存在的插件或在播放器中構建自己的插件一步步：插件開發。

在這種情況下，您將使用一個非常簡單的預先存在的插件在視頻上覆蓋一條短信。以下是創建HTML段落標籤的插件的代碼。注意：將一個類名和文本分配給該段落，然後將其附加到現有播放器上。為了您的方便，這個插件已經存在，位於//solutions.brightcove.com/bcls/video-js/new-player/first-plugin.js .

        videojs.registerPlugin('firstPlugin', function() {
          var player = this,
          overlay = document.createElement('p');
          overlay.className = 'vjs-overlay';
          overlay.innerHTML = "First Plugin Working!";
          player.el().appendChild(overlay);
        });

需要對段落進行樣式設置，以防止在這種特定情況下出現黑色背景上的黑色文本。以下是應用於段落類名的樣式。為方便起見，此樣式表已存在，位於//solutions.brightcove.com/bcls/video-js/new-player/first-plugin.css .

        .vjs-overlay {
          background-color: #333333;
          color: white;
          font-size: 2em;
          padding: 5px;
          position: absolute;
          top: 100px;
          left: 20px;
          width: 150px;
        }

就像您使用curl語句分配視頻源和海報圖像一樣，您將使用curl語句告訴播放器以下內容：

  • 包含插件代碼的JavaScript文件的位置
  • 樣式表的位置
  • 與播放器一起使用的插件名稱
  1. 使用以下 curl 語句使插件正常運行。
    筆記：即使您沒有提供任何選項，您仍然必須輸入插件的名稱，如下所示。
            curl\
        --header "Content-Type: application/json"\
        --user $EMAIL\
        --request PATCH\
        --data '{
          "scripts": [
            "//solutions.brightcove.com/bcls/video-js/new-player/first-plugin.js"
           ],
           "stylesheets": [
            "//solutions.brightcove.com/bcls/video-js/new-player/first-plugin.css"
           ],
           "plugins": [{
             "name": "firstPlugin"
           }]
          }'\
        https: //players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration
  2. 使用以下 curl 語句發布更新的播放器。這與您在上一節中用於發布播放器的發布代碼相同。 
            curl \
          --header "Content-Type: application/json" \
          --user $EMAIL \
          --request POST \
          https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish
  3. 複製返回的URL並瀏覽。您將看到插件正在運行並顯示插件文本。
    在您看到插件正常運行之前可能會有一點延遲，因為發布播放器可能需要一些時間。如果您沒有看到該插件，請定期刷新您的頁面。

當使用播放器管理 API 將插件添加到播放器時，無論是 Brightcove 提供的插件還是您構建的自定義插件，該插件都會成為播放器代碼本身的一部分。

如果您開發自己的插件並更改源代碼，請務必重新發布您的播放器，因為除非您這樣做，否則新插件代碼不會成為 JavaScript 文件的一部分。

顯示配置

  1. 為此，請使用GET 方法與configuration 網址。

投放系統API

Delivery System API 允許管理和部署一組文件，稱為存儲庫。這些文件通過 REST API 和 git 進行管理。大多數人在創建或編輯播放器時不應該需要這些 API，但它們可以是一組非常有趣的 API，可用於其他目的。如果您想嘗試一下，可以在這裡做。有關動手操作的介紹，請嘗試逐步：輸送系統文檔。
