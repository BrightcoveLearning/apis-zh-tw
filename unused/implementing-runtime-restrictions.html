此功能適用於有權訪問 Brightcove 播放限制的有限可用性階段的一組特定客戶。

介紹

當將動態傳送與受DRM保護或HTTP實時流加密（HLSe）內容一起使用時，Brightcove播放限制提供了更高級別的安全性。

此功能使您可以通過需要令牌來授權播放來控制內容播放。需要有效的 Jason Web 令牌 (JWT) 才能獲得解密加密 HLS 所需的密鑰，或解密受 DRM 保護的內容所需的許可證。

如果您不熟悉此功能，請參閱概述：Brightcove播放限制文檔。

工作流程

要使用播放授權服務，請按照下列步驟操作：

  1. 創建一個JSON 網絡令牌
  2. 配置播放器以使用令牌

創建一個JSON 網絡令牌

要創建在與 Brightcove 的 Playback API 通信時使用的令牌，請參閱創建一個JSON 網絡令牌 (智威湯遜)文檔。

配置播放器

令牌隨播放請求一起傳遞。如果令牌無效或已過期，則將限制對內容的訪問。玩家必須配置為使用令牌。

網絡播放器

有關配置Brightcove Web播放器的詳細信息，請參閱Brightcove Player的密鑰/許可證保護文檔。

本機Android播放器

有關為 Android 配置本機播放器的詳細信息，請參閱使用本機 SDK 進行密鑰/許可證保護文檔。

本機iOS播放器

有關為 iOS 配置本機播放器的詳細信息，請參閱使用本機 SDK 進行密鑰/許可證保護文檔。

局限性

以下限制適用於此功能：

  • 對於HLSe（AES-128），令牌將附加到主清單請求中，並將傳播到加密密鑰URL。由於這些URL被附加到主清單中，因此客戶應考慮限制使用次數，以最大程度地減少其內容的顯示。
  • 對於HLSe（AES-128），播放器在播放視頻時會發出多個請求，通常每個再現一個。這馬蘇索賠需要設置得足夠高以解決這些請求。您應該考慮其他聲明，以盡量減少您的內容的曝光。切換演繹將請求新的許可證。
  • 跟踪maxu 和 maxip 聲明最終是一致的。這意味著如果使用這些聲明的令牌被快速連續重複使用多次，則可能會看到該令牌被接受的次數比這些聲明聲明的次數要多。跟踪在幾秒鐘內變得一致，在該窗口之後，令牌將按預期被阻止。
