介紹

本主題中的信息主要適用於自定義攝取配置文件。標準配置文件所有存檔主文件（請參閱以下 )

如果您在攝取視頻時將母帶存檔，則以後可以重新對視頻進行轉碼，而無需再次攝取。本主題介紹了用於存檔母版的選項。請注意，存檔母帶意味著將母帶視頻文件存儲在您的帳戶中，以便將來進行處理。看 為動態交付創建攝取配置文件 想要查詢更多的信息。

請注意，如果您在攝取視頻時存儲了數字母帶，然後使用未存檔母帶的配置文件對視頻進行了重新轉碼，則視頻的數字母帶將不是被刪除，您仍然可以重新對視頻進行轉碼。

歸檔大師

要為您的視頻存檔母版，您需要包括digital_master攝取配置文件中的對象：

    {
        "account_id": 3423487123048,
        "digital_master": {
            "rendition": "passthrough",
            "distribute": false
        },
        "name": "nondrm-default",
        ...
    }

來源或演繹

對於動態投放（包括Context Aware編碼）配置文件，您可以將視頻源文件存檔為主文件（換句話說，設置為"rendition": "passthrough"在裡面digital_master個人資料的一部分）。digital_master 將不是為這些配置文件分發（作為可用的副本推送到 CDN），即使您設置了"distribute": true .

對於“舊版攝取"配置文件，您可以選擇將視頻源文件用作主文件，也可以選擇高質量的副本作為主文件。

要將您攝取的視頻源文件用作母版，請在您的攝取配置文件中包含以下代碼：

    "digital_master": {
        "rendition": "passthrough",
        "distribute": false
    }

要使用母版的再現而不是源文件（僅限 Legacy Ingest 配置文件），請將rendition的領域digital_master反對參考編號對於演繹：

    "digital_master": {
            "rendition": "av0",
            "distribute": true
        },
        "name": "nondrm-av0",
        "packages": [],
        "renditions": [
        {
            "media_type": "video",
            "reference_id": "av0", ...

如果您設定rendition至none，沒有主人將被存檔， 而且你會不是能夠重新轉換視頻。但是，您可以代替通過再次攝取視頻。

分配

分發僅適用於舊版攝取檔案。

您可以選擇是否分發 master。分佈式意味著將母版推送到CDN並作為可播放的副本使用。如果母版未分發，則由Video Cloud存檔，但不會推送到CDN。

筆記：
  • 如果您選擇了其中一種演繹形式作為主音色，並進行了設置distribute至false，該音色將不會被推送到CDN，並且將無法播放
  • 如果將再現設置為passthrough和distribute至true，僅當母版為h264格式時才會分發母版
  • 原始視頻可能會被修改，以使其可在互聯網上播放。然而，存檔的母版將與原始文件相同，除非在極少數情況下我們需要修改它以對其進行轉碼
  • 分發大師確實不是適用於使用提取的視頻動態投放個人資料。如果您查看“動態投放"攝取配置文件，則會看到distribute設定為false。如果您複製配置文件並嘗試在Studio中進行更改，則該值將恢復為false自動地。

要將 master 分發到 CDN，請設置"distribute": true在裡面digital_master目的：

    "digital_master": {
        "distribute": true,
        "rendition": "passthrough"
    }

為了防止master被分發，設置distribute假的：

    "digital_master": {
        "distribute": false,
        "rendition": "passthrough"
    }

默認行為

默認情況下（如果配置文件不包括digital_master場），動態攝取 API和上傳模塊將表現得與配置文件中包含以下內容完全一樣：

    "digital_master": {
        "distribute": false,
        "rendition": "none"
    }

換句話說，將不會存儲任何數字主機。

如果您的個人資料中包含以下內容：

    "digital_master": {
        "distribute": false
    }

... 沒有rendition字段，用於指定應歸檔的內容（源視頻）將被存檔作為數字大師。
