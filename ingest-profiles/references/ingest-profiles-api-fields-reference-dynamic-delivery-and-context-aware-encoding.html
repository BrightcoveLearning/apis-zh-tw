個人資料欄位

如果用戶以編程方式提取配置文件並打算將其用作新自定義配置文件的基礎，則用戶將需要刪除ID , 版本 , 創建日期 , 上次修改日期， 和Brightcove_standard字段，以及姓名在帳戶中必須是新的和唯一的。
頂級配置文件選項
財產 類型 描述 DD 合格證書 默認
id 細繩 個人資料 ID（只讀） 沒有任何
version 整數 配置文件版本（只讀） 沒有任何
account_id 整數 Video Cloud 帳戶 ID 沒有任何
name 細繩 個人資料名稱 - 在帳戶中必須是唯一的 沒有任何
description 細繩 個人資料說明 沒有任何
date_created 整數（紀元時間戳（毫秒） 日期配置文件已創建（只讀） 沒有任何
date_last_modified 整數（紀元時間戳，以毫秒為單位） 日期配置文件上次修改（只讀） 沒有任何
brightcove_standard 布爾值 只讀 - false用於所有自定義配置文件 錯誤的
digital_master 目的 定義是否以及如何歸檔數字母版 {}
dynamic_origin 目的 定義演繹和圖像 {}
renditions 大批 再現對像數組；不用於DD或CAE，因此這些配置文件將為空數組 []

數字主場

這digital_master object 定義是否以及如何存儲數字母版，您可以稍後使用它進行重新轉碼。看存檔大師詳情。

數字主場
財產 類型 描述 DD 合格證書 默認
rendition 細繩 存檔的再現 - 要么passthrough或者none是 DD 和 CAE 配置文件的唯一選項 直通
distribute 布爾值 是否應該將 digital_master 推送到 CDN 以供播放；此選項當前不適用於 DD 或 CAE 配置文件，將恢復為false如果你把它設置為true 錯誤的

動態原產地

動態原產地
財產 類型 描述 DD 合格證書 默認
renditions 大批 產生的演繹數組；對於CAE，請僅包含所需的任何純音頻形式。見您可以選擇的可用演繹 []
images 大批 數組圖像海報和縮略圖的對象 []
dynamic_profile_options 目的 定義CAE配置文件的再現選項 {}

圖像場
財產 類型 描述 DD 合格證書 默認
label 細繩 圖像類型：要么poster或者thumbnail 沒有任何
height 數字 圖片高度（以像素為單位） 沒有任何

動態配置文件選項字段

當您使用上下文感知編碼時，動態配置文件選項字段為呈現創建提供指導。

動態配置文件選項
財產 類型 描述 DD 合格證書 默認
video_codecs 大批 對於混合編解碼器輸出，要為其創建再現的編解碼器數組。目前，允許的項目值為h264和hevc [2-1] 沒有任何
video_codec_options 大批 對於混合編解碼器輸出，一個對象，為每個編解碼器指定選項。每個屬性都是一個具有以下名稱的對象h264或者hevc，其屬性是該編解碼器的演繹約束集。這些對象允許的屬性是：
  • min_renditions必需的
  • min_bitrate
  • max_bitrate
  • max_first_rendition_bitrate
  • min_resolution
  • max_resolution
  • max_first_rendition_resolution
注意：這些設置也用於全局級別。當您在全局和每個編解碼器級別使用設置時，這些值必須兼容。例如，在全局級別設置的 max_renditions 必須大於或等於在每個編解碼器級別設置的 min_rendition 值的總和
 [2-2] 沒有任何
min_renditions 數字 應創建的最小再現數（必需） 沒有任何
max_renditions 數字 應創建的最大再現數（必需） 沒有任何
min_resolution 目的 再現的最低分辨率 沒有任何
min_resolution.width 數字 以像素為單位的再現的最小寬度 沒有任何
min_resolution.height 數字 以像素為單位的再現的最小高度 沒有任何
max_resolution 目的 再現的最大分辨率 默認為源分辨率
max_resolution.width 數字 以像素為單位的再現的最大寬度 源寬度
max_resolution.height 數字 以像素為單位的再現的最大高度 源高度
max_bitrate 數字 以 kbps 為單位的再現的最大比特率 源比特率
max_first_rendition_bitrate 數字 以 kbps 為單位的最低比特率再現的最大比特率 源比特率
max_frame_rate 數字 以 fps 為單位的再現的最大幀速率 30
keyframe_rate 數字 再現的每秒關鍵幀數（因此，值為 0.5 將導致每兩秒一個關鍵幀；值為 3 將導致每秒三個關鍵幀） 0.5
select_baseline_profile_configuration 布爾值 配置文件中使用的至少一個演繹格式將是基準配置文件 錯誤的
select_distinct_configurations 布爾值 如果為true，則CAE將確保沒有兩個再現使用以下相同的組合：分辨率，幀速率和編解碼器配置文件 錯誤的
watermarks 大批 數組水印指定一個或多個要添加到演繹版本中的水印的對象 錯誤的

筆記

  • [2-1] 如果需要video_codec_options存在。看HEVC視頻想要查詢更多的信息。
  • [2-2] 如果需要video_codecs存在。看HEVC視頻想要查詢更多的信息。

    • 水印字段

    水印圖像被刻錄到右下角的每個視頻幀中。

    請注意，可以通過下面顯示的字段在 CAE 配置文件中指定水印。如果要將水印添加到靜態動態傳遞攝取配置文件，請聯繫 Brightcove 支持。
    財產 類型 描述 默認
    url 細繩 圖片的網址 沒有任何
    width 細繩 [ 3-2] 水印寬度作為像素或幀寬度的百分比 - 例如20% [ 3-1] 沒有任何
    height 細繩 [ 3-2] 水印的高度作為像素或幀高度的百分比 - 例如10% [ 3-1] 沒有任何
    x 細繩 [ 3-2] 從視頻幀的左邊緣到水印中心的距離，以像素或幀寬度的百分比表示 沒有任何
    y 細繩 [ 3-2] 從視頻幀的頂部邊緣到水印中心的距離，以像素或幀寬度的百分比表示 沒有任何

    筆記

    • [ 3-1] 如果只指定寬度，圖像縱橫比將自動保持
    • [ 3-2] 雖然width , height , x， 和y字段是字符串值；它們包含數字（像素或百分比）
